## 📚 学习内容

### 1. OpenClaw 是什么

OpenClaw 是一个**智能体网关平台**，用于：
- 将 AI 智能体连接到各种消息渠道（WhatsApp, Telegram, Discord 等）
- 实现多智能体路由和隔离
- 提供统一的网关服务管理智能体、会话和工具

**核心理念**:
- Gateway (网关): 唯一的控制平面，管理所有消息渠道
- Agent (智能体): 具有独立工作空间和记忆的"大脑"
- Session (会话): 聊天历史的存储和管理
- Skill (技能): 可扩展的工具集

### 2. 架构设计

**主要组件**:
- Gateway Daemon (控制平面)
- Channels (消息渠道)
- Agents (独立智能体)
- Tools (技能工具)
- Clients (CLI, macOS app, WebUI)
- Nodes (移动设备/无头服务)

**安全模型**:
- 个人助手模式：一个受信任的操作者边界
- 不支持多租户/敌对用户共享单一网关
- 每个信任边界运行独立网关

### 3. 多智能体系统

**Agent 结构**:
- Workspace: 工作空间（AGENTS.md, SOUL.md, USER.md）
- State Directory: 状态目录（认证配置、模型注册）
- Session Store: 会话存储

**路由机制 (Bindings)**:
1. peer 匹配（精确 DM/群组 ID）
2. parentPeer 匹配（线程继承）
3. guildId + roles（Discord 角色）
4. guildId（Discord 服务器）
5. teamId（Slack 团队）
6. accountId 匹配
7. 渠道级匹配
8. 回退到默认智能体

### 4. 会话管理

**会话 Key 格式**:
- 直接聊天：`agent:<agentId>:<mainKey>`
- 隔离模式：`agent:<agentId>:dm:<peerId>`
- 群组：`agent:<agentId>:<channel>:group:<id>`
- 线程：`agent:<agentId>:<channel>:topic:<threadId>`

**DM 作用域 (dmScope)**:
- `main`: 所有 DM 共享主会话
- `per-peer`: 按发件人 ID 隔离
- `per-channel-peer`: 按渠道 + 发件人隔离（推荐）
- `per-account-channel-peer`: 按账号 + 渠道 + 发件人隔离

### 5. 技能系统

**Skills**:
- 是 Teach 智能体如何使用工具的文件夹
- 每个技能包含 SKILL.md 和可选脚本/资源文件
- 位置优先级：workspace > global > bundled

**ClawHub**:
- OpenClaw 的公共技能注册表
- 命令：`clawhub install/update/sync`

### 6. 安全模型

**信任边界**:
- Gateway 是控制平面和策略表面
- Node 是远程执行表面
- 单个 Gateway 实例中的认证操作者访问是受信任的控制平面角色

**安全加固**:
- 文件权限：`~/.openclaw` 为 700，`openclaw.json` 为 600
- 网络暴露：绑定到 loopback 或 Tailscale
- 认证：始终使用 token 或密码
- DM 策略：使用 pairing 或 allowlist

**安全审计**:
```bash
openclaw security audit --deep --fix
```

### 7. 提示注入防御

**核心原则**:
1. 身份优先：控制谁可以对话
2. 作用域优先：控制可以执行什么操作
3. 模型最后：假设模型可能被操纵

**防御措施**:
- 锁定 DM 通道 (配对/白名单)
- 群组中使用提及门控
- 将链接/附件/粘贴内容视为威胁
- 沙箱化敏感工具执行
- 使用强模型

### 8. 记忆系统

**三层架构**:
1. 每日日志：`memory/YYYY-MM-DD.md`（追加写入）
2. 精选记忆：`MEMORY.md`（长期记忆）
3. 归档：`life/archives/`（7 天后）

**记忆工具**:
- `memory_search`: 语义检索
- `memory_get`: 定向读取

**高级功能**:
- 向量搜索（多种 Provider）
- 混合搜索（BM25 + 向量）
- MMR 重排序（多样性）
- 时间衰减（新鲜度）

### 9. 上下文压缩

**作用**: 当会话接近模型上下文窗口限制时，自动总结旧消息

**配置**:
```json5
{
  agents: {
    defaults: {
      compaction: {
        reserveTokensFloor: 20000,
        softThresholdTokens: 4000
      }
    }
  }
}
```

**使用方式**:
- 自动触发
- 手动：`/compact` 命令

### 10. 插件系统

**插件类型**:
1. 渠道插件
2. 模型认证插件
3. 记忆插件
4. 上下文引擎插件

**插件注册方法**:
- Gateway RPC 方法
- HTTP 路由
- 智能体工具
- CLI 命令
- 后台服务

**插件安全**:
- 插件与 Gateway 同进程运行
- 视为可信代码
- 使用 `plugins.allow` 白名单

### 11. 最佳实践

**部署安全**:
- 绑定到 loopback 或使用 Tailscale Serve
- 始终启用认证
- 限制 DM 策略
- 使用推荐配置
- 定期安全审计

**多用户场景**:
- 为每个信任边界运行独立网关
- 启用严格 DM 隔离
- 限制工具权限
- 使用沙箱模式

**技能管理**:
- 将第三方技能视为不可信代码
- 启用技能监听器
- 定期更新
- 阅读后再启用
