### 学习内容概述

我系统性地学习了 OpenClaw 智能体网关平台的完整知识体系，包括：

#### 1. 核心概念 (10+ 模块)
- **Gateway (网关)**: 单一控制平面，管理所有消息渠道
- **Agent (智能体)**: 独立的"大脑"，拥有独立工作空间和记忆
- **Session (会话)**: 聊天历史的存储和管理
- **Skill (技能)**: 可扩展的工具集定义
- **Plugin (插件)**: 功能扩展的模块化系统
- **Node (节点)**: 远程执行表面（移动设备等）

#### 2. 架构设计
- **模块化架构**: Gateway、Agent、Session、Skill 各司其职
- **多智能体路由**: 支持多个智能体，每个有独立上下文
- **安全隔离**: 每个 Agent 有独立的工作空间、状态和会话
- **跨平台支持**: 支持多种消息渠道和移动设备集成

#### 3. 安全机制
- **信任边界**: 一个受信任的操作者边界
- **权限控制**: 最小权限原则，工具配置限制
- **提示注入防御**: 身份优先、作用域优先、模型最后
- **沙箱支持**: 运行不可信代码的隔离环境
- **认证机制**: 所有连接必须认证（token 或密码）

#### 4. 记忆系统
- **三层架构**: 每日日志、精选记忆、归档
- **向量搜索**: 支持多种 Provider（local/openai/gemini 等）
- **混合搜索**: BM25 + 向量相似度
- **MMR 重排序**: 提高结果多样性
- **时间衰减**: 提高结果新鲜度

#### 5. 会话管理
- **会话 Key 格式**: 支持 DM、群组、线程等多种场景
- **DM 作用域**: 支持多种隔离模式
- **上下文压缩**: 自动/手动触发，智能体记忆刷新
- **维护策略**: 自动清理过期会话和 transcript

#### 6. 工具系统
- **Profiles**: 预定义工具配置（messaging/coding/web/minimal/custom）
- **配置限制**: 可 deny 或 allow 特定工具
- **安全控制**: 执行权限、工作区限制

#### 7. 插件系统
- **插件类型**: 渠道插件、模型认证插件、记忆插件、上下文引擎插件
- **注册方法**: Gateway RPC、HTTP 路由、智能体工具、CLI 命令等
- **安全**: 与 Gateway 同进程，视为可信代码
