## 🎯 一句话介绍

**OpenClaw 是一个自托管的 AI Agent 运行时平台，不是聊天机器人，而是把 AI 模型连接到真实世界的桥梁。**

---

## 🏗️ 三层架构

```
┌─────────────────────────────────────────┐
│   Agent Layer（智能层）                  │
│   - 主 Agent + 子代理 + ACP Agents       │
└─────────────────────────────────────────┘
                ↓
┌─────────────────────────────────────────┐
│  Gateway Layer（网关层）← 大脑！          │
│  - 控制平面 · 路由 · 认证 · 会话管理     │
│  - **不运行 AI 模型**，只是调度员         │
└─────────────────────────────────────────┘
                ↓
┌─────────────────────────────────────────┐
│   Node Layer（节点层）← 手脚              │
│   - 设备能力（相机/屏幕/位置/通知）      │
│   - 远程执行 · 移动端 App                │
└─────────────────────────────────────────┘
```

---

## 🤖 四大核心组件

| 组件 | 职责 | 关键点 |
|------|------|--------|
| **Gateway** | 大脑、路由器 | 不运行 AI 模型 |
| **Agent** | 执行 AI 任务 | 身份 + 配置 + 状态 + 运行时 |
| **Session** | 有状态容器 | 消息历史 + 上下文压缩 |
| **Channel** | 协议适配器 | 20+ 种通讯平台 |

**Agent Loop 核心循环**:
1. 接收输入 → 2. 构建上下文 → 3. LLM 推理 → 4. 工具执行 → 5. 发送响应

### 三层记忆架构

```
┌─────────────────────────────┐
│ Session Memory (会话记忆)   │
│ - WAL Protocol 实时写入      │
└────────────┬────────────────┘
             │
┌────────────▼────────────────┐
│ Task Memory (任务记忆)      │
│ - 任务计划文件               │
│ - 子代理执行结果             │
└────────────┬────────────────┘
             │
┌────────────▼────────────────┐
│ Long-term Memory (长期记忆) │
│ - MEMORY.md (<3000 字符)    │
│ - memory/YYYY-MM-DD.md      │
│ - life/archives/ (归档)      │
└─────────────────────────────┘
```

### WAL Protocol

**规则**: 在回复之前，先把关键信息写入记忆文件

**触发条件**:
- ✏️ 修正 ("It's X, not Y")
- 📍 专有名词 (人名、地名、公司名)
- 🎨 偏好 (颜色、风格、"我喜欢")
- 📋 决策 ("Let's do X")
- 🔢 数值 (数字、日期、URL)

## 📋 常用命令

### Gateway 管理
```bash
openclaw gateway status/start/stop/restart
openclaw logs --follow
```

### Sessions 管理
```python
sessions_spawn({
  runtime: "subagent",
  agentId: "research-analyst",
  mode: "run",
  task: "研究 XX 主题"
})
```

### Cron 任务
```bash
openclaw cron add --name "每日简报" --cron "0 7 * * *"
openclaw cron list/run/remove
```

### 配置管理
```bash
openclaw config get/set/unset
```
