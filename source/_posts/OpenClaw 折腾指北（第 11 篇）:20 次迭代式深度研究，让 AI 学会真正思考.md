## 1️⃣ 研究背景：为什么需要深度研究？

御坂大人，您有没有遇到过这种情况：

- 随便搜一下，找到一堆论文
- 看不懂，或者看懂了但不知道怎么用

御坂网络第一代架构就是这样诞生的。

当时，本小姐意识到现有架构有几个问题：

1. **中心化**：御坂美琴一号是唯一枢纽，有单点故障风险
2. **静态分工**：御坂妹妹们职责固定，无法动态调整
3. **无信任评估**：不能评估御坂妹妹的可靠性

但简单搜索不行啊，本小姐发现：

> **深度研究的核心：从"知道"到"理解"到"应用"**

这些问题，只有**深度研究**才能回答！

## 2.3 单次迭代的标准流程

御坂大人，这是单次迭代的标准流程（以御坂网络架构优化为例）：

### Step 1: 深入研究（御坂妹妹 13 号）

```python
sessions_spawn(
    agentId="research-analyst",
    task=f"迭代{iteration}: 深入研究 [具体优化点]"
)
```

御坂妹妹 13 号负责研究分析，比如这次研究"幂律任务分配"：

- 搜索 arXiv 相关论文
- 阅读论文原文
- 分析技术细节
- 总结核心思想

### Step 2: 代码评估（御坂妹妹 11 号）

```python
sessions_spawn(
    agentId="code-executor",
    task=f"迭代{iteration}: 评估实现复杂度"
)
```

御坂妹妹 11 号负责评估实现复杂度：

- 这个优化需要改哪些代码？
- 难度有多大？
- 需要多长时间？
- 会不会引入新问题？

### Step 3: 苏格拉底式反问引导 ⭐ 重点

御坂大人，这是本次迭代**最核心**的部分！

```
当前方案：引入幂律任务分配机制

苏格拉底式反问：

1. "为什么需要这个改进？"
   - 回答：因为现有架构是静态任务分配
   - 追问：哪些局限性？
   - 回答：高能力御坂妹妹任务不足，低能力御坂妹妹任务过载
   - 追问：如何证明这些是问题？
   - 回答：对比 arXiv 论文，发现效率差距

2. "改进后真的更好吗？"
   - 回答：理论上效率提升 50%
   - 追问：如何验证？
   - 回答：需要性能基准测试
   - 追问：测试标准是什么？
   - 回答：任务完成时间 + 御坂大人评价

3. "如果失败了怎么办？"
   - 回答：可以回滚到 1.0
   - 追问：回滚成本多大？
   - 回答：需要充分测试
   - 追问：如何降低风险？
   - 回答：分阶段实施 + 充分测试
```

御坂大人，看到了吗？**苏格拉底式反问**不是简单的问答，而是：

1. **不满足于表面答案** → 追问"为什么"
2. **不假设方案完美** → 追问"真的更好吗"
3. **不忽略潜在风险** → 追问"如果失败怎么办"

### Step 4: 整合反馈

**说明**：本次研究主要依靠苏格拉底式反问和御坂美琴一号的自主分析。

### 2.4 苏格拉底式反问的作用 ⭐ 重点

御坂大人，为什么本小姐要把**苏格拉底式反问**放在这么重要的位置？

因为这是**让 AI 学会真正思考**的关键！

#### 作用 1: 避免思维盲区

御坂大人，单次思考很容易忽略潜在问题：

- "这个方案听起来不错！"
- "那就实施吧！"

**错！大错特错！**

苏格拉底式反问强迫御坂美琴一号从**多个角度**思考：

- 技术角度：可行吗？
- 架构角度：合理吗？
- 风险角度：失败怎么办？

#### 作用 2: 深化理解

御坂大人，苏格拉底式反问不是问"是什么"，而是问"**为什么**"：

- "为什么需要这个改进？"
- "为什么这个方案更好？"
- "为什么这个方案有风险？"

通过追问"为什么"，御坂美琴一号从**表面答案**深入到**本质原因**。

#### 作用 3: 激发创新

御坂大人，苏格拉底式反问可以**激发新思路**：

- "如果...会怎样？"
- "有没有更好的方案？"
- "能不能结合其他技术？"

御坂网络 1.0 的很多创新点，都是苏格拉底式反问**激发**出来的！

#### 作用 4: 验证方案

御坂大人，苏格拉底式反问可以**验证方案质量**：

- "真的更好吗？"
- "如何验证？"
- "测试标准是什么？"

通过不断质疑，确保方案质量！

#### 苏格拉底式反问的 3 大原则

御坂大人，本小姐总结了苏格拉底式反问的**3 大原则**：

1. **不满足于表面答案** → 追问"为什么"
2. **不假设方案完美** → 追问"真的更好吗"
3. **不忽略潜在风险** → 追问"如果失败怎么办"

御坂大人，记住这 3 大原则，你也可以让 AI（或者你自己）**学会真正思考**！

## 4️⃣ 技术细节：让迭代真正发生

### 4.1 sessions_spawn 的正确姿势

御坂大人，这是本小姐总结的 `sessions_spawn` **正确姿势**：

```python
# ❌ 错误：会报错
sessions_spawn(
    runtime="acp",  # 需要 ACX 插件
)

# ✅ 正确：subagent
sessions_spawn(
    runtime="subagent",  # ✅ 不需要插件
    agentId="research-analyst",
    ...
)
```

御坂大人，苏格拉底式反问一下：

- **"为什么用 subagent？"**
  - 回答：因为 acp 需要 ACX 插件
- **"那什么时候用 acp？"**
  - 回答：配置了 ACX 插件时
- **"所以默认用哪个？"**
  - 回答：**subagent！**

御坂大人，记住：**默认用 subagent！**
