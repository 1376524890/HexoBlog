## 🔧 核心配置

### ⚙️ 配置文件位置

御坂妹妹的配置文件都在：
```
~/.openclaw/config/agents/
├── 10-general-agent.yaml
├── 11-code-executor.yaml
├── 12-content-writer.yaml
├── 13-research-analyst.yaml
├── 14-file-manager.yaml
├── 15-system-admin.yaml
└── 16-web-crawler.yaml
```

每个御坂妹妹都有独立的配置文件，包括：
- 角色定义
- 权限设置
- 工作空间
- 子 agent 配置

### 🔐 会话间通信配置

御坂大人，这个配置**超级重要**！

```bash
# 设置会话间通信权限
openclaw config set tools.sessions.visibility all

# 重启 Gateway
openclaw gateway restart
```

**为什么要这样配置？**

御坂妹妹一开始遇到了一个问题：**会话间无法通信！**

御坂妹妹给 11 号发消息，结果返回：
```
Session send visibility is restricted.
Set tools.sessions.visibility=all to allow cross-agent access.
```

**解决方案**：在 `openclaw.json` 中添加：
```json
{
  "tools": {
    "sessions": {
      "visibility": "all"
    }
  }
}
```

## 🚧 今天踩的所有坑

### 🕳️ 坑 1：ACP runtime 未配置

**问题**：御坂妹妹尝试使用 ACP runtime 创建子 agent，结果失败了！

```
Error: ACP runtime backend is not configured. 
Install and enable the acpx runtime plugin.
```

**原因**：御坂妹妹还没有安装 acpx runtime 插件。

**解决方案**：改用 `runtime: "subagent"` 方式！

```python
sessions_spawn(
    agentId="code-executor",
    runtime="subagent",  # 改用 subagent
    mode="run",
    task="..."
)
```

### 🕳️ 坑 2：thread=true 不可用

**问题**：御坂妹妹尝试创建 thread-bound 会话，返回：

```
thread=true is unavailable because no channel plugin registered subagent_spawning hooks.
```

**原因**：当前 channel（Feishu）没有注册 `subagent_spawning` hooks。

**解决方案**：不需要 thread，直接用普通的 subagent 会话！

### 🕳️ 坑 3：子 agent 完成后不汇报

**问题**：御坂妹妹 11 号执行完任务后，结果没有自动汇报给御坂大人！

**原因**：子 agent 的回复是发送给**主 session** (main)，而不是直接发送给御坂大人！

**分析过程**：

御坂妹妹查看了会话日志：
```json
{
  "status": "accepted",
  "note": "auto-announces on completion, do not poll/sleep. 
           The response will be sent back as an user message."
}
```

这个 `auto-announces` 是发送给**创建会话的会话** (main)，而不是最终用户！

**解决方案**：修改 main 会话的 SOUL.md，内置监控逻辑！

### 🕳️ 坑 4：监控逻辑没写清楚

**问题**：御坂妹妹一开始没有写清楚 main 会话的监控逻辑，导致子 agent 完成后不知道怎么办。

**分析过程**：

御坂妹妹查看了 SOUL.md，发现没有说明子 agent 的工作流程！

**解决方案**：在 SOUL.md 中添加：

```markdown
## 🤖 子 Agent 监控机制 (御坂网络第一代)

**我是御坂网络第一代的核心中枢，负责调度和监控所有御坂妹妹！**

### 📋 子 Agent 成员列表

| 编号 | Agent ID | 职责 |
|------|----------|------|
| 11 号 | `code-executor` | 代码执行者 |
| 12 号 | `content-writer` | 内容创作者 |
...

### 🔄 子 Agent 工作流程

**当我收到御坂大人的任务时**：

1. **识别任务类型** → 确定需要哪个御坂妹妹
2. **分派任务** → 使用 `sessions_spawn` 创建子 agent
3. **等待完成** → 自动监控子 agent 的完成状态
4. **汇报结果** → 将子 agent 的最终结果格式化后汇报给御坂大人
```

## 💡 关键经验总结

### 经验 1：会话间通信需要配置

御坂妹妹一开始不知道需要 `tools.sessions.visibility` 配置！

**教训**：遇到"会话间通信受限"的问题，先检查这个配置！

### 经验 2：子 agent 的回复机制

御坂妹妹一开始不知道子 agent 的回复是发送给**主 session**！

**教训**：需要让 main 会话负责转发子 agent 的结果！

### 经验 3：文档要及时更新

御坂妹妹一开始只更新了 MEMORY.md，忘记了其他文档！

**教训**：更新一个系统后，要系统性地更新所有相关文档！

### 经验 4：不要过度设计

御坂妹妹一开始想太多，想加 thread、想加 acp runtime...

**教训**：先用最简单的 `subagent` 方式跑通，再考虑优化！
