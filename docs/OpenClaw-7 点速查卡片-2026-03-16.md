## 📌 7 点速查卡片（核心内容）

### 1️⃣ 一句话定义（必背）

> **OpenClaw 是 AI Agent 运行时平台**，不是聊天机器人，而是把 AI 连接到真实世界的桥梁。

---

### 2️⃣ 三层架构（记忆口诀）

```
智能层（脑）→ 网关层（路由）→ 节点层（手）
```

**详细结构**:
```
┌─────────────────────────┐
│ Agent Layer（智能层）   │ ← 大脑
├─────────────────────────┤
│ Gateway Layer（网关层） │ ← 路由器（不运行 AI，只是调度员）
├─────────────────────────┤
│ Node Layer（节点层）    │ ← 手脚
└─────────────────────────┘
```

---

### 3️⃣ 四大核心理念（必须背诵）⭐⭐⭐⭐⭐

| 理念 | 含义 | 重要性 |
|------|------|--------|
| **Access control before intelligence** | 访问控制先于智能 | ⭐⭐⭐⭐⭐ |
| **隐私优先** | 私有数据保持私有 | ⭐⭐⭐⭐ |
| **记忆即文件** | 所有记忆写入 Markdown 文件 | ⭐⭐⭐⭐⭐ |
| **工具优先** | 第一类工具而非 skill 包裹 | ⭐⭐⭐ |

**核心理念解读**:
1. ✅ **访问控制先于智能** - AI 可以很聪明，但必须先定义"谁能做什么"
2. ✅ **隐私优先** - 所有数据本地存储，支持自托管
3. ✅ **记忆即文件** - 纯 Markdown 文件，易于编辑和版本控制
4. ✅ **工具优先** - 通过 MCP 协议标准化 AI 与外部世界的交互

### 6️⃣ 记忆系统（三层架构）

```
Layer 1: 会话记忆（Session Memory）
  ↓
Layer 2: 任务记忆（Task Memory）
  ↓
Layer 3: 长期记忆（Long-term Memory）
  - MEMORY.md: 精选记忆
  - memory/YYYY-MM-DD.md: 每日日志
```

**最佳实践**:
1. ✅ DECIDE to write → MEMORY.md
2. ✅ Daily notes → memory/YYYY-MM-DD.md
3. ✅ 定期 review → 清理 MEMORY.md
4. ✅ Ask to remember → 重要事项写入记忆
