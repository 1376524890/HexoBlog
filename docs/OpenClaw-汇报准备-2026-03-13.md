## 📚 一、OpenClaw 是什么（核心定义）

### 一句话总结
> **OpenClaw = 智能体网关 + 多通道支持 + 技能系统 + 记忆系统 + 自动化能力**

**核心定位**: AI Agent 运行时平台（不是聊天机器人！）

### 与 ChatGPT 的关键区别

| 对比项 | ChatGPT | OpenClaw |
|--------|---------|----------|
| 定位 | 聊天机器人 | Agent 运行时平台 |
| 能力 | 生成文本 | 真正执行任务 |
| 记忆 | 会话内临时 | 持久化到磁盘文件 |
| 工具 | API 调用有限 | 文件系统、命令执行、浏览器控制等 |
| 部署 | 云端 SaaS | 本地部署，数据私有 |
| 安全性 | 受限于平台 | 多层次安全控制，审计完善 |

## 🏗️ 二、核心架构（三层 + 四组件）

### 1. 三层架构

```
用户交互层
  ↓
Gateway 层（大脑）← 核心！
  ↓
Agent 层（智能）
  ↓
Node 层（手脚）
```

**关键点**:
- **Gateway 层**: 控制平面、策略层、路由器 - **不运行 AI 模型，只是调度员**
- **Agent 层**: 执行 AI 任务，包含主会话和子代理
- **Node 层**: 远程执行表面，支持摄像头、屏幕、通知等

### 2. 四大核心组件

| 组件 | 作用 | 关键点 |
|------|------|--------|
| **Gateway** | 大脑、路由器 | 会话管理、工具调度、安全控制 |
| **Agent** | 执行 AI 任务 | 身份 + 配置 + 状态 + 运行时 |
| **Session** | 有状态容器 | 消息历史、上下文、工具状态 |
| **Channel** | 协议适配器 | Telegram、Discord、飞书等 |

### 3. Agent Loop（工作流程）

1. 接收输入 → 用户通过 Channel 发送消息
2. 构建上下文 → 组装 Session 历史、系统提示词、工具列表
3. LLM 推理 → 模型决定"直接回复"还是"调用工具"
4. 工具执行 → 如需多步骤，通过 Gateway 调用外部工具
5. 循环或结束 → 多步推理继续，否则返回最终结果
6. 发送响应 → Gateway 通过原 Channel 发送给用户

## 🧠 三、核心特性

### 1. 记忆系统（三层架构）

```
会话记忆（Session Memory）
     ↓
任务记忆（Task Memory）← memory/tasks/
     ↓
长期记忆（Long-term Memory）← MEMORY.md + memory/YYYY-MM-DD.md
```

**特点**:
- ✅ **纯 Markdown 文件** - 易于编辑和版本控制
- ✅ **向量检索** - 支持语义搜索
- ✅ **混合搜索** - BM25 + 向量相似度
- ✅ **自动压缩** - 临近 token 上限时自动整理

### 2. 工具系统（8 大分类）

| 分类 | 工具示例 | 说明 |
|------|----------|------|
| Runtime | `exec`, `process`, `gateway` | 运行时工具 |
| Filesystem | `read`, `write`, `edit` | 文件系统工具 |
| Session | `sessions_list`, `sessions_spawn` | 会话管理工具 |
| Memory | `memory_search`, `memory_get` | 记忆工具 |
| Web | `web_search`, `web_fetch` | Web 工具 |
| UI | `browser`, `canvas` | UI 工具 |
| Node | `nodes` | 节点控制 |
| Messaging | `message`, `tts` | 消息工具 |

**Feishu 集成**: `feishu_doc`, `feishu_drive`, `feishu_wiki`, `feishu_bitable_*`

### 3. 工具安全策略

**工具 Profile**:
- `minimal` - 只有 `session_status`
- `coding` - 文件系统 + 运行时 + 记忆
- `messaging` - 消息相关工具
- `full` - 无限制

**工具组（shorthands）**:
- `group:runtime`, `group:fs`, `group:sessions`, `group:memory`, `group:web`, `group:ui`, `group:messaging`, `group:nodes`

## 🔐 四、安全模型

### 核心安全原则（必背⭐⭐⭐⭐⭐）

1. **Access control before intelligence**（访问控制先于智能）
2. **隐私优先**：私有数据保持私有
3. **记忆即文件**：所有记忆写入 Markdown 文件
4. **工具优先**：第一类工具而非 skill 包裹

### 权限层级

| 级别 | 说明 | 适用对象 |
|------|------|----------|
| Level 5 | 主 Agent - 完全权限 | 主会话 |
| Level 4 | 可信子 Agent - 受限系统权限 | 需批准 |
| Level 3 | 标准子 Agent - 标准开发权限 | 开发代理 |
| Level 2 | 受限子 Agent - 严格受限权限 | 通用代理 |
| Level 1 | 只读子 Agent - 只读访问 | 读取代理 |

### 安全审计命令

```bash
openclaw security audit           # 基本检查
openclaw security audit --deep    # 深度检查
openclaw security audit --fix     # 自动修复
openclaw security audit --json    # JSON 格式
```

### 安全加固配置

```json
{
  "gateway": {
    "mode": "local",
    "bind": "loopback",
    "auth": { "mode": "token", "token": "长随机 token" }
  },
  "tools": {
    "profile": "messaging",
    "deny": ["group:automation", "group:runtime", "group:fs"]
  }
}
```
