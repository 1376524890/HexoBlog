## 📚 一、OpenClaw 是什么？

### 一句话定义
> **OpenClaw 是 AI Agent 运行时平台**，不是聊天机器人，而是把 AI 连接到真实世界的桥梁。

### 核心理念（必背）⭐⭐⭐⭐⭐
1. **Access control before intelligence**（访问控制先于智能）
2. **隐私优先**：私有数据保持私有
3. **记忆即文件**：所有记忆写入 Markdown 文件
4. **工具优先**：第一类工具而非 skill 包裹

### 关键定位
- **自托管**：在用户硬件上运行，用户完全控制数据
- **跨平台**：单一网关服务多个聊天应用（WhatsApp、Telegram、Discord、iMessage、Slack 等）
- **智能体原生**：为编码智能体设计，支持工具调用、会话、记忆和智能体路由
- **开源**：MIT 许可，社区驱动
- **不是多租户平台**：不支持相互敌对的用户共享同一个 Gateway

## 🏗️ 二、核心架构

### 2.1 三层架构

```
┌─────────────────────────────────────┐
│         Agent Layer（智能层）         │
│  - 主 Agent（御坂美琴一号）           │
│  - Subagents（子代理）                │
│  - ACP Agents（编码代理）             │
└─────────────────────────────────────┘
              ↓
┌─────────────────────────────────────┐
│       Gateway Layer（网关层）         │
│  - 控制平面、策略层、路由              │
│  - 身份认证、工具策略、会话管理        │
│  - 频道适配器 (Discord/WhatsApp 等)    │
└─────────────────────────────────────┘
              ↓
┌─────────────────────────────────────┐
│      Node Layer（节点层）             │
│  - 远程执行表面                       │
│  - 设备能力 (摄像头、屏幕、通知)       │
│  - macOS  companion app               │
└─────────────────────────────────────┘
```

**Gateway 角色**：不运行 AI 模型，只是调度员！

### 2.2 四大核心组件

| 组件 | 作用 | 关键点 |
|------|------|--------|
| **Gateway** | 大脑、路由器 | 控制平面、路由、安全、会话管理 |
| **Agent** | 执行 AI 任务 | 身份 + 配置 + 状态 + 运行时 |
| **Session** | 有状态容器 | 消息历史、上下文、工具状态 |
| **Channel** | 协议适配器 | Telegram、Discord、飞书等 |

**Agent Loop**: 接收输入 → 思考决策 → 执行动作 → 循环或发送响应

## 🧠 四、记忆系统

### 4.1 三层记忆架构

```
Layer 1: 会话记忆（Session Memory）
- 当前会话上下文
- 临时决策和中间结果
    ↓ 同步关键信息
Layer 2: 任务记忆（Task Memory）
- 任务计划文件
- 子代理执行结果
    ↓ 同步重要发现
Layer 3: 长期记忆（Long-term Memory）
- MEMORY.md：精选记忆
- memory/YYYY-MM-DD.md：每日日志
```

### 4.2 记忆工具

| 工具 | 作用 | 说明 |
|------|------|------|
| `memory_search` | 语义检索 | 在 MEMORY.md + memory/*.md 中搜索 |
| `memory_get` | 读取文件 | 读取特定记忆文件 |

**自动记忆刷新**：在压缩前触发 silent turn 存储重要记忆

### 4.3 记忆安全实践

**操作规则**：
1. ✅ 使用 `trash` 而不是 `rm`
2. ✅ 操作前备份：`cp file.md backups/file.md.bak`
3. ✅ 修改 `MEMORY.md` 前自动备份
4. ✅ 所有操作后立即 `git add` 和 `git commit`

**安全检查清单**：
- 操作前：运行安全检查脚本、检查 Git 状态、确认文件存在
- 操作中：每次操作后立即检查、使用 `trash`
- 操作后：检查文件完整性、立即提交

## 🔐 五、安全模型

### 5.1 权限层级

| 等级 | 名称 | 权限 | 说明 |
|------|------|------|------|
| Level 5 | 主 Agent | 完全权限 | 御坂美琴一号 |
| Level 4 | 可信子 Agent | 受限系统权限 | 需批准 |
| Level 3 | 标准子 Agent | 标准开发权限 | 日常操作 |
| Level 2 | 受限子 Agent | 严格受限权限 | 只读 + 特定目录 |
| Level 1 | 只读子 Agent | 只读访问 | 查看文件 |

### 5.2 核心安全原则

1. **Gateway 和 Node 属于同一信任域**
2. `sessionKey` 是路由键，不是认证 token
3. Prompt guardrails 不是安全边界
4. 本地 TUI `!` shell 是明确的操作者触发

### 5.3 安全配置建议

```json
{
  "gateway": {
    "mode": "local",
    "bind": "loopback",
    "auth": { "mode": "token", "token": "长随机 token" }
  },
  "session": { "dmScope": "per-channel-peer" },
  "tools": {
    "profile": "messaging",
    "deny": ["group:automation", "group:runtime", "group:fs"],
    "fs": { "workspaceOnly": true },
    "exec": { "security": "deny", "ask": "always" },
    "elevated": { "enabled": false }
  }
}
```

### 5.4 安全审计命令

```bash
openclaw security audit           # 基本检查
openclaw security audit --deep    # 深度检查
openclaw security audit --fix     # 自动修复
openclaw security audit --json    # JSON 格式
```

## 🧩 八、Session 与 Context

### 8.1 Session 概念

**Session key 格式**：
- **主会话**：`agent:<agentId>:<mainKey>`（默认 `main`）
- **DM 会话**：
  - `main`（默认）：所有 DM 共享主会话
  - `per-peer`：按发送者隔离
  - `per-channel-peer`：按通道 + 发送者隔离（推荐）
  - `per-account-channel-peer`：多账户场景
- **群组会话**：`agent:<agentId>:<channel>:group:<id>`

### 8.2 Context（上下文）

**上下文 = 发送给模型的所有内容**
- **系统提示**：OpenClaw 构建，包含工具列表、技能、时间、运行时信息
- **对话历史**：用户的消息 + 助手的回复
- **工具调用/结果**：命令输出、文件读取、图片等
- **限制**：受模型的上下文窗口限制（token 限制）

**查看上下文**：
- `/context list` - 快速概览
- `/context detail` - 详细分析
- `/usage tokens` - 使用统计

### 8.3 Compaction（压缩）

**当会话接近上下文窗口限制时，自动压缩历史**

- **方式**：
  - 自动压缩：接近窗口限制时触发
  - 手动压缩：`/compact` 命令
- **过程**：
  1. 总结旧对话为压缩条目
  2. 保留最近的消息
  3. 将压缩结果持久化到 JSONL
- **配置**：
  ```json
  {
    "agents": {
      "defaults": {
        "compaction": {
          "mode": "safeguard",
          "reserveTokensFloor": 24000
        }
      }
    }
  }
  ```

### 8.4 Heartbeat（心跳）

**定期在主要会话中运行的代理轮询，用于主动提醒**

**默认行为**：
- **频率**：默认 30 分钟
- **提示词**：读取 HEARTBEAT.md 并执行
- **响应**：
  - `HEARTBEAT_OK`：无事可做
  - 有提醒：直接发送提醒内容
