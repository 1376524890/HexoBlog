## 🎯 核心一句话（开场必背）⭐⭐⭐⭐⭐

> **OpenClaw 是 AI Agent 运行时平台**，不是聊天机器人，而是把 AI 连接到真实世界的桥梁。

**对比**:
- ChatGPT = 聊天机器人
- OpenClaw = Agent 运行时平台（能真正执行任务）

---

## 📐 三层架构（记忆口诀）⭐⭐⭐⭐⭐

```
Agent Layer（智能层）← 大脑
  ↓
Gateway Layer（网关层）← 路由器
  ↓
Node Layer（节点层）← 手脚
```

**Gateway 的核心职责**:
1. 生命周期管理
2. 消息路由
3. 工具协调
4. 安全控制
5. 状态持久化

**Gateway 本身不运行 AI 模型**，只是调度员！

---

## 💡 四大核心理念（必背）⭐⭐⭐⭐⭐

| 理念 | 说明 |
|------|------|
| **Access control before intelligence** | 访问控制先于智能 ⭐⭐⭐⭐⭐ |
| **隐私优先** | 私有数据保持私有 ⭐⭐⭐⭐ |
| **记忆即文件** | 所有记忆写入 Markdown 文件 ⭐⭐⭐⭐⭐ |
| **工具优先** | 第一类工具而非 skill 包裹 ⭐⭐⭐ |

## 🧠 记忆系统（三层架构）⭐⭐⭐⭐⭐

```
Layer 1: 会话记忆（Session Memory）
  ↓
Layer 2: 任务记忆（Task Memory）
  ↓
Layer 3: 长期记忆（Long-term Memory）
  - MEMORY.md: 精选记忆
  - memory/YYYY-MM-DD.md: 每日日志
```

**安全操作规则**:
1. ✅ 使用 `trash` 而不是 `rm`
2. ✅ 操作前备份
3. ✅ 检查 Git 状态
4. ✅ 立即提交

## ❓ 常见问题（30 秒内回答）⭐⭐⭐⭐⭐

**Q: OpenClaw 和 ChatGPT 的区别？**
A: ChatGPT 只是聊天机器人，OpenClaw 能真正执行任务。

**Q: 数据安全性如何保障？**
A: 自托管、三层权限模型、审计日志、沙箱隔离。

**Q: 记忆会丢失吗？**
A: 不会，记忆即文件，持久化到磁盘。

**Q: 如何扩展功能？**
A: 通过 Skills 系统，自定义或从 ClawHub 安装。

**Q: 是否支持中文？**
A: 支持，所有文档和界面都支持多语言。

**Q: 是否需要付费？**
A: OpenClaw 开源免费，但需要第三方 API。
