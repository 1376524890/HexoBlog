## 🎯 一句话介绍（开场用）

> **OpenClaw 是一个自托管的 AI Agent 运行时平台**，不是聊天机器人，而是把 AI 模型连接到真实世界的桥梁。

---

## 🏗️ 核心架构（必讲）

```
Agent Layer（大脑）
    ↓
Gateway Layer（路由器 - 不运行 AI，只是调度员）← 核心！
    ↓
Node Layer（手脚 - 设备能力、远程执行）
```

**四大组件**:
1. **Gateway** - 核心中枢，管理所有 Agent 和会话
2. **Agent** - 独立的"大脑"，有自己的工作空间
3. **Session** - 有状态的对话容器
4. **Channel** - 20+ 种通讯平台适配器

## 🔐 安全模型

**5 级权限**:
- Level 5: 主 Agent - 完全权限
- Level 4: 可信子 Agent - 需批准
- Level 3: 标准子 Agent - 开发权限
- Level 2: 受限子 Agent - 严格限制
- Level 1: 只读子 Agent - 只读访问

**审计命令**:
```bash
openclaw security audit       # 基本检查
openclaw security audit --deep # 深度检查
```

## 🚀 核心命令

### Gateway 管理
```bash
openclaw gateway status    # 查看状态
openclaw gateway restart   # 重启网关
openclaw logs --follow     # 跟踪日志
```

### 会话管理
```python
sessions_spawn({
  runtime: "subagent",
  agentId: "research-analyst",
  task: "研究 XX 主题"
})
```

### 定时任务
```bash
openclaw cron list         # 列出所有任务
openclaw cron add --name "每日简报" --cron "0 7 * * *"
```
