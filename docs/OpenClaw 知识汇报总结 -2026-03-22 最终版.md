## 📚 一、核心概念（必背）

### 1.1 什么是 OpenClaw？

**OpenClaw 不是**：
- ❌ 聊天机器人（如 ChatGPT）
- ❌ 多租户共享 AI 服务
- ❌ 云端托管的 SaaS 产品
- ❌ 单一 AI 模型的替代品

**OpenClaw 是**：
- ✅ **自托管平台**: 运行在自己的设备上，数据本地化
- ✅ **单一控制平面**: Gateway 统一管理多个渠道
- ✅ **Agent 原生**: 内置工具调用、会话管理、多智能体路由
- ✅ **开源项目**: MIT 许可，社区驱动

### 1.2 四大核心理念 ⭐⭐⭐⭐⭐

1. ✅ **Access control before intelligence**（访问控制先于智能）⭐⭐⭐⭐⭐
2. ✅ **隐私优先**: 私有数据保持私有
3. ✅ **记忆即文件**: 所有记忆写入 Markdown 文件
4. ✅ **工具优先**: 第一类工具而非 skill 包裹

**重要性排序**: 访问控制 > 隐私 > 记忆 > 工具

### 1.3 与 ChatGPT 的本质区别

| 维度 | ChatGPT | OpenClaw |
|------|---------|----------|
| 定位 | 聊天机器人 | Agent 运行时平台 |
| 能力 | 生成文本 | 真正执行任务 |
| 记忆 | 会话内临时 | 持久化到磁盘 |
| 工具 | 有限 API | 文件系统、命令、浏览器等 |
| 部署 | 云端 | 本地/私有 |
| 安全 | 受限于平台 | 多层次控制 |

└─────────────────────────────────────────────────────────────┘

### 2.1 四大核心组件

| 组件 | 职责 | 关键点 |
|------|------|--------|
| **Gateway** | 大脑、路由器 | 不运行 AI 模型，只是调度员 |
| **Agent** | 执行 AI 任务 | 身份 + 配置 + 状态 + 运行时 |
| **Session** | 有状态容器 | 消息历史、上下文、工具状态 |
| **Channel** | 协议适配器 | 20+ 种通讯平台支持 |

### 2.2 Agent Loop 核心循环

```
1. 接收输入 → 2. 构建上下文 → 3. LLM 推理 → 4. 工具执行 → 5. 发送响应
```

**关键点**: 模型拥有决策权，主动决定需要什么信息、调用什么工具、如何组织答案。

### 3.3 WAL Protocol（写入前置日志协议）

**规则**: 在回复之前，先把关键信息写入记忆文件

**触发条件**:
- ✏️ **修正**: "It's X, not Y" / "Actually..." / "No, I meant..."
- 📍 **专有名词**: 人名、地名、公司名、产品名
- 🎨 **偏好**: 颜色、风格、方法、"我喜欢/不喜欢"
- 📋 **决策**: "Let's do X" / "Go with Y" / "Use Z"
- 🔢 **数值**: 数字、日期、ID、URL

## 🔐 四、安全模型（必讲）

### 4.1 权限层级

| 级别 | 类型 | 权限范围 | 适用对象 |
|------|------|----------|----------|
| Level 5 | 主 Agent | 完全权限 | 主会话 |
| Level 4 | 可信子 Agent | 受限系统权限（需批准） | 系统管理 |
| Level 3 | 标准子 Agent | 标准开发权限 | 开发代理 |
| Level 2 | 受限子 Agent | 严格受限权限 | 通用代理 |
| Level 1 | 只读子 Agent | 只读访问 | 读取代理 |

### 4.2 安全审计命令

```bash
openclaw security audit       # 基本检查
openclaw security audit --deep # 深度检查
openclaw security audit --fix  # 自动修复
openclaw security audit --json # JSON 格式输出
```

### 4.3 信任边界

| 边界或控制 | 含义 | 常见误解 |
|----------|------|---------|
| `gateway.auth` | 认证调用者到 gateway APIs | "需要每个帧的签名" |
| `sessionKey` | 路由键用于上下文/会话选择 | "sessionKey 是用户认证边界" |
| Prompt guardrails | 减少模型滥用风险 | "prompt injection 等于认证绕过" |
| Node pairing | 配对设备的远程执行 | "设备控制应视为未授权访问" |

## 📋 六、常用命令速查

### 6.1 Gateway 管理

```bash
openclaw gateway status    # 查看网关状态
openclaw gateway start     # 启动网关
openclaw gateway stop      # 停止网关
openclaw gateway restart   # 重启网关
openclaw gateway install   # 安装为系统服务
openclaw logs --follow     # 跟踪日志
```

### 6.2 Channels 管理

```bash
openclaw channels add              # 添加新频道
openclaw channels login --channel feishu
openclaw pairing list feishu       # 查看配对请求
openclaw pairing approve feishu <CODE>  # 批准配对
```

### 6.3 Sessions 管理

```bash
# Python 方式
sessions_spawn({
  runtime: "subagent",
  agentId: "research-analyst",
  mode: "run",
  task: "研究 XX 主题"
})

# CLI 方式
openclaw sessions --json   # 列出所有会话
openclaw sessions cleanup  # 清理会话数据
```

### 6.4 定时任务（Cron）

```bash
# 添加任务
openclaw cron add \
  --name "每日简报" \
  --cron "0 7 * * *" \
  --tz "Asia/Shanghai" \
  --session isolated \
  --message "生成每日简报" \
  --announce

# 管理任务
openclaw cron list         # 列出所有任务
openclaw cron run <id>     # 立即运行
openclaw cron remove <id>  # 删除任务
```

### 6.5 配置管理

```bash
openclaw config get agents.defaults.workspace
openclaw config set agents.defaults.heartbeat.every "2h"
openclaw config unset plugins.entries.brave.config.webSearch.apiKey
```
