## 🎯 四大核心理念（必背）⭐⭐⭐⭐⭐

### 1. Access control before intelligence（访问控制先于智能）⭐⭐⭐⭐⭐

**核心原则**: 在 AI 开始工作之前，先决定谁能与系统交互、能做什么。

**为什么重要**: 
- 防止 prompt injection 导致的恶意操作
- 即使模型被欺骗，工具调用仍受限制
- 安全边界由策略层控制，而非依赖模型行为

**实现方式**:
- Channel allowlists/pairing
- Tool deny/allow lists
- Sandbox isolation
- Exec approvals

### 2. 隐私优先（Private things stay private）

**核心原则**: 私有数据保持私有，不上传到第三方。

**实现方式**:
- 自托管部署（Self-hosted）
- 本地数据存储
- 沙箱隔离
- 设备认证

### 3. 记忆即文件（Memory is files）⭐⭐⭐⭐⭐

**核心原则**: 所有记忆持久化到 Markdown 文件，不丢失。

**架构**:
```
Layer 1: 会话记忆（Session Memory）
- 当前会话上下文
- 临时决策和中间结果
  ↓ 同步关键信息
Layer 2: 任务记忆（Task Memory）
- 任务计划文件
- 子代理执行结果
  ↓ 同步重要发现
Layer 3: 长期记忆（Long-term Memory）
- MEMORY.md：精选记忆（<3000 字符）
- memory/YYYY-MM-DD.md：每日日志（无限存储）
- life/archives/: 长期归档（高价值保存）
```

**WAL Protocol（写后读协议）**:
- **STOP**: 不要立即回复
- **WRITE**: 更新记忆文件
- **THEN**: 回复用户

### 4. 工具优先（First-class tools）

**核心原则**: 工具是 typed functions，不是 skill 包裹。

**三层结构**:
1. **Tools**: 代理调用的 typed functions（如 `exec`, `browser`, `web_search`）
2. **Skills**: markdown 文件（SKILL.md），教代理何时如何使用工具
3. **Plugins**: 包，可以注册任何组合的能力（channels, tools, skills...）

### 1. Gateway（网关）

**作用**: 大脑、路由器

**关键点**: 
- 不运行 AI 模型，只是调度员
- 管理所有 Channel 连接
- 维护 Session 状态

**核心功能**:

- 消息路由
- 工具调用代理
- 会话持久化
- 身份认证
- 策略控制

**默认端口**: 18789

### 2. Agent（智能体）

**作用**: 执行 AI 任务

**组成**:
- **身份**: Agent ID 和认证信息
- **配置**: 工具列表、权限设置
- **状态**: 当前运行状态
- **运行时**: 执行环境

**Agent Loop（核心循环）⭐⭐⭐⭐⭐**

```
1. 接收输入 → 用户通过 Channel 发送消息
2. 构建上下文 → 组装 Session 历史、系统提示词、工具列表
3. LLM 推理 → 模型决定是"直接回复"还是"调用工具"
4. 工具执行 → 如需多步骤，通过 Gateway 调用外部工具
5. 循环或结束 → 多步推理继续，否则返回最终结果
6. 发送响应 → Gateway 通过原 Channel 发送给用户
```

**关键点**: 模型拥有决策权，主动决定需要什么信息、调用什么工具。

### 3. Session（会话）

**作用**: 有状态容器

**内容**:
- 消息历史
- 上下文信息
- 工具状态
- 临时决策和中间结果

**会话键格式**:
- `agent:<agentId>:main` - 直接聊天
- `agent:<agentId>:direct:<peerId>` - 直接消息
- `agent:<agentId>:<channel>:group:<id>` - 群组聊天
- `agent:<agentId>:<channel>:channel:<id>` - 频道聊天

**会话隔离**:
- Direct chats collapse into agent's main session
- Groups are isolated per channel

### 4. Channel（频道）

**作用**: 协议适配器

**支持平台（15+ 个）**:
- Telegram（最快）⭐
- WhatsApp
- Discord
- Slack
- WhatsApp Business
- iMessage
- 飞书（Feishu）
- Mattermost
- 更多...

**DM 策略**:
- `pairing`（默认）: 未知发送者需要配对码
- `allowlist`: 未知发送者被阻止
- `open`: 允许任何人 DM（需要显式 opt-in）
- `disabled`: 忽略所有 DM

### 沙箱隔离机制

**配置方式**:
```json5
{
  agents: {
    list: [
      {
        id: "restricted",
        sandbox: {
          mode: "all",
          scope: "agent",
        },
      },
    ],
  },
}
```

**沙箱类型**:
- `off`: 无沙箱（默认）
- `all`: 总是沙箱化
- `agent`: 每个 agent 一个容器
- `shared`: 共享容器

### 工具控制

```json5
{
  agents: {
    list: [
      {
        id: "restricted",
        tools: {
          allow: ["read"],
          deny: ["exec", "write", "edit"],
        },
      },
    ],
  },
}
```

### Exec 工具安全

**host 选项**:
- `sandbox`（默认）：在 Docker 容器中执行
- `gateway`：在网关主机上执行（需要审批）
- `node`：在配对设备上执行（需要审批）

**security 选项**:
- `deny`: 拒绝执行
- `allowlist`: 仅允许白名单
- `full`: 完全执行

**ask 选项**:
- `off`: 不请求批准
- `on-miss`: 未命中白名单时请求
- `always`: 总是请求批准

### 沙箱 vs 工具策略 vs Elevated（关键区别）⭐⭐⭐⭐⭐

```
┌─────────────────────────────────────────────────────────┐
│  Sandbox（沙箱）                                          │
│  - 在哪里执行（where）                                  │
│  - 隔离级别                                             │
│  - 默认：off                                            │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│  Tool Policy（工具策略）                                  │
│  - 能做什么（what）                                     │
│  - 允许/拒绝工具                                        │
│  - 默认：tools.profile="full"                           │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│  Elevated（提升权限）                                     │
│  - 谁可以触发（who）                                    │
│  - 发送者权限                                           │
│  - 默认：off                                            │
└─────────────────────────────────────────────────────────┘
```

**示例配置**:
```json5
{
  tools: {
    exec: {
      host: "sandbox",    // 在哪里执行
      security: "deny",   // 谁可以触发
      ask: "always",      // 是否需要批准
    },
  },
}
```

## 📝 学习心得

### 核心设计亮点

1. **隔离性**: 每个 agent 完全独立，互不干扰
2. **灵活性**: 支持多种部署场景（单用户、多用户、家庭、企业）
3. **安全性**: 多层权限控制，沙箱隔离，设备认证
4. **可维护性**: 自动会话清理，配置热重载，性能优化

### 系统架构优势

系统通过 WebSocket 协议统一所有通信，Gateway 作为中央枢纽管理所有渠道连接，Agent 作为独立大脑处理业务逻辑，Node 作为设备延伸扩展物理能力。这种架构非常清晰且可扩展。

### 御坂网络设计

多智能体架构通过专业分工提高效率，主 Agent 负责任务拆解和调度，子代理处理具体领域任务。这种设计既保证了专业性，又保持了系统的灵活性。
