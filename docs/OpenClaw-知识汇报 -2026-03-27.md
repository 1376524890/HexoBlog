## 🔧 四大核心组件

### 1. Gateway（网关）⭐⭐⭐⭐⭐
- **作用**: 大脑、路由器
- **关键点**: 不运行 AI 模型，只是调度员
- **默认端口**: 18789
- **连接模式**: WebSocket + typed API (JSON Schema 验证)
- **事件类型**: `agent`、`chat`、`presence`、`health`、`heartbeat`、`cron`

### 2. Agent（智能体）
- **Agent Loop**: 接收输入 → 构建上下文 → LLM 推理 → 工具执行 → 发送响应
- **关键点**: 模型拥有决策权
- **Workspace**: 每个 agent 完全独立（`~/.openclaw/workspace-<agentId>`）
- **Session Store**: `~/.openclaw/agents/<agentId>/sessions`

### 3. Session（会话）
- **作用**: 有状态容器
- **Session Key 格式**:
  - 直接聊天：`agent:main:direct:<peerId>`
  - 群组聊天：`agent:main:<channel>:group:<id>`
  - 频道聊天：`agent:main:<channel>:channel:<id>`
  - Cron 任务：`cron:<jobId>`
  - Node 运行：`node-<nodeId>`

### 4. Channel（频道）
- **支持平台**: 15+ 个（Telegram、Discord、飞书、WhatsApp 等）
- **协议适配器**: 统一接口，独立可替换

```
┌─────────────────────────────────────────────────────────┐
│ Layer 1: 会话记忆（Session Memory）                       │
│ - 当前会话上下文                                        │
│ - 临时决策和中间结果                                    │
└─────────────────────────────────────────────────────────┘
              ↓ 同步关键信息
┌─────────────────────────────────────────────────────────┐
│ Layer 2: 任务记忆（Task Memory）                          │
│ - 任务计划文件                                          │
│ - 子代理执行结果                                        │
└─────────────────────────────────────────────────────────┘
              ↓ 同步重要发现
┌─────────────────────────────────────────────────────────┐
│ Layer 3: 长期记忆（Long-term Memory）                     │
│ - MEMORY.md：精选记忆                                   │
│ - memory/YYYY-MM-DD.md：每日日志                        │
└─────────────────────────────────────────────────────────┘
```

### WAL Protocol（写后读协议）

- **STOP** — 不要立即回复
- **WRITE** — 更新记忆文件
- **THEN** — 回复用户

**触发条件**: 修正、专有名词、偏好、决策、编辑、数值

## 🔐 安全模型（必讲）

### 5 级权限体系

| 级别 | 类型 | 权限范围 |
|------|------|----------|
| Level 5 | 主 Agent | 完全权限 |
| Level 4 | 可信子 Agent | 受限系统权限（需批准）|
| Level 3 | 标准子 Agent | 标准开发权限 |
| Level 2 | 受限子 Agent | 严格受限权限 |
| Level 1 | 只读子 Agent | 只读访问 |

### 安全机制

1. **设备配对**: 所有连接包含 device identity，新设备需要批准
2. **沙箱隔离**: 每 agent 沙箱，支持 `mode: "all"` + `scope: "agent"`
3. **工具控制**: `tools.allow` / `tools.deny`，按 provider 限制
4. **审计日志**: `openclaw security audit --deep`

### 安全审计命令

```bash
openclaw security audit       # 基本检查
openclaw security audit --deep # 深度检查
openclaw security audit --fix  # 自动修复
openclaw security audit --json # JSON 格式输出
```

## 🔄 Cron vs Heartbeat

### 使用 Heartbeat 当:
- 多个检查可以批量处理（邮件 + 日历 + 通知）
- 需要对话上下文
- 时间可以稍有漂移（每~30 分钟）
- 想通过组合周期性检查减少 API 调用

### 使用 Cron 当:
- 精确时间要求（"每周一 9:00 整"）
- 任务需要与主会话历史隔离
- 想要不同的模型或思考级别
- 一次性提醒（"20 分钟后提醒我"）
- 输出应直接传递到 channel 而不涉及主会话
