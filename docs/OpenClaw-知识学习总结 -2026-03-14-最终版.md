## 核心概念

### 什么是 OpenClaw？

OpenClaw 是一个**个人助理代理平台**，特点：

- **单一信任边界**：一个 Gateway 服务于单个用户/信任边界
- **多代理架构**：一个 Gateway 可以运行多个 Agent
- **工具驱动**：通过 Skills 系统让 AI 能够执行任务
- **持久化记忆**：使用 Markdown 文件作为记忆存储
- **多通道支持**：支持 Discord、WhatsApp、Telegram、Feishu 等 20+ 个平台

## Gateway 架构

### Gateway 角色

- **控制平面**：处理 API 请求、会话路由、工具调用
- **安全边界**：执行权限控制、工具白名单、沙箱隔离
- **持久化**：管理会话状态、记忆文件、配置

### 部署模式

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| `loopback` | 仅本地访问 | 个人使用、开发测试 |
| `lan` | LAN 访问 | 家庭/办公室网络 |
| `tailnet` | Tailscale 网络 | 远程访问 |
| `custom` | 自定义绑定 | 特殊部署需求 |

**推荐默认配置**：

```json5
{
  gateway: {
    bind: "loopback",  // 默认，仅本地
    auth: {
      mode: "token",   // 必须认证
      token: "your-long-random-token"
    }
  }
}
```

### Network Exposure (网络暴露安全)

**关键原则**：

1. **默认不暴露**：`bind: "loopback"` 是唯一安全默认值
2. **需要认证**：远程访问必须配置 token/password
3. **防火墙**：LAN 暴露需要 IP 白名单
4. **Tailscale Serve**：推荐替代方案（自动 TLS + 身份验证）

**安全检查清单**：

- ✅ 使用 `openclaw security audit` 定期检查
- ✅ 配置文件权限：`chmod 700 ~/.openclaw`
- ✅ 配置档案权限：`chmod 600 ~/.openclaw/openclaw.json`
- ✅ 禁用 mDNS 广播或设置为 minimal 模式
- ✅ 启用 sandboxing 运行敏感 Agent

## 会话管理

### Session Key 结构

```
agent:main:discord:group:123456789
        ↑      ↑        ↑        ↑
     agent  channel  type    group_id
```

- `main`：主会话（直接聊天）
- `discord:group:xxx`：Discord 群组
- `telegram:channel:xxx`：Telegram 频道
- `cron:job.id`：定时任务

### 会话状态持久化

**两层存储**：

1. **Session Store** (`sessions.json`)
   - 小型可修改映射
   - 存储：会话 ID、最后活动时间、令牌计数器、设置

2. **Transcript** (`*.jsonl`)
   - 追加式对话历史
   - 树形结构（支持分支）
   - 包含：用户消息、AI 回复、工具调用、摘要

### Auto-Compaction（自动压缩）

**触发条件**：

- **Overflow**：模型返回上下文溢出错误
- **Threshold**：上下文 > 窗口 - 保留令牌

**配置**：

```json5
{
  agents: {
    defaults: {
      compaction: {
        reserveTokens: 20000,  // 默认保留空间
        keepRecentTokens: 200000  // 保留最近对话
      }
    }
  }
}
```

**Silent Memory Flush**（内存刷新）

在压缩前自动运行 silent turn 写入记忆文件，防止重要上下文丢失：

```json5
{
  agents: {
    defaults: {
      compaction: {
        memoryFlush: {
          enabled: true,
          softThresholdTokens: 4000
        }
      }
    }
  }
}
```

## 记忆系统

### 三层架构

```
┌────────────────────────────────────────┐
│        MEMORY.md (长期记忆)             │
│  • 精选记忆                             │
│  • 仅主会话加载                         │
│  • 避免组聊天泄露                        │
└────────────────────────────────────────┘
              ↑ 定期整理
┌────────────────────────────────────────┐
│    memory/YYYY-MM-DD.md (每日日志)      │
│  • 追加式记录                           │
│  • 自动读取今天 + 昨天                   │
│  • 通过 memory_search 工具访问           │
└────────────────────────────────────────┘
```

### Memory Tools

**语义搜索** (`memory_search`)：

- 返回片段 + 路径 + 行号
- 支持混合搜索（BM25 + 向量）
- 可配置 MMR 去重和时间衰减

**精准读取** (`memory_get`)：

- 读取特定文件/行范围
- 安全路径验证（仅允许 memory/目录）

### Hybrid Search（混合搜索）

```json5
{
  agents: {
    defaults: {
      memorySearch: {
        query: {
          hybrid: {
            enabled: true,
            vectorWeight: 0.7,
            textWeight: 0.3,
            mmr: {
              enabled: true,
              lambda: 0.7
            },
            temporalDecay: {
              enabled: true,
              halfLifeDays: 30
            }
          }
        }
      }
    }
  }
}
```

## 技能系统 (Skills)

### Skills 位置与优先级

```
/workspace/skills/          ← 最高优先级（用户自维护）
~/.openclaw/skills/         ← 中间优先级（本地管理）
bundled skills              ← 最低优先级（包内置）
```

**冲突规则**：同名技能，用户 workspace 优先。

### 加载过滤器

```markdown

name: gemini-cli
description: 使用 Gemini CLI 辅助编码

  {
    "openclaw": {
      "requires": {

```

**注意**：技能数量影响提示大小和 Token 消耗！

## 安全模型

### 信任边界原则

⚠️ **OpenClaw 假设单用户信任边界**

| 边界 | 含义 | 常见误解 |
|------|------|----------|
| `gateway.auth` | 认证调用者 | "需要每消息签名才算安全" |
| `sessionKey` | 路由键 | "sessionKey 是用户认证边界" |
| `exec` 批准 | 执行批准 | "完全隔离不同用户" |

### 安全审计检查项

**关键检查项（按优先级）**：

1. `fs.state_dir.perms_world_writable` - 关键
2. `gateway.bind_no_auth` - 关键
3. `security.exposure.open_groups_with_elevated` - 关键
4. `sandbox.dangerous_network_mode` - 关键
5. `models.small_params` - 警告

**运行审计**：

```bash
openclaw security audit          # 基础检查
openclaw security audit --deep   # 深度检查
openclaw security audit --fix    # 自动修复
```

### Prompt Injection 防护

**防护原则**：

1. **锁定 DM**：使用 pairing/allowlist
2. **提及门控**：群组消息必须提及 Bot
3. **内容即敌**：链接、附件、粘贴内容视为威胁
4. **沙箱隔离**：敏感工具在沙箱中运行
5. **模型选择**：使用最新、最强大的模型

## 工具系统

### Tools 配置

**推荐的安全配置**：

```json5
{
  tools: {
    profile: "messaging",  // 最小化配置
    fs: { workspaceOnly: true },
    exec: { security: "deny", ask: "always" },
    elevated: { enabled: false }
  }
}
```

**工具层级**：

- **messaging**：仅消息发送/接收
- **minimal**：添加基础工具
- **core**：核心工具集
- **full**：全部工具

### 沙箱隔离

**Docker 沙箱配置**：

```json5
{
  agents: {
    defaults: {
      sandbox: {
        mode: "docker",
        docker: {
          network: "bridge",  // 禁用 host 模式
          privileged: false   // 禁用特权模式
        }
      }
    }
  }
}
```

## 实践技巧

### Token 优化

1. **定期压缩**：使用 `/compact` 或自动压缩
2. **降低图像尺寸**：`agents.defaults.imageMaxDimensionPx: 1200`
3. **保持技能描述简短**
4. **使用缓存**：Anthropic 1M 上下文窗口的缓存策略

### 缓存策略

```json5
{
  agents: {
    defaults: {
      model: {
        primary: "anthropic/claude-opus-4-6"
      },
      models: {
        "anthropic/claude-opus-4-6": {
          params: {
            cacheRetention: "long"  // 长期缓存
          }
        }
      }
    }
  }
}
```

**保持缓存暖热**：

- 设置 heartbeat 略小于 cache TTL
- 例如：TTL 1 小时 → heartbeat 55 分钟

## 配置最佳实践

### 完整安全配置示例

```json5
{
  // Gateway 安全
  gateway: {
    bind: "loopback",
    auth: {
      mode: "token",
      token: "your-long-random-token"
    },
    discovery: {
      mdns: { mode: "minimal" }  // 禁用敏感信息广播
    }
  },

  // 会话隔离
  session: {
    dmScope: "per-channel-peer"  // DM 隔离
  },

  // 工具策略
  tools: {
    profile: "messaging",
    deny: [
      "group:automation",
      "group:runtime",
      "group:fs",
      "sessions_spawn",
      "sessions_send"
    ],
    fs: { workspaceOnly: true },
    exec: { security: "deny", ask: "always" },
    elevated: { enabled: false }
  },

  // 通道策略
  channels: {
    whatsapp: {
      dmPolicy: "pairing",
      groups: {
        "*": {
          requireMention: true
        }
      }
    },
    discord: {
      dmPolicy: "pairing",
      guilds: {
        "*": {
          requireMention: true
        }
      }
    },
    telegram: {
      dmPolicy: "pairing",
      groups: {
        "*": {
          requireMention: true
        }
      }
    }
  },

  // 模型安全
  agents: {
    defaults: {
      compaction: {
        reserveTokens: 20000
      },
      sandbox: {
        mode: "docker",
        docker: {
          network: "bridge"
        }
      }
    }
  }
}
```

## 常见问题

### Q1: Session 如何重置？

**命令**：`/new` 或 `/reset`
**效果**：创建新的 sessionID，清空当前对话

**自动重置**：

- 每日 4:00 AM（网关主机时间）
- 空闲超时（`session.reset.idleMinutes`）

### Q2: 如何查看 Token 使用？

**命令**：

- `/status` - 显示会话状态和令牌使用
- `/usage tokens` - 显示每回复的令牌计数
- `/usage cost` - 显示本地成本摘要

### Q3: 记忆文件如何管理？

**最佳实践**：

1. `memory/YYYY-MM-DD.md` - 追加记录日常事件
2. `MEMORY.md` - 定期整理，保留长期记忆
3. 修改前备份：`trash` 比 `rm` 安全
4. Git 提交：记忆文件立即 `git add && git commit`

### Q4: 如何防止 Prompt Injection？

**推荐配置**：

- 使用最新、最强的模型
- 锁定 DM 访问（pairing 模式）
- 启用沙箱
- 限制工具权限
- 避免在小模型上运行工具代理

## 🎯 关键要点总结

1. **安全优先**：单用户信任边界，默认 `loopback` + token 认证
2. **会话隔离**：DM 使用 `per-channel-peer`，群组使用提及门控
3. **记忆三层**：每日日志 → 精选记忆 → 长期归档
4. **工具最小化**：从 `messaging` 配置开始，按需扩展
5. **定期维护**：`openclaw security audit`、内存整理、压缩
6. **缓存优化**：设置 heartbeat 保持缓存暖热
