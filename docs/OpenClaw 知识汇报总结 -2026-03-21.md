## 📚 一、OpenClaw 核心概念

### 1.1 什么是 OpenClaw

**OpenClaw** 是一个**自托管网关系统**，用于连接各种聊天应用（WhatsApp、Telegram、Discord、iMessage 等）到 AI 编程代理（如 Pi、Codex、Claude Code 等）。

**核心定位**：
- 在用户的个人机器上运行单个 Gateway 进程
- 作为消息应用和始终可用的 AI 助手之间的桥梁
- 用户可以从任何地方发送消息，获取 AI 代理的响应

**目标用户**：
- 需要个人 AI 助手的开发者和高级用户
- 想在任何地方使用 AI，但不愿放弃数据控制权的人
- 不依赖托管服务，希望自托管的用户

**核心特点**：
- ✅ **自托管**：在用户自己的硬件上运行，遵守自己的规则
- ✅ **多通道**：单个 Gateway 同时服务 WhatsApp、Telegram、Discord 等
- ✅ **代理原生**：为编程代理构建，支持工具使用、会话、记忆和多代理路由
- ✅ **开源**：MIT 许可，社区驱动

---

### 1.2 三层架构

```
用户消息 → Gateway 路由 → Agent 处理 → Tool 调用 → 输出 → Gateway 返回
```

| 层级 | 职责 | 存储位置 |
|------|------|----------|
| **Gateway（网关）** | 核心中枢，管理所有 Agent 和会话 | `~/.openclaw/openclaw.json` |
| **Agent（代理）** | 独立的"大脑"，拥有自己的工作空间 | `~/.openclaw/agents/<agentId>/` |
| **Skills（技能）** | 工具扩展，教会 Agent 使用特定工具 | `~/.openclaw/skills/` 或工作区技能 |

---

### 1.3 核心组件

#### Gateway（网关）
- 管理所有 Agent 的运行
- 处理所有消息路由
- 运行定时任务
- 维护会话状态
- 处理工具调用

**存储位置**：
- 配置文件：`~/.openclaw/openclaw.json`
- 会话存储：`~/.openclaw/agents/<agentId>/sessions/sessions.json`
- 会话历史：`~/.openclaw/agents/<agentId>/sessions/<SessionId>.jsonl`
- Cron 任务：`~/.openclaw/cron/jobs.json`
- Cron 历史：`~/.openclaw/cron/runs/<jobId>.jsonl`

#### Agent（代理）
每个 Agent 是完整的"大脑"，包含：
- **工作空间**：文件、AGENTS.md/SOUL.md/USER.md、本地笔记
- **状态目录**：认证配置、模型注册表、每 Agent 配置
- **会话存储**：聊天记录和路由状态

**路径结构**：
```

  ├── auth-profiles.json          # 认证配置（每 Agent 独立）
  └── config/                     # 每 Agent 配置
```

#### Skills（技能）
Skills 是 OpenClaw 的工具扩展系统，用于教会 Agent 如何使用特定工具。

**Skills 加载路径**（优先级从高到低）：
1. 工作区技能：`<workspace>/skills`
2. 管理技能：`~/.openclaw/skills`
3. 捆绑技能：随 OpenClaw 安装包

## 🏗️ 二、系统架构详解

### 2.1 Gateway 架构

**Gateway 是核心中枢**，负责：
1. **管理所有 Agent 的运行** - 多代理支持，工作空间隔离
2. **处理所有消息路由** - 基于绑定规则（Bindings）
3. **运行定时任务** - 内置 Cron 调度器
4. **维护会话状态** - 输入/输出 token 计数
5. **处理工具调用** - 支持 20+ 工具类型

**运行模式**：
- **守护进程**：通过 systemd 运行
- **端口**：默认 18789
- **绑定**：默认 `lan`（监听 0.0.0.0）
- **Web Control UI**：http://127.0.0.1:18789/

---

### 2.2 Agent 系统

#### 单 Agent 模式（默认）
```bash
agentId: "main"
workspace: "~/.openclaw/workspace"
agentDir: "~/.openclaw/agents/main/agent"
```

#### 多 Agent 模式
支持多个独立 Agent，每个 Agent 有：
- 独立的工作空间
- 独立的会话
- 独立的认证配置
- 独立的路由规则

**路由规则（Bindings）** - 确定性路由，最具体优先：
1. `peer` 匹配（精确 DM/群组/频道 id）
2. `parentPeer` 匹配（线程继承）
3. `guildId + roles`（Discord 角色路由）
4. `guildId`（Discord）
5. `teamId`（Slack）
6. `accountId` 匹配
7. 频道级别匹配
8. 回退到默认 Agent

---

### 2.3 会话系统（Sessions）

**核心概念**：一个 Agent 一个直接聊天会话是核心。

**会话类型**：
- **直接聊天**：归并为 `agent:<agentId>:<mainKey>`
- **群组聊天**：使用各自的 key
- **Cron 任务**：独立会话或持久会话

**会话存储**：
- **Store 文件**: `~/.openclaw/agents/<agentId>/sessions/sessions.json`
- **Transcripts**: `~/.openclaw/agents/<agentId>/sessions/<SessionId>.jsonl`
- **每会话字段**: `inputTokens`, `outputTokens`, `totalTokens`, `contextTokens`

**会话映射**：
- 直接聊天：`agent:<agentId>:<mainKey>` 或 `agent:<agentId>:direct:<peerId>`
- 群组聊天：`agent:<agentId>:<channel>:group:<id>`
- 频道聊天：`agent:<agentId>:<channel>:channel:<id>`
- Cron 任务：`cron:<job.id>`（独立）或 `session:<custom-id>`（持久）
- Webhook: `hook:<uuid>`
- Node 运行：`node-<nodeId>`

**会话生命周期管理**：
1. **创建**：首次收到消息时创建
2. **维护**：存储输入/输出 token 计数
3. **重置**：每日、空闲或显式触发
4. **清洗**：自动清理过期会话

**会话重置策略**：
```json
{
  "reset": {
    "mode": "daily",           // 默认：每天凌晨 4 点重置
    "atHour": 4,
    "idleMinutes": 120,        // 可选：空闲 2 小时后重置
  },
  "resetByType": {
    "thread": { mode: "daily", atHour: 4 },
    "direct": { mode: "idle", idleMinutes: 240 },
    "group": { mode: "idle", idleMinutes: 120 },
  },
  "resetTriggers": ["/new", "/reset"],
}
```

- Mattermost

**DM 策略（dmPolicy）**：
- `pairing`（默认）：未知发送者获得一次性配对码
- `allowlist`：只有 `allowFrom` 中的发送者
- `open`：允许所有入站 DM（需要 `allowFrom: ["*"]`）
- `disabled`：忽略所有 DM

**群组策略（groupPolicy）**：
- `open`：默认允许所有群组
- `allowlist`：只有 `groupAllowFrom` 中的群组
- `disabled`：忽略所有群组

### 3.3 定时任务（Cron）

**什么是 Cron**：Gateway 的内置调度器，负责：
- 持久化任务
- 按时唤醒 Agent
- 可选地将输出推送到聊天

**三种执行模式**：
1. **Main session**：在主会话上下文中运行系统事件
2. **Isolated**：在独立的 cron 会话中运行
3. **Current session**：绑定到创建时的当前会话

**任务类型**：

1. **一次性任务**（at）：
```json
{
  "name": "Reminder",
  "schedule": { "kind": "at", "at": "2026-01-12T18:00:00Z" },
  "sessionTarget": "main",
  "payload": { "kind": "systemEvent", "text": "Reminder text" },
  "deleteAfterRun": true
}
```

2. **周期性任务**（cron）：
```json
{
  "name": "Morning brief",
  "schedule": { "kind": "cron", "expr": "0 7 * * *", "tz": "America/Los_Angeles" },
  "sessionTarget": "isolated",
  "payload": {
    "kind": "agentTurn",
    "message": "Summarize overnight updates.",
    "lightContext": true
  },
  "delivery": {
    "mode": "announce",
    "channel": "slack",
    "to": "channel:C1234567890"
  }
}
```

**CLI 使用**：
```bash
# 添加任务
openclaw cron add \
  --name "Morning brief" \
  --cron "0 7 * * *" \
  --tz "Asia/Shanghai" \
  --session isolated \
  --message "Summarize overnight updates." \
  --announce

# 管理任务
openclaw cron list                # 列出所有任务
openclaw cron run <job-id>        # 立即运行
openclaw cron runs --id <id>      # 查看运行历史
openclaw cron edit <job-id>       # 编辑任务
openclaw cron remove <job-id>     # 删除任务
```

**调度类型**：
1. **at**: 一次性时间戳
2. **every**: 固定间隔（毫秒）
3. **cron**: 5 字段 cron 表达式（或带秒的 6 字段）

**错误重试**：
- 一次性任务：瞬态错误最多重试 3 次（30 秒 → 1 分钟 → 5 分钟）
- 周期性任务：任何错误应用指数退避，任务保持启用

**轻量模式**：
```bash
openclaw cron add \
  --name "Light task" \
  --cron "0 9 * * *" \
  --session isolated \
  --message "Simple task" \
  --light-context  # 不注入工作区文件
```

### 3.4 心跳机制（Heartbeat）

**用途**：周期性检查任务，如邮件、日历、天气、提醒等。

**配置**：
```json
{
  "agents": {
    "defaults": {
      "heartbeat": {
        "every": "30m",      // 每 30 分钟
        "target": "last",    // 最后一个聊天会话
        "directPolicy": "allow"  // DM 策略
      }
    }
  }
}
```

**规则**：
- 当用户说"Read HEARTBEAT.md if it exists"时检查
- 检查内容来自 `HEARTBEAT.md` 文件
- 如果无需要关注的事项，回复 `HEARTBEAT_OK`
- 如果有新内容，报告具体内容

**检查项**（示例）：
- 邮件 - 是否有未读紧急消息？
- 日历 - 未来 24-48 小时内有事件吗？
- 提及 - 社交媒体通知？
- 天气 - 用户是否需要出行信息？

### 4.2 配置管理

**配置文件位置**：`~/.openclaw/openclaw.json`（JSON5 格式）

**配置热重载**：
- Gateway 自动监控配置文件变化
- **hybrid 模式**（默认）：安全变更立即应用，关键变更自动重启
- **hot 模式**：仅热应用安全变更，需要重启时警告
- **restart 模式**：任何变更都重启
- **off 模式**：禁用文件监控

**配置热重载分类**：
| 分类 | 字段 | 需要重启？ |
|------|------|----------|
| Channels | `channels.*` | 否 |
| Agent & models | `agent`, `agents`, `models`, `routing` | 否 |
| Automation | `hooks`, `cron`, `agent.heartbeat` | 否 |
| Sessions & messages | `session`, `messages` | 否 |
| Tools & media | `tools`, `browser`, `skills`, `audio` | 否 |
| Gateway server | `gateway.*` (port, bind, auth) | **是** |
| Infrastructure | `discovery`, `canvasHost`, `plugins` | **是** |

**CLI 配置命令**：
```bash
openclaw config get agents.defaults.workspace
openclaw config set agents.defaults.heartbeat.every "2h"
openclaw config unset plugins.entries.brave.config.webSearch.apiKey
```

**配置 RPC（程序化更新）**：
- `config.apply`：完全替换配置
- `config.patch`：部分更新（JSON 合并补丁语义）
- 速率限制：每 60 秒最多 3 次请求
