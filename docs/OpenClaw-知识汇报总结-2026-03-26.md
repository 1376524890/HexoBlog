## 🔧 Gateway 架构深化（2026-03-23 新增）⭐⭐⭐⭐⭐

### Gateway 核心职责
- **单一长期运行**: 一个 Gateway 控制单一 Baileys 会话（WhatsApp）
- **控制平面**: WebSocket 连接，typed API（JSON Schema 验证）
- **事件类型**: `agent`、`chat`、`presence`、`health`、`heartbeat`、`cron`
- **连接生命周期**:
  ```
  Client -> Gateway: req:connect
  Gateway -> Client: res (ok) + presence + health snapshot
  Client -> Gateway: req:agent
  Gateway -> Client: event:agent (streaming) + res:agent (final)
  ```
- **安全机制**:
  - **设备身份**: 所有连接包含 device identity
  - **配对流程**: 新设备需要批准，颁发 device token
  - **本地连接**: loopback 可自动批准
  - **签名验证**: 所有连接必须签名 connect.challenge

- **监听端口**: 18789
- **监听地址**: 0.0.0.0 (LAN)
- **Canvas 路径**: `/__openclaw__/canvas/` 和 `/__openclaw__/a2ui/`

## 🗄️ Session 管理（精通）⭐⭐⭐⭐⭐

### Session Key 格式
| 类型 | Session Key 格式 |
|------|------------------|
| 直接聊天（默认） | `agent:<agentId>:main` |
| 直接聊天（per-peer） | `agent:<agentId>:direct:<peerId>` |
| 直接聊天（per-channel-peer） | `agent:<agentId>:<channel>:direct:<peerId>` |
| 直接聊天（per-account-channel-peer） | `agent:<agentId>:<channel>:<accountId>:direct:<peerId>` |
| 群组聊天 | `agent:<agentId>:<channel>:group:<id>` |
| 频道聊天 | `agent:<agentId>:<channel>:channel:<id>` |
| Cron 任务 | `cron:<jobId>` |
| Webhook | `hook:<uuid>` |
| Node 运行 | `node-<nodeId>` |

### dmScope 配置（安全 DM 模式）⭐⭐⭐⭐⭐
- `main`（默认）: 所有 DM 共享主会话（单用户场景，连续性优先）
- `per-peer`: 按发送者 ID 隔离
- `per-channel-peer`: 按渠道 + 发送者隔离（**多用户推荐**）⭐⭐⭐⭐⭐
- `per-account-channel-peer`: 按账户 + 渠道 + 发送者隔离（**多账户推荐**）⭐⭐⭐⭐⭐

**安全 DM 模式必要性**:
```json5
{
  session: {
    dmScope: "per-channel-peer",  // 多用户场景必须启用！
  },
}
```

**问题场景**:
- Alice 发送私密消息（医疗预约）
- Bob 问"我们刚才说什么来着？"
- 因为共享会话，模型可能用 Alice 的上下文回答 Bob

**解决方案**: 设置 `dmScope: "per-channel-peer"` 隔离 DM 上下文

### Session 维护（2026-03-23 新增）⭐⭐⭐⭐⭐
```json5
{
  session: {
    maintenance: {
      mode: "enforce",      // 强制执行清理策略
      pruneAfter: "45d",    // 清理 45 天前的会话
      maxEntries: 800,      // 最多 800 个会话
      rotateBytes: "20mb",  // 超过 20MB 时轮换
      maxDiskBytes: "1gb",  // 磁盘上限
      highWaterBytes: "800mb", // 高水位线（80%）
    },
  },
}
```

**维护流程**:
1. 删除超过 `pruneAfter` 的旧条目
2. 限制条目数到 `maxEntries`（最老的先）
3. 归档不再引用的对话文件
4. 清理旧的归档文件
5. 旋转 `sessions.json`
6. 如果设置了 `maxDiskBytes`，强制磁盘预算

**CLI 命令**:
```bash
openclaw sessions cleanup --dry-run      # 查看将要清理的内容
openclaw sessions cleanup --enforce      # 强制执行清理
```

## 🧠 记忆系统（WAL Protocol）⭐⭐⭐⭐⭐

### 三层记忆架构

```
Layer 1: 会话记忆（Session Memory）
- 当前会话上下文
- 临时决策和中间结果
    ↓ 同步关键信息
Layer 2: 任务记忆（Task Memory）
- 任务计划文件
- 子代理执行结果
    ↓ 同步重要发现
Layer 3: 长期记忆（Long-term Memory）
- MEMORY.md：精选记忆
- memory/YYYY-MM-DD.md：每日日志
- life/archives/：长期归档
```

### WAL Protocol（写后读协议）

**核心原则**: 先写入，再读取，最后回复

```
STOP
  ↓
WRITE  → 更新 memory/YYYY-MM-DD.md 或 MEMORY.md
  ↓
THEN   → 回复用户
```

**触发条件**:
- ✅ 修正（用户纠正我的错误）
- ✅ 专有名词（人名、地名、技术名词）
- ✅ 偏好（用户喜好、习惯）
- ✅ 决策（重要决定、方案选择）
- ✅ 编辑（文件修改、配置变更）
- ✅ 数值（数量、日期、时间）

**记忆工具**:
- `memory_search(query, maxResults)` - 语义搜索
- `memory_get(path, from, lines)` - 精确读取

## ⏰ Cron vs Heartbeat（决策树）⭐⭐⭐⭐⭐

### 使用 Heartbeat 当

- ✅ 多个检查可以批量处理（邮件 + 日历 + 通知）
- ✅ 需要对话上下文
- ✅ 时间可以稍有漂移（每~30 分钟）
- ✅ 想通过组合周期性检查减少 API 调用

**Heartbeat 配置**:
```json5
{
  agents: {
    defaults: {
      heartbeat: {
        every: "30m",
        activeHours: { start: "08:00", end: "22:00" },
      },
    },
  },
}
```

**HEARTBEAT.md 示例**:
```md
# Heartbeat checklist

- Check email for urgent messages
- Review calendar for events in next 2 hours
- If a background task finished, summarize results
- If idle for 8+ hours, send a brief check-in
```

### 使用 Cron 当

- ✅ 精确时间要求（"每周一 9:00 整"）
- ✅ 任务需要与主会话历史隔离
- ✅ 想要不同的模型或思考级别
- ✅ 一次性提醒（"20 分钟后提醒我"）
- ✅ 输出应直接传递到 channel 而不涉及主会话

**Cron 配置示例**:
```bash
openclaw cron add \
  --name "Daily brief" \
  --cron "0 8 * * *" \
  --session isolated \
  --message "Summarize yesterday's work." \
  --announce \
  --channel feishu \
  --to "oc_xxx"
```

**决策流程**:
```
Does the task need to run at an EXACT time?
  YES -> Use cron
  NO  -> Continue...

Does the task need isolation from main session?
  YES -> Use cron (isolated)
  NO  -> Continue...

Can this task be batched with other periodic checks?
  YES -> Use heartbeat (add to HEARTBEAT.md)
  NO  -> Use cron

Is this a one-shot reminder?
  YES -> Use cron with --at
  NO  -> Continue...

Does it need a different model or thinking level?
  YES -> Use cron (isolated) with --model/--thinking
  NO  -> Use heartbeat
```

## ❓ 常见问题预判（Q&A）

| 问题 | 回答 |
|------|------|
| **OpenClaw 和 ChatGPT 的区别？** | ChatGPT 是聊天机器人，OpenClaw 是 Agent 运行时，能真正执行任务、控制文件、调用工具 |
| **数据安全性如何保障？** | 自托管、三层权限模型、审计日志、沙箱隔离、设备配对认证 |
| **能否在云端部署？** | 可以，但推荐本地部署保证数据私有 |
| **如何扩展功能？** | 通过 Skills 系统，自定义 Skill 或从 ClawHub 安装 |
| **是否支持中文？** | 支持，所有文档和界面都支持多语言 |
| **是否需要付费？** | 开源免费（MIT 许可），但需要第三方 API |
| **Gateway 的作用是什么？** | 网关是调度中心，不运行 AI 模型，只是路由和控制 |
| **Session 和 Chat 有什么区别？** | Session 是有状态容器，存储上下文和工具状态；Chat 只是消息流 |
| **Cron 和 Heartbeat 有什么区别？** | Cron 精确定时（每周一 9 点），Heartbeat 批量处理（每 30 分钟检查） |
| **如何保护小模型安全？** | 启用沙箱 `agents.defaults.sandbox.mode="all"`，禁用 web_search/web_fetch/browser |
