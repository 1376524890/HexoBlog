## 📌 第一部分：OpenClaw 是什么？（5 分钟）

### 一句话定义

> **OpenClaw 是 AI Agent 运行时平台**，不是聊天机器人，而是把 AI 连接到真实世界的**智能网关**。

### 核心定位

**类比理解**:
- ChatGPT = 聊天机器人（生成文本）
- OpenClaw = Agent 运行时平台（真正执行任务）

**与 ChatGPT 的关键区别**：

| 对比项 | ChatGPT | OpenClaw |
|--------|---------|----------|
| **定位** | 聊天机器人 | Agent 运行时平台 |
| **能力** | 生成文本 | 真正执行任务 |
| **记忆** | 会话内临时 | 持久化到磁盘文件 |
| **工具** | API 调用有限 | 文件系统、命令执行、浏览器控制 |
| **部署** | 云端 SaaS | 本地部署，数据私有 |
| **安全性** | 受限于平台 | 多层次安全控制，审计完善 |

### 四大核心理念（**必背** ⭐⭐⭐⭐⭐）

1. **Access control before intelligence**（访问控制先于智能）⭐⭐⭐⭐⭐
   - **第一原则**：安全控制优先于 AI 能力
   - 身份优先 → 作用域次之 → 模型最后

2. **隐私优先**：私有数据保持私有
   - 本地部署，数据不出机器
   - 可自托管，完全掌控

3. **记忆即文件**：所有记忆写入 Markdown 文件
   - 纯 Markdown 格式
   - 易于编辑和版本控制
   - 不是数据库，是文件

4. **工具优先**：第一类工具而非 skill 包裹
   - 工具是原生能力
   - 不是 skill 的包裹层
   - 直接调用，透明可控

## 🏗️ 第二部分：核心架构（10 分钟）

### 1. 三层架构

```
┌─────────────────────────────────────────────────────────┐
│  Agent Layer（智能层）                                    │
│  - 主 Agent（main）                                      │
│  - Subagents（子代理）                                  │
│  - ACP Agents（编码代理）                               │
└─────────────────────────┬───────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│  Gateway Layer（网关层）← **大脑！**                     │
│  - 控制平面、策略层、路由器                              │
│  - 身份认证、工具策略、会话管理                          │
│  - 频道适配器（Discord/WhatsApp/飞书等）                 │
│  ⚠️ Gateway 本身**不运行 AI 模型**，只是调度员              │
└─────────────────────────┬───────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│  Node Layer（节点层）← **手脚**                          │
│  - 远程执行表面                                          │
│  - 设备能力（摄像头、屏幕、通知、位置）                  │
│  - macOS companion app                                   │
└─────────────────────────────────────────────────────────┘
```

**关键点**：
- **Gateway 层**：控制平面、策略层、路由器 - **不运行 AI 模型，只是调度员**
- **Agent 层**：执行 AI 任务，包含主会话和子代理
- **Node 层**：远程执行表面，支持摄像头、屏幕、通知等

### 2. 四大核心组件

| 组件 | 作用 | 关键点 |
|------|------|--------|
| **Gateway** | 大脑、路由器 | 会话管理、工具调度、安全控制 |
| **Agent** | 执行 AI 任务 | 身份 + 配置 + 状态 + 运行时 |
| **Session** | 有状态容器 | 消息历史、上下文、工具状态 |
| **Channel** | 协议适配器 | Telegram、Discord、飞书等 |

### 3. Agent Loop（工作流程）

```
1. 接收输入 → 用户通过 Channel 发送消息
2. 构建上下文 → 组装 Session 历史、系统提示词、工具列表
3. LLM 推理 → 模型决定"直接回复"还是"调用工具"
4. 工具执行 → 如需多步骤，通过 Gateway 调用外部工具
5. 循环或结束 → 多步推理继续，否则返回最终结果
6. 发送响应 → Gateway 通过原 Channel 发送给用户
```

**示例流程**:
```
用户："帮我写一篇文章"
   ↓
Gateway 收到消息
   ↓
构建上下文：Session 历史 + 系统提示词 + 工具列表
   ↓
LLM 分析："需要写文章 → 调用 content-writer 子代理"
   ↓
Gateway 创建 content-writer 会话
   ↓
LLM 执行多步骤：搜索素材 → 撰写大纲 → 写作 → 润色
   ↓
返回最终结果
   ↓
用户收到："文章已完成，已保存到 source/_posts/xxx.md"
```

## 🧠 第三部分：核心特性（10 分钟）

### 1. 记忆系统（三层架构）

```
┌─────────────────────────────────────────────────────────┐
│  Layer 1: 会话记忆（Session Memory）                     │
│  - 当前会话上下文                                        │
│  - 临时决策和中间结果                                    │
└─────────────────────────────────────────────────────────┘
              ↓ 同步关键信息
┌─────────────────────────────────────────────────────────┐
│  Layer 2: 任务记忆（Task Memory）                        │
│  - 任务计划文件                                          │
│  - 子代理执行结果                                        │
└─────────────────────────────────────────────────────────┘
              ↓ 同步重要发现
┌─────────────────────────────────────────────────────────┐
│  Layer 3: 长期记忆（Long-term Memory）                   │
│  - MEMORY.md：精选记忆                                   │
│  - memory/YYYY-MM-DD.md：每日日志                        │
└─────────────────────────────────────────────────────────┘
```

**特点**:
- ✅ **纯 Markdown 文件** - 易于编辑和版本控制
- ✅ **向量检索** - 支持语义搜索
- ✅ **混合搜索** - BM25 + 向量相似度
- ✅ **自动压缩** - 临近 token 上限时自动整理

**文件结构**:
```
~/openclaw/workspace/
├── MEMORY.md              # 长期记忆（精选）
└── memory/
    ├── 2026-03-13.md      # 今日日志
    ├── 2026-03-12.md      # 昨日日志
    └── tasks/
        └── ACTIVE-task-id.md  # 活跃任务计划
```

**工具**:

### 记忆安全最佳实践

1. ✅ **使用 `trash` 而不是 `rm`**
2. ✅ **操作前备份**：修改 `MEMORY.md` 前自动备份到 `memory/backups/`
3. ✅ **检查 Git 状态**：操作前确认 `git status`
4. ✅ **安全检查**：运行 `python3 scripts/safety-check-memory.py`
5. ✅ **立即提交**：操作后 `git add` 和 `git commit`
