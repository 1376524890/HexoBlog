## 📚 学习内容概览

### 1. OpenClaw 定义与核心理念

**一句话定义**:
> OpenClaw 是 AI Agent 运行时平台，核心是智能网关（Runtime Gateway）。  
> **它不是聊天机器人，而是把 AI 连接到真实世界的桥梁。**

**官方文档**: https://docs.openclaw.ai

**四大核心理念（必背）**:
1. Access control before intelligence（访问控制先于智能）
2. 隐私优先：私有数据保持私有
3. 记忆即文件：所有记忆写入 Markdown 文件
4. 工具优先：第一类工具而非 skill 包裹

---

### 2. 三层架构

```
Agent Layer（智能层）← 大脑
  ↓
Gateway Layer（网关层）← 路由器（不运行 AI，只是调度员）
  ↓
Node Layer（节点层）← 手脚
```

**Gateway 深化知识点**:
- 不运行 AI 模型，只是调度员和控制平面
- WebSocket 控制平面，typed API（JSON Schema 验证）
- 事件类型：agent、chat、presence、health、heartbeat、cron
- 连接生命周期：Client→Gateway→Client（req:connect → res + presence → req:agent → event:agent）
- 安全机制：设备身份、配对流程、签名验证

### 3. Session 管理

**Session Key 格式**:
- 直接聊天：`agent:<agentId>:main` 或 `agent:<agentId>:direct:<peerId>`
- 群组聊天：`agent:<agentId>:<channel>:group:<id>`
- 频道聊天：`agent:<agentId>:<channel>:channel:<id>`
- Cron 任务：`cron:<jobId>`
- Webhook: `hook:<uuid>`
- Node 运行：`node-<nodeId>`

**dmScope 配置（安全 DM 模式）**:
- `main`: 所有 DM 共享主会话（单用户场景）
- `per-peer`: 按发送者 ID 隔离
- `per-channel-peer`: 按渠道 + 发送者隔离（**多用户推荐**）⭐⭐⭐⭐⭐
- `per-account-channel-peer`: 按账户 + 渠道 + 发送者隔离（**多账户推荐**）⭐⭐⭐⭐⭐

**Session 维护策略**:
```json
{
  "mode": "enforce",
  "pruneAfter": "45d",
  "maxEntries": 800,
  "rotateBytes": "20mb",
  "maxDiskBytes": "1gb",
  "highWaterBytes": "800mb"
}
```

### 4. Cron vs Heartbeat

**使用 Heartbeat 当**:
- ✅ 多个检查可以批量处理（邮件 + 日历 + 通知）
- ✅ 需要对话上下文
- ✅ 时间可以稍有漂移（每~30 分钟）
- ✅ 想通过组合周期性检查减少 API 调用

**使用 Cron 当**:
- ✅ 精确时间要求（"每周一 9:00 整"）
- ✅ 任务需要与主会话历史隔离
- ✅ 想要不同的模型或思考级别
- ✅ 一次性提醒（"20 分钟后提醒我"）
- ✅ 输出应直接传递到 channel 而不涉及主会话

### 6. 记忆系统（WAL Protocol）

**三层记忆架构**:
- Layer 1: 会话记忆（Session Memory）
- Layer 2: 任务记忆（Task Memory）
- Layer 3: 长期记忆（Long-term Memory）
  - MEMORY.md：精选记忆
  - memory/YYYY-MM-DD.md：每日日志

**WAL Protocol（写后读协议）**:
```
STOP — 不要立即回复
  ↓
WRITE — 更新 memory/YYYY-MM-DD.md 或 MEMORY.md
  ↓
THEN — 回复用户
```

**触发条件**: 修正、专有名词、偏好、决策、编辑、数值

**记忆工具**:
- `memory_search(query, maxResults)` - 语义搜索
- `memory_get(path, from, lines)` - 精确读取

### 8. 安全模型

**5 级权限体系**:
| 级别 | 类型 | 权限范围 |
|------|------|----------|
| Level 5 | 主 Agent | 完全权限 |
| Level 4 | 可信子 Agent | 受限系统权限（需批准） |
| Level 3 | 标准子 Agent | 标准开发权限 |
| Level 2 | 受限子 Agent | 严格受限权限 |
| Level 1 | 只读子 Agent | 只读访问 |

**安全机制**:
- 设备配对：所有连接包含 device identity，新设备需要批准
- 沙箱隔离：每 agent 沙箱，支持 `mode: "all"` + `scope: "agent"`
- 工具控制：`tools.allow` / `tools.deny`，按 provider 限制
- 审计日志：`openclaw security audit --deep`
- 速率限制：`gateway.auth.rateLimit`（可选配置）

**安全审计命令**:
```bash
openclaw security audit       # 基本检查
openclaw security audit --deep # 深度检查
openclaw security audit --fix  # 自动修复
```
