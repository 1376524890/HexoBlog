## 🎯 一句话定义（开场必讲）

> **OpenClaw 是 AI Agent 运行时平台**，不是聊天机器人，而是把 AI 连接到真实世界的桥梁。

**官方定义**: OpenClaw 是一个**自托管网关（self-hosted gateway）**，将聊天应用（WhatsApp、Telegram、Discord、iMessage 等）与 AI 编码智能体连接起来。

**核心特点**:
1. ✅ **自托管** - 数据在自己手中
2. ✅ **多通道** - 一个网关服务多个平台
3. ✅ **智能体原生** - 为 AI 编码智能体设计
4. ✅ **开源** - MIT 许可

## 🏗️ 三层架构（必会画图）⭐⭐⭐⭐⭐

```
Agent Layer（智能层）
  ↓
Gateway Layer（网关层）← 大脑！不运行 AI，只是调度员
  ↓
Node Layer（节点层）← 手脚
```

### 四大核心组件

| 组件 | 作用 | 关键点 |
|------|------|--------|
| **Gateway** | 大脑、路由器 | 不运行 AI 模型 |
| **Agent** | 执行 AI 任务 | 身份 + 配置 + 状态 |
| **Session** | 有状态容器 | 消息历史、上下文 |
| **Channel** | 协议适配器 | Telegram、Discord 等 |

### Agent Loop

```
1. 接收输入 → 用户发送消息
2. 构建上下文 → 组装历史、提示词、工具
3. LLM 推理 → 决定回复或调用工具
4. 工具执行 → 调用外部工具
5. 循环或结束 → 返回最终结果
6. 发送响应 → 发送给用户
```
