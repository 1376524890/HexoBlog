## 🧩 核心概念（必问）

### 1. Gateway（网关）
- 维护所有通道连接
- 通过 WebSocket 暴露 API
- 是会话、路由的唯一事实来源

### 2. Session（会话）
- 每条对话是一个独立会话
- 支持精细的会话隔离
- 自动维护清理（默认 30 天）

### 3. Agent（代理）
- 每个代理有独立工作空间和配置
- 支持多代理路由
- 每代理沙箱隔离

### 4. Node（节点）
- iOS/Android/macos/headless
- 提供硬件访问（摄像头、屏幕等）
- 通过 Gateway 统一管理

## 🔐 安全要点（必问）

1. **DM 安全模式**：`dmScope: "per-channel-peer"`（多用户场景必须）
2. **认证令牌**：所有连接必须签署 nonce
3. **工具策略**：通过 allow/deny 控制权限
4. **沙箱隔离**：每代理独立沙箱

**安全警告**：如果代理接收多人的 DM，必须启用安全 DM 模式，否则隐私信息会泄露！

## 📊 配置要点

### 会话管理配置
```json
{
  "session": {
    "dmScope": "per-channel-peer", // 安全 DM 模式
    "maintenance": {
      "mode": "enforce", // 自动清理
      "pruneAfter": "30d",
      "maxEntries": 500
    }
  }
}
```

### 多代理路由配置
```json
{
  "agents": {
    "list": [
      {"id": "alex", "workspace": "~/.openclaw/workspace-alex"},
      {"id": "mia", "workspace": "~/.openclaw/workspace-mia"}
    ]
  },
  "bindings": [
    {"agentId": "alex", "match": {"channel": "whatsapp", "peer": {"id": "+123"}}},
    {"agentId": "mia", "match": {"channel": "whatsapp", "peer": {"id": "+456"}}}
  ]
}
```

## 🔄 工作流程示例

### 用户发消息
1. 消息发送到 Gateway
2. Gateway 匹配绑定规则
3. 路由到对应代理
4. 代理使用工具处理

## ❓ 常见问题

### Q: OpenClaw 和 Pi 有什么区别？
A: OpenClaw 是网关，Pi 是代理。OpenClaw 可以连接 Pi 或其他模型。

### Q: 支持本地模型吗？
A: 支持！可以连接 vLLM、Ollama 等本地模型。

### Q: 数据安全吗？
A: 非常安全。自托管，所有数据在你的设备上。

### Q: 支持中文吗？
A: 支持！取决于你选择的模型。

### Q: 如何添加新功能？
A: 可以通过 Skills（技能）或 Plugins（插件）扩展。
