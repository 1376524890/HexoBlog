## 📌 一句话介绍

> **OpenClaw 是一个自托管的 AI Agent 运行时平台**，核心是**智能网关（Runtime Gateway）**。  
> 它不是聊天机器人，而是把 AI 模型连接到真实世界的桥梁。通过单一 Gateway 控制平面，可以统一管理 20+ 消息渠道（WhatsApp、Telegram、Discord、飞书等），实现多智能体协作。

---

## 🎯 四大核心理念（必背）⭐⭐⭐⭐⭐

1. **Access control before intelligence**（访问控制先于智能）⭐⭐⭐⭐⭐
2. **隐私优先**：私有数据保持私有
3. **记忆即文件**：所有记忆写入 Markdown 文件
4. **工具优先**：第一类工具而非 skill 包裹

## 🔧 四大核心组件

| 组件 | 作用 | 关键点 |
|------|------|--------|
| **Gateway** | 大脑、路由器 | 不运行 AI 模型，只是调度员 |
| **Agent** | 执行 AI 任务 | 身份 + 配置 + 状态 + 运行时 |
| **Session** | 有状态容器 | 消息历史、上下文、工具状态 |
| **Channel** | 协议适配器 | Telegram、Discord、飞书等 15+ 个 |

**Agent Loop 流程**: 接收输入 → 构建上下文 → LLM 推理 → 工具执行 → 循环或发送响应

## 🧠 三层记忆架构

```
Layer 1: 会话记忆（Session Memory）
  - 当前会话上下文
  - 临时决策和中间结果
      ↓ 同步关键信息
Layer 2: 任务记忆（Task Memory）
  - 任务计划文件
  - 子代理执行结果
      ↓ 同步重要发现
Layer 3: 长期记忆（Long-term Memory）
  - MEMORY.md：精选记忆
  - memory/YYYY-MM-DD.md：每日日志
```

**WAL Protocol（写入前置日志协议）**：
- 在回复之前，先把关键信息写入 `SESSION-STATE.md`
- 触发条件：修正、专有名词、偏好、决策、数值

## 🔐 安全模型

### 安全核心理念
- ✅ **Access control before intelligence**（访问控制先于智能）
- ✅ **隐私优先**: 私有数据保持私有
- ✅ **记忆即文件**: 所有记忆写入 Markdown 文件
- ✅ **工具优先**: 第一类工具而非 skill 包裹

### 权限层级

| 级别 | 类型 | 权限范围 |
|------|------|----------|
| Level 5 | 主 Agent | 完全权限 |
| Level 4 | 可信子 Agent | 受限系统权限（需批准） |
| Level 3 | 标准子 Agent | 标准开发权限 |
| Level 2 | 受限子 Agent | 严格受限权限 |
| Level 1 | 只读子 Agent | 只读访问 |

### 安全配置示例
```json5
{
  gateway: {
    mode: "local",
    bind: "loopback",
    auth: { mode: "token", token: "replace-with-long-random-token" },
  },
  session: {
    dmScope: "per-channel-peer",  // 隔离 DM 会话
  },
  tools: {
    profile: "messaging",
    deny: ["group:automation", "group:runtime", "group:fs", "sessions_spawn"],
    fs: { workspaceOnly: true },
  },
}
```

### 安全审计命令
