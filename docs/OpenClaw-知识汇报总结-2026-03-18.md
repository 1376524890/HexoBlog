## 📚 一、OpenClaw 核心概念

### 1.1 什么是 OpenClaw？

OpenClaw 是一个**自托管网关（self-hosted gateway）**，它将聊天应用与 AI 编程代理连接起来。

**四个核心特点**:
1. **自托管** - 运行在你的硬件上，数据完全私有
2. **多通道** - 单个 Gateway 服务多个平台（WhatsApp、Telegram、Discord、iMessage 等）
3. **代理原生** - 为 AI 代理构建，支持多代理路由
4. **开源** - MIT 许可证，完全可控

### 1.2 系统架构

```
聊天应用 → Gateway → AI Agent
              ↓
         CLI / Web UI / 节点
```

**三个核心组件**:

1. **Gateway** (唯一事实来源)
   - 唯一事实来源（single source of truth）
   - 维护通道连接
   - 通过 WebSocket 暴露 API
   - 处理认证、路由、会话管理
   - 内置 cron 调度器

2. **客户端**
   - macOS app (官方应用)
   - CLI 工具 (命令行)
   - Web UI (控制面板 http://127.0.0.1:18789)
   - OpenClaw.app (macOS 应用)

3. **节点** (Nodes)
   - macOS、iOS、Android、headless 支持
   - Canvas 可视化界面
   - Camera 拍照/录像
   - Screen Record 屏幕录制
   - Location 位置获取
   - Voice Wake 语音唤醒

## 🧠 三、会话管理

### 3.1 会话 Key 结构

```
直接聊天：agent:<agentId>:<mainKey>
群聊：agent:<agentId>:<channel>:group:<id>
线程：agent:<agentId>:<channel>:group:<id>:thread:<threadId>
```

### 3.2 DM 隔离模式 (dmScope)

**四种模式**:

1. **`main`** - 所有 DM 共享主会话（单用户场景）
2. **`per-peer`** - 按发送者隔离（多用户）
3. **`per-channel-peer`** - 按通道 + 发送者隔离（**推荐**）
4. **`per-account-channel-peer`** - 多账户场景

**⚠️ 安全警告**: 如果代理接收多人的 DM，必须启用安全 DM 模式（`dmScope: "per-channel-peer"`），否则隐私信息会泄露！

### 3.3 会话绑定

- **线程绑定 (Thread Bindings)**: Discord 支持 `/focus`, `/unfocus` 命令
- **会话范围 (dmScope)**: `main` | `per-peer` | `per-channel-peer` | `per-account-channel-peer`
- **重置策略**: `daily` (每日重置) 或 `idleMinutes` (空闲重置)

## 🔀 四、多代理路由

### 4.1 多代理系统

每个代理拥有：
- **独立工作空间** - 不同项目/场景
- **独立状态目录** - 各自的状态和记忆
- **独立会话存储** - 互不干扰

### 4.2 路由规则（最具体优先）

1. **peer 匹配** - 精确 DM/群组 id
2. **parentPeer 匹配** - 线程继承
3. **guildId + roles** - Discord 角色匹配
4. **guildId** - Discord 群组匹配
5. **teamId** - Slack 团队匹配
6. **accountId 匹配** - 账户级匹配
7. **通道级匹配** - 通道级匹配
8. **回退到默认代理**

### 4.3 配置示例

```json5
{
  agents: {
    list: [
      { id: "home", default: true, workspace: "~/.openclaw/workspace-home" },
      { id: "work", workspace: "~/.openclaw/workspace-work" },
    ],
  },
  bindings: [
    { agentId: "home", match: { channel: "whatsapp", accountId: "personal" } },
    { agentId: "work", match: { channel: "whatsapp", accountId: "biz" } },
  ],
}
```

## ⏰ 五、自动化系统

### 5.1 Cron Jobs (定时任务)

OpenClaw 内置**Gateway 调度器**，支持：

**三种调度模式**:
1. **`at`** - 单次定时任务（ISO 8601）
2. **`every`** - 固定间隔（毫秒）
3. **`cron`** - Cron 表达式（5 字段或 6 字段）

**两种执行方式**:
1. **Main session** - 在心跳时运行主会话
2. **Isolated session** - 专用会话，不干扰主聊天

**交付模式**:
1. **`announce`** - 向聊天发送摘要
2. **`webhook`** - HTTP POST 到指定 URL
3. **`none`** - 仅内部运行

### 5.2 Cron vs Heartbeat

**使用心跳 (Heartbeat)**:
- 多个检查可以批处理
- 需要对话上下文
- 时间可以漂移
- 减少 API 调用

**使用 Cron**:
- 精确时间要求
- 任务需要隔离
- 不同模型/思考级别
- 一次性提醒

### 5.3 配置示例

```json5
{
  cron: {
    enabled: true,
    maxConcurrentRuns: 2,
    sessionRetention: "24h",
    runLog: {
      maxBytes: "2mb",
      keepLines: 2000,
    },
  },
}
```

**CLI 使用**:
```bash
# 添加任务
openclaw cron add --name "Morning brief" --cron "0 7 * * *" --session isolated --message "Summarize overnight updates." --announce

# 查看任务
openclaw cron list

# 运行任务
openclaw cron run <jobId>
```

## 🔐 六、安全性与认证

### 6.1 关键安全要点

1. **DM 安全模式** - 多用户场景必须启用 `dmScope: "per-channel-peer"`
2. **认证令牌** - 所有连接必须签署 nonce
3. **工具策略** - 通过 allow/deny 控制权限
4. **沙箱隔离** - 每代理独立沙箱

### 6.2 认证流程

**所有连接必须通过签名认证**:
- Gateway 生成 nonce
- 客户端签名返回
- Gateway 验证签名

**多租户隔离**:
- 每个用户独立会话
- 工作空间隔离
- 工具权限控制

### 6.3 沙箱支持

**Docker 沙箱模式**:
- `off` - 无沙箱（默认）
- `non-main` - 非主会话沙箱
- `all` - 所有会话沙箱

**构建镜像**: `scripts/sandbox-setup.sh`

## 📋 七、配置文件

### 7.1 配置文件位置

- **路径**: `~/.openclaw/openclaw.json`
- **格式**: JSON5（支持注释和尾随逗号）
- **热重载**: 自动检测变化并应用

### 7.2 最小配置

```json5
// ~/.openclaw/openclaw.json
{
  agents: { defaults: { workspace: "~/.openclaw/workspace" } },
  channels: { whatsapp: { allowFrom: ["+15555550123"] } },
}
```

### 7.3 配置管理

**三种编辑方式**:

1. **交互向导**: `openclaw onboard` / `openclaw configure`
2. **CLI 命令**: `openclaw config get/set/unset`
3. **控制面板**: http://127.0.0.1:18789 的 Config 标签

**热重载模式**:
- `hybrid` (默认) - 安全变化立即应用，关键变化自动重启
- `hot` - 仅安全变化
- `restart` - 任何变化都重启
- `off` - 禁用自动重载

### 7.4 环境变量

**加载顺序**:
1. 父进程环境变量
2. 当前目录 `.env`
3. `~/.openclaw/.env`
4. 配置中的 inline env

**支持**:
- 环境变量替换：`${VAR_NAME}`
- SecretRefs：`{ source: "env", provider: "default", id: "VAR_NAME" }`
