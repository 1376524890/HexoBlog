## 📋 一、OpenClaw 是什么？

### 1.1 一句话定义

**OpenClaw 是 AI Agent 运行时平台（AI Agent Runtime Platform）**，核心是一个**智能网关（Runtime Gateway）**。

它不是聊天机器人，而是**把 AI 连接到真实世界的桥梁**——将前沿 AI 模型连接到实际的消息通道（WhatsApp、Discord、Telegram、飞书等）和真实工具（文件、执行、浏览器、摄像头、设备等）。

### 1.2 核心定位（四个必背要点）⭐⭐⭐⭐⭐

1. **Access control before intelligence**（访问控制先于智能）- 安全优先
2. **隐私优先**：私有数据保持私有，数据在自己硬件上
3. **记忆即文件**：所有记忆写入磁盘 Markdown 文件，不丢失
4. **工具优先**：第一类工具而非 skill 包裹

### 1.3 与 ChatGPT 的本质区别

| 对比项 | ChatGPT | OpenClaw |
|--------|---------|----------|
| **定位** | 聊天机器人 | Agent 运行时平台 |
| **能力** | 生成文本 | 真正执行任务 |
| **记忆** | 会话内临时存储 | 持久化到磁盘文件 |
| **工具** | API 调用有限 | 文件系统、执行命令、浏览器控制、设备能力 |
| **部署** | 云端 SaaS | 本地部署，数据私有 |
| **安全性** | 受限于平台 | 多层次安全控制，审计完善 |
| **扩展性** | 插件系统有限 | Skills 模块化扩展 |
| **多租户** | 支持 | 不支持敌对多租户（单一用户信任边界） |

## 🏗️ 二、核心架构

### 2.1 三层架构

```
┌─────────────────────────────────────────────────────────┐
│              Agent Layer（智能层）                        │
│  - Main Agent（主会话，直接聊天）                        │
│  - Subagents（子代理，后台运行，深度可嵌套）              │
│  - ACP Agents（编码代理，支持 Codex/Claude Code 等）      │
└─────────────────────────┬───────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│           Gateway Layer（网关层）← 大脑！                  │
│  - 控制平面、策略层、路由                                │
│  - 身份认证、工具策略、会话管理                          │
│  - 频道适配器（Discord/WhatsApp/飞书/Telegram 等）        │
│  - ⚠️ Gateway 本身不运行 AI 模型，只是调度员                 │
└─────────────────────────┬───────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│              Node Layer（节点层）← 手脚                   │
│  - 远程执行表面                                          │
│  - 设备能力（摄像头、屏幕录制、位置、通知、声音）        │
│  - macOS companion app、iOS/Android 节点                 │
└─────────────────────────────────────────────────────────┘
```

### 2.2 四大核心组件

| 组件 | 作用 | 关键点 |
|------|------|--------|
| **Gateway** | 大脑、路由器 | 不运行 AI 模型，只是调度员，负责生命周期管理 |
| **Agent** | 执行 AI 任务 | 每个 Agent 包含：身份 (Identity)+ 配置 (Config)+ 状态 (State)+ 运行时 (Runtime) |
| **Session** | 有状态容器 | 包含消息历史、上下文窗口（经过压缩）、工具状态、元数据 |
| **Channel** | 协议适配器 | Telegram、Discord、Slack、WhatsApp、飞书、微信、IRC 等，插件化设计 |

### 2.3 Agent Loop（核心循环）⭐

理解 OpenClaw 的关键是理解 **Agent Loop**——AI Agent 持续运行的核心循环。

```
┌──────────────────────────────────────────────────────────┐
│                      Agent Loop                          │
│                                                          │
│   1. 接收输入 → 用户通过 Channel 发送消息                  │
│   2. 构建上下文 → 组装 Session 历史、系统提示词、工具列表 │
│   3. LLM 推理 → 模型决定是"直接回复"还是"调用工具"        │
│   4. 工具执行 → 如需多步骤，通过 Gateway 调用外部工具      │
│   5. 循环或结束 → 多步推理继续，否则返回最终结果           │
│   6. 发送响应 → Gateway 通过原 Channel 发送给用户          │
└──────────────────────────────────────────────────────────┘
```

**关键点**：
- 模型拥有决策权，主动决定需要什么信息、调用什么工具、如何组织答案
- Gateway 负责路由和工具协调，不直接处理 AI 推理

### 2.4 System Prompt（系统提示词）

**动态生成机制**（每个 session 启动时构建）：

1. **基础身份**：Agent 的名称、描述、emoji
2. **工具描述**：当前可用的所有工具（JSON Schema 格式）
3. **运行时信息**：当前时间、日期、环境变量
4. **安全提示**：沙箱边界、禁止行为
5. **格式说明**：如何输出工具调用

**Bootstrap 文件注入**（加载到 Session）：
- `AGENTS.md` - 工作空间规则
- `SOUL.md` - 身份认知
- `USER.md` - 用户信息
- `TOOLS.md` - 工具配置
- `IDENTITY.md` - 用户身份
- `MEMORY.md` - 精选记忆（仅主会话）

### 3.2 工具安全策略

**工具 Profile**（四种配置）：

| Profile | 可用工具 | 适用场景 |
|---------|----------|----------|
| `minimal` | 只有 `session_status` | 基础对话 |

| `messaging` | 消息相关工具 | 消息处理 |
| `full` | 无限制 | 全功能 |

**安全控制机制**：
- `tools.allow` / `tools.deny` - 允许/拒绝工具
- `tools.byProvider` - 按模型 provider 细化
- `sandbox` - 沙箱隔离
- `elevated` - 提权执行（需显式启用）

**工具组 (shorthands)**：
- `group:runtime` - exec/bash/process
- `group:fs` - read/write/edit
- `group:sessions` - 会话管理
- `group:memory` - 记忆工具
- `group:web` - 网络搜索
- `group:ui` - 浏览器/canvas
- `group:messaging` - 消息工具
- `group:nodes` - 节点控制

## 👥 四、子代理系统 (Subagents)

### 4.1 什么是子代理

子代理是从主会话启动的**后台代理运行**，用于：
- 并行化耗时任务（研究、长任务、慢工具调用）
- 隔离敏感/复杂操作
- 支持嵌套编排模式（main → orchestrator → worker）

### 4.2 深度层级架构

```
Depth 0: Main Agent (agent:main:discord:123456)
         ↓ 可派生
Depth 1: Sub-agent (可进一步派生，当 maxSpawnDepth≥2)
         ↓ 可派生
Depth 2: Leaf worker (不可再派生)
```

**最大嵌套深度**: 1-5（推荐 2，用于编排模式）

### 4.3 启动方式

**工具方式**（推荐）：
```python
sessions_spawn({
  task: "研究 XX 主题",
  agentId: "code-executor",  # 可选，指定 agent
  model: "模型 ID",           # 可选，覆盖默认模型
  thinking: "level",         # 可选，覆盖思考级别
  label: "任务标签",         # 可选，清晰标识
  thread: true,              # 线程绑定（Discord 支持）
  mode: "session",           # "run"=单次，"session"=持久
  sandbox: "inherit",        # "inherit"| "require"
  cleanup: "delete",         # "delete"| "keep"
  runTimeoutSeconds: 900     # 可选，超时时间
})
```

**Slash 命令**：
```bash
/subagents spawn <agentId> <task> [--model <model>] [--thinking <level>]
/subagents list
/subagents kill <id|#|all>
/subagents log <id|#> [limit] [tools]
/subagents info <id|#>
/subagents steer <id|#> <message>
```

### 4.4 通知机制（Announce）

子代理完成时会**announce**结果回主会话：

**包含内容**：
- **Result**（回复文本）
- **Status**（success/error/timeout/unknown）
- **Runtime**（运行时长）
- **Token usage**（输入/输出/总计）
- **Estimated cost**（如果配置了模型价格）
- **Session 信息**（key、id、transcript 路径）

**重要原则**：结果需重写为正常助理语气，**不**直接转发 raw 内部元数据

### 4.5 并发控制

- **全局并发**: `maxConcurrent`（默认 8）
- **单代理子代理数**: `maxChildrenPerAgent`（默认 5）
- **专用队列**: `subagent` 队列（独立于主会话）
- **级联停止**: 停止父代理会停止所有子代理

### 4.6 线程绑定 (Thread Binding)

当线程绑定启用时，子代理可保持在同一个线程中：

**支持通道**：
- **Discord**（目前唯一支持）：支持 `/focus`、`/unfocus`、`/agents`、`/session idle`、`/session max-age`

**工作流程**：
1. 使用 `sessions_spawn` 启动，`thread: true`
2. 创建或绑定线程到该会话
3. 后续消息路由到绑定的子代理
4. 使用 `/unfocus` 手动解绑

## 🧠 五、记忆系统

### 5.1 三层记忆架构

```
┌─────────────────────────────────────────────────────────┐
│ Layer 1: 会话记忆（Session Memory）                      │
│ - 当前会话上下文                                        │
│ - 临时决策和中间结果                                    │
│ - 生命周期：单次会话                                     │
└─────────────────────────────────────────────────────────┘
                    ↓ 同步关键信息
┌─────────────────────────────────────────────────────────┐
│ Layer 2: 任务记忆（Task Memory）                         │
│ - 任务计划文件                                          │
│ - 子代理执行结果                                        │
│ - 路径：memory/tasks/ACTIVE-<task-id>.md                │
└─────────────────────────────────────────────────────────┘
                    ↓ 同步重要发现
┌─────────────────────────────────────────────────────────┐
│ Layer 3: 长期记忆（Long-term Memory）                    │
│ - MEMORY.md（精选记忆）                                 │
│ - memory/YYYY-MM-DD.md（每日日志）                      │
│ - 持久化存储，跨会话                                     │
└─────────────────────────────────────────────────────────┘
```

### 5.2 Memory 工具

| 工具 | 功能 |
|------|------|
| `memory_search` | 语义检索 MEMORY.md + memory/*.md |
| `memory_get` | 读取特定文件的片段（from/lines 参数） |

### 5.3 最佳实践

1. **Write Immediately**：及时写入，上下文最清晰时
2. **WAL Before Responding**：回复前先写入关键信息
3. **Buffer in Danger Zone**：60% 上下文时记录所有交互
4. **Recover from Buffer**：从缓冲区恢复，不询问"我们之前在做什么"
5. **Search Before Giving Up**：尝试所有来源再放弃

## 🔐 六、安全机制

### 6.1 安全原则

1. **Private things stay private**：私密信息不泄露
2. **Ask before acting externally**：外部行动前确认
3. **Never send half-baked replies**：不要发送半成品回复
4. **Be careful in group chats**：在群组中不要代表用户说话

### 6.2 信任边界

**OpenClaw 假设**：
- 单一用户信任边界 per Gateway
- 不支持敌对多租户
- 如需隔离需分 Gateway/OS User/Host

**核心安全规则**：
1. Gateway 和 Node 属于同一信任域
2. `sessionKey` 是路由键，不是认证 token
3. Prompt guardrails 不是安全边界
4. 本地 TUI `!` shell 是明确的操作者触发

### 6.3 安全审计

**审计命令**：
```bash
openclaw security audit           # 基本检查
openclaw security audit --deep    # 深度检查
openclaw security audit --fix     # 自动修复
openclaw security audit --json    # JSON 格式
```

**审计重点**：
- Gateway 绑定/认证暴露
- 浏览器控制暴露
- 提权工具允许
- 文件系统权限
- Node 配对和执行命令

### 6.4 凭证存储

```
~/.openclaw/
├── credentials/
│   ├── whatsapp/<accountId>/creds.json
│   └── <channel>-allowFrom.json
├── agents/
│   └── <agentId>/
│       └── auth-profiles.json
└── secrets.json  # 可选
```

### 6.5 沙箱隔离

**模式**：
- `off` - 无沙箱
- `non-main` - 仅沙箱非主会话
- `all` - 所有会话沙箱

**配置**：
```json
{
  "agents": {
    "defaults": {
      "sandbox": {
        "mode": "non-main",
        "scope": "agent"
      }
    }
  }
}
```

## 📊 九、配置系统

### 9.1 配置文件

位置：`~/.openclaw/openclaw.json`（JSON5 格式，支持注释）

**最小配置示例**：
```json5
{
  agents: { defaults: { workspace: "~/.openclaw/workspace" } },
  channels: { whatsapp: { allowFrom: ["+15555550123"] } },
}
```

### 9.2 核心配置项

#### 会话管理
```json
{
  "session": {
    "dmScope": "per-channel-peer",  // main | per-peer | per-channel-peer
    "threadBindings": {
      "enabled": true,
      "idleHours": 24,
      "maxAgeHours": 0
    },
    "reset": {
      "mode": "daily",
      "atHour": 4,
      "idleMinutes": 120
    }
  }
}
```

#### 子代理配置
```json
{
  "agents": {
    "defaults": {
      "subagents": {
        "maxSpawnDepth": 2,      // 最大嵌套深度
        "maxChildrenPerAgent": 5, // 单代理子代理数
        "maxConcurrent": 8,       // 全局并发
        "runTimeoutSeconds": 900  // 默认超时
      }
    }
  }
}
```

#### 工具策略
```json
{
  "tools": {
    "profile": "coding",  // minimal | coding | messaging | full
    "deny": ["group:automation"],
    "fs": { "workspaceOnly": true },
    "exec": { "security": "deny", "ask": "always" }
  }
}
```

#### 安全加固
```json
{
  "gateway": {
    "mode": "local",
    "bind": "loopback",
    "auth": { "mode": "token", "token": "长随机 token" }
  },
  "session": { "dmScope": "per-channel-peer" },
  "tools": {
    "profile": "messaging",
    "deny": ["group:automation", "group:runtime", "group:fs"],
    "sandbox": { "mode": "non-main" }
  }
}
```

### 9.3 配置热重载

**模式**：
- `hybrid`（默认）：安全变更热应用，关键变更自动重启
- `hot`：仅热应用安全变更
- `restart`：任何变更都重启
- `off`：禁用文件监控
