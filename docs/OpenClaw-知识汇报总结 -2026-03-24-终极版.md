## 🔧 四大核心组件（必懂！⭐⭐⭐⭐⭐）

### 1. Gateway（网关）⭐⭐⭐⭐⭐
- **作用**: 大脑、路由器、控制平面
- **关键点**: 不运行 AI 模型，只是调度员
- **默认端口**: 18789
- **协议**: WebSocket, JSON 文本帧
- **事件类型**: `agent`、`chat`、`presence`、`health`、`heartbeat`、`cron`
- **连接生命周期**:
  ```
  Client -> Gateway: req:connect
  Gateway -> Client: res (ok) + presence + health snapshot
  Client -> Gateway: req:agent
  Gateway -> Client: event:agent (streaming) + res:agent (final)
  ```

### 2. Agent（智能体）
- **Agent Loop**: 接收输入 → 构建上下文 → LLM 推理 → 工具执行 → 发送响应
- **关键点**: 模型拥有决策权，主动决定需要什么信息、调用什么工具
- **组成**: Workspace + State directory + Session store

### 3. Session（会话）⭐⭐⭐⭐⭐
- **作用**: 有状态容器，存储消息历史、上下文信息
- **维护**:
  - `session.maintenance.mode`: `enforce`（推荐）/ `warn`
  - `session.maintenance.pruneAfter`: `45d`（30 天默认）
  - `session.maintenance.maxEntries`: `800`（500 默认）
  - `session.maintenance.rotateBytes`: `20mb`
  - `session.maintenance.maxDiskBytes`: `1gb`
  - `session.maintenance.highWaterBytes`: `800mb`
- **Key 格式**:
  - 直接聊天：`agent:<agentId>:main` 或 `agent:<agentId>:direct:<peerId>`
  - 群组聊天：`agent:<agentId>:<channel>:group:<id>`
  - 频道聊天：`agent:<agentId>:<channel>:channel:<id>`
  - Cron 任务：`cron:<jobId>`
  - Webhook: `hook:<uuid>`
  - Node 运行：`node-<nodeId>`
- **dmScope（安全 DM 模式）**:
  - `main`: 所有 DM 共享主会话（单用户场景）
  - `per-peer`: 按发送者 ID 隔离
  - `per-channel-peer`: 按渠道 + 发送者隔离（**多用户推荐**）
  - `per-account-channel-peer`: 按账户 + 渠道 + 发送者隔离（**多账户推荐**）
- **Reset 策略**:
  - 每日重置：默认凌晨 4 点
  - 空闲重置：`idleMinutes: 120`
  - 手动重置：`/new` 或 `/reset`

### 4. Channel（频道）
- **支持平台**: 15+ 个（WhatsApp、Telegram、Discord、飞书、Signal、Slack、iMessage、LINE、Matrix、Mattermost、Microsoft Teams、Google Chat、Nextcloud Talk、Nostr、Twitch、Zalo 等）
- **协议**: 每个频道有自己的实现（Baileys for WhatsApp、grammY for Telegram 等）

## 🔄 五、Agent Loop（核心循环）⭐⭐⭐⭐⭐

```
1. 接收输入 → 用户通过 Channel 发送消息
2. 构建上下文 → 组装 Session 历史、系统提示词、工具列表
3. LLM 推理 → 模型决定是"直接回复"还是"调用工具"
4. 工具执行 → 如需多步骤，通过 Gateway 调用外部工具
5. 循环或结束 → 多步推理继续，否则返回最终结果
6. 发送响应 → Gateway 通过原 Channel 发送给用户
```

**关键点**: 模型拥有决策权，主动决定需要什么信息、调用什么工具。

## 🧠 七、三层记忆架构（必讲！⭐⭐⭐⭐⭐）

```
Layer 1: 会话记忆（Session Memory）
- 当前会话上下文
- 临时决策和中间结果
    ↓ 同步关键信息
Layer 2: 任务记忆（Task Memory）
- 任务计划文件
- 子代理执行结果
    ↓ 同步重要发现
Layer 3: 长期记忆（Long-term Memory）
- MEMORY.md：精选记忆
- memory/YYYY-MM-DD.md：每日日志
```

**WAL Protocol（写后读协议）**:
- **STOP** — 不要立即回复
- **WRITE** — 更新记忆文件
- **THEN** — 回复用户

**触发条件**: 修正、专有名词、偏好、决策、编辑、数值

## ⏰ 十、Cron vs Heartbeat 对比（新增！⭐⭐⭐⭐⭐）

### 使用 Heartbeat 当：
- 多个检查可以批量处理（邮箱 + 日历 + 通知）
- 需要对话上下文
- 时间可以稍有漂移（每~30 分钟）
- 想通过组合周期性检查减少 API 调用

### 使用 Cron 当：
- 精确时间要求（"每周一上午 9:00 整"）
- 任务需要与主会话历史隔离
- 想要不同的模型或思考级别
- 一次性提醒（"20 分钟后提醒我"）
- 输出应直接传递到 channel 而不涉及主会话
