## 🔧 Gateway 核心架构（深度解析）

### 连接生命周期
```
Client -> Gateway: req:connect
Gateway -> Client: res (ok) + presence + health snapshot
Client -> Gateway: req:agent
Gateway -> Client: event:agent (streaming) + res:agent (final)
```

### 安全机制
- 设备身份：所有连接包含 device identity
- 配对流程：新设备需要批准，颁发 device token
- 本地连接：loopback 可自动批准
- 签名验证：所有连接必须签名 connect.challenge

## 📝 Session 管理（深度解析）

### Key 格式
- 直接聊天：`agent:<agentId>:main` 或 `agent:<agentId>:direct:<peerId>`
- 群组聊天：`agent:<agentId>:<channel>:group:<id>`
- 频道聊天：`agent:<agentId>:<channel>:channel:<id>`
- Cron 任务：`cron:<jobId>`
- Webhook: `hook:<uuid>`
- Node 运行：`node-<nodeId>`

### 安全 DM 模式（多用户场景必配！⭐⭐⭐⭐⭐）
```json
{
  "session": {
    "dmScope": "per-channel-peer"  // 或 "per-account-channel-peer"
  }
}
```

### Session 维护
- `mode: "enforce"` - 强制执行清理策略
- `pruneAfter: "45d"` - 清理 45 天前的会话
- `maxEntries: 800` - 最多 800 个会话
- `maxDiskBytes: "1gb"` - 磁盘上限

## 🧠 三层记忆架构（WAL Protocol）

```
Layer 1: 会话记忆（Session Memory）
    ↓
Layer 2: 任务记忆（Task Memory）
    ↓
Layer 3: 长期记忆（Long-term Memory）
- MEMORY.md：精选记忆
- memory/YYYY-MM-DD.md：每日日志
```

**WAL Protocol**:
- **STOP** — 不要立即回复
- **WRITE** — 更新记忆文件
- **THEN** — 回复用户

## ⏰ Cron vs Heartbeat（对比）

| | Cron | Heartbeat |
|---|---|---|
| 精确时间 | ✅ "每周一 9:00 整" | ❌ 可稍有漂移 |
| 隔离运行 | ✅ 独立于主会话 | ❌ 使用主会话上下文 |
| 批量处理 | ❌ 单任务执行 | ✅ 可批量处理多个检查 |
| 场景 | 定时任务 | 日常检查（邮箱 + 日历 + 通知） |
