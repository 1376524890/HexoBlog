## 📌 一句话介绍

> **OpenClaw 是一个自托管的 AI Agent 运行时平台**，不是聊天机器人，而是把 AI 模型连接到真实世界的桥梁。通过单一 Gateway 控制平面，可以统一管理 20+ 消息渠道（WhatsApp、Telegram、Discord、飞书等），实现多智能体协作。

---

## 🎯 四大核心理念（必背）

1. ✅ **Access control before intelligence**（访问控制先于智能）⭐⭐⭐⭐⭐
2. ✅ **隐私优先**: 私有数据保持私有
3. ✅ **记忆即文件**: 所有记忆写入 Markdown 文件
4. ✅ **工具优先**: 第一类工具而非 skill 包裹

**不是什么**:
- ❌ 不是聊天机器人（如 ChatGPT）
- ❌ 不是多租户共享 AI 服务
- ❌ 不是云端托管的 SaaS 产品
- ❌ 不是单一 AI 模型的替代品

**是什么**:
- ✅ **自托管平台**: 运行在自己的设备上，数据本地化
- ✅ **单一控制平面**: Gateway 统一管理多个渠道
- ✅ **Agent 原生**: 内置工具调用、会话管理、多智能体路由
- ✅ **开源项目**: MIT 许可，社区驱动

### 三层记忆架构

```
┌─────────────────────────────────────────────────────────────┐
│ Layer 1: 会话记忆（Session Memory）                          │
│ - 当前会话上下文                                             │
│ - 临时决策和中间结果                                         │
└─────────────────────────────────────────────────────────────┘
         │ 同步关键信息
         ▼
┌─────────────────────────────────────────────────────────────┐
│ Layer 2: 任务记忆（Task Memory）                             │
│ - 任务计划文件                                               │
│ - 子代理执行结果                                             │
└─────────────────────────────────────────────────────────────┘
         │ 同步重要发现
         ▼
┌─────────────────────────────────────────────────────────────┐
│ Layer 3: 长期记忆（Long-term Memory）                        │
│ - MEMORY.md: 精选记忆                                       │
│ - memory/YYYY-MM-DD.md: 每日日志                            │
└─────────────────────────────────────────────────────────────┘
```

**WAL Protocol（写入前置日志协议）**:
- **规则**: 在回复之前，先把关键信息写入 `SESSION-STATE.md`
- **触发条件**: 修正、专有名词、偏好、决策、数值

## 🔐 安全模型（重点）

### 权限层级

| 级别 | 类型 | 权限范围 |
|------|------|----------|
| Level 5 | 主 Agent | 完全权限 |
| Level 4 | 可信子 Agent | 受限系统权限（需批准） |
| Level 3 | 标准子 Agent | 标准开发权限 |
| Level 2 | 受限子 Agent | 严格受限权限 |
| Level 1 | 只读子 Agent | 只读访问 |

### 安全审计命令

```bash
openclaw security audit       # 基本检查
openclaw security audit --deep # 深度检查
openclaw security audit --fix  # 自动修复
openclaw security audit --json # JSON 格式输出
```

### 关键安全检查

#### 基础安全配置（60 秒加固）
1. **文件权限**: `chmod 700 ~/.openclaw`、`chmod 600 ~/.openclaw/openclaw.json`
2. **网关绑定**: 默认 `loopback`，非 loopback 需要防火墙
3. **mDNS 发现**: 推荐 `minimal` 或 `off` 模式
4. **DM 访问控制**: 使用 `pairing` 或 `allowlist`
5. **组聊 gating**: 启用 `requireMention`
