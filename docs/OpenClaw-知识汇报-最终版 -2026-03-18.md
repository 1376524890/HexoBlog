## 🏗️ 核心架构

### 三层架构模型

```
┌─────────────────────────────────────────────────────────┐
│              Agent Layer（智能层）                        │
│  - Main Agent（主会话）                                  │
│  - Subagents（子代理）                                   │
│  - ACP Agents（编码代理）                                │
└─────────────────────────┬───────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│          Gateway Layer（网关层）← 大脑！                   │
│  - 控制平面、策略层、路由                                │
│  - 身份认证、工具策略、会话管理                          │
│  - 频道适配器（20+ 种：WhatsApp、Telegram、Discord、飞书等）│
│  ⚠️ Gateway 本身**不运行 AI 模型**，只是调度员                │
└─────────────────────────┬───────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│              Node Layer（节点层）← 手脚                   │
│  - 远程执行表面                                          │
│  - 设备能力（摄像头、屏幕、通知、位置）                  │
│  - macOS/Windows/Linux companion app                     │
│  - iOS/Android mobile app                                │
└─────────────────────────────────────────────────────────┘
```

### 四大核心组件

| 组件 | 作用 | 关键点 |
|------|------|--------|
| **Gateway** | 大脑、路由器 | 不运行 AI 模型，只是调度员 |
| **Agent** | 执行 AI 任务 | 身份 + 配置 + 状态 + 运行时 |
| **Session** | 有状态容器 | 消息历史、上下文、工具状态 |
| **Channel** | 协议适配器 | Telegram、Discord、飞书等 |

## 🔄 Agent Loop（核心循环）

```
1. 接收输入 → 用户通过 Channel 发送消息
2. 构建上下文 → 组装 Session 历史、系统提示词、工具列表
3. LLM 推理 → 模型决定是"直接回复"还是"调用工具"
4. 工具执行 → 如需多步骤，通过 Gateway 调用外部工具
5. 循环或结束 → 多步推理继续，否则返回最终结果
6. 发送响应 → Gateway 通过原 Channel 发送给用户
```

**System Prompt 动态生成**:
- 基础身份（AGENTS.md、SOUL.md、TOOLS.md、USER.md、IDENTITY.md）
- 工具描述（JSON Schema 格式）
- 运行时信息（时间、日期、环境变量）
- 安全提示（沙箱边界、禁止行为）
- **MEMORY.md**（仅主会话加载）

## 🧠 三层记忆架构

```
┌─────────────────────────────────────────────────────────┐
│ Layer 1: 会话记忆（Session Memory）                       │
│ - 当前会话上下文                                         │
│ - 临时决策和中间结果                                     │
└─────────────────────────┬───────────────────────────────┘
                          ↓ 同步关键信息
┌─────────────────────────────────────────────────────────┐
│ Layer 2: 任务记忆（Task Memory）                          │
│ - 任务计划文件                                           │
│ - 子代理执行结果                                         │
└─────────────────────────┬───────────────────────────────┘
                          ↓ 同步重要发现
┌─────────────────────────────────────────────────────────┐
│ Layer 3: 长期记忆（Long-term Memory）                     │
│ - MEMORY.md：精选记忆（精华提取）                        │
│ - memory/YYYY-MM-DD.md：每日日志                         │
└─────────────────────────────────────────────────────────┘
```

**记忆管理原则**:
- ✅ **DECIDE to write**: 决定、偏好、持久事实 → MEMORY.md
- ✅ **Daily notes**: 日常记录 → memory/YYYY-MM-DD.md
- ✅ **定期 review**: 清理 MEMORY.md，移除过时信息
- ✅ **Ask to remember**: 重要事项明确让 Agent 写入记忆

**记忆工具**:
- `memory_search` - 语义检索
- `memory_get` - 读取特定文件

## 🔐 安全模型（核心！）

### 四大核心理念

1. ✅ **Access control before intelligence**（访问控制先于智能）⭐⭐⭐⭐⭐
2. ✅ **隐私优先**：私有数据保持私有
3. ✅ **记忆即文件**：所有记忆写入 Markdown 文件
4. ✅ **工具优先**：第一类工具而非 skill 包裹

### 安全配置示例

```json
{
  "gateway": {
    "mode": "local",
    "bind": "loopback",
    "auth": { "mode": "token", "token": "长随机 token" }
  },
  "session": { "dmScope": "per-channel-peer" },
  "tools": {
    "profile": "messaging",
    "deny": ["group:automation", "group:runtime", "group:fs"],
    "fs": { "workspaceOnly": true },
    "exec": { "security": "deny", "ask": "always" },
    "elevated": { "enabled": false }
  }
}
```

### MCP 协议

OpenClaw 的工具系统基于 **MCP（Model Context Protocol）**，Anthropic 提出的开放标准。

**核心思想**: 标准化 AI 与外部世界的交互接口。

**Skills 就是 MCP 的实现**:
- 每个 Skill 是一个独立的包
- 通过 JSON Schema 描述工具
- Gateway 负责 Skill 的注册、发现和调用
- Agent 通过标准接口与 Skill 交互

## 🎯 核心总结（6 大观点）

### 1. 不是聊天机器人，是做事的 Agent
OpenClaw 与 ChatGPT 的最大区别在于**真正的任务执行能力**。它不仅生成文本，还能：
- 读取/修改文件
- 执行系统命令
- 控制浏览器
- 操作设备（相机、屏幕）
- 发送消息到 20+ 平台

### 2. 记忆即文件，永不丢失
所有记忆持久化到 Markdown 文件：
- **MEMORY.md**: 精选记忆
- **memory/YYYY-MM-DD.md**: 每日日志
- 三层架构确保记忆不丢失

### 3. 安全第一，访问控制先于智能
- 三层权限模型（Level 1-5）
- 安全审计工具完善
- 沙箱隔离保护
- 最小权限原则

### 4. 模块化设计，高度可扩展
- Skills 独立可替换
- Channels 支持 20+ 平台
- Agents 支持自定义
- MCP 协议标准化

### 5. 多智能体协作，专业分工
- 御坂网络第一代系统
- 9 个子代理各司其职
- 主代理负责任务调度
- 支持嵌套和并发

### 6. 本地化部署，数据私有
- Gateway 本地运行
- 所有数据本地存储
- 支持 Docker 部署
- 支持远程节点
