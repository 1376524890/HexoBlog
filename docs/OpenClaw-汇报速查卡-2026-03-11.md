## 🎯 一句话说清楚

**OpenClaw 是 AI Agent 运行时平台**，核心是一个智能网关，把 AI 连接到真实世界——支持 WhatsApp、Discord、飞书等消息通道，能真正执行任务（读写文件、执行命令、浏览器控制、设备能力等）。

**不是聊天机器人，是 Agent 运行时！**

---

## 💎 四个必背要点（⭐⭐⭐⭐⭐）

1. **Access control before intelligence**（访问控制先于智能）
2. **隐私优先**：数据在自己硬件上
3. **记忆即文件**：Markdown 文件持久化
4. **工具优先**：第一类工具而非 skill 包裹

---

## 🏗️ 架构三要素

```
┌─────────────────┐
│ Agent Layer     │ ← 主 Agent + 子代理（后台运行）
│ (智能层)         │
└────────┬────────┘
         ↓
┌─────────────────┐
│ Gateway Layer   │ ← 大脑！路由、策略、会话管理
│ (网关层)         │    ⚠️不运行 AI 模型，只是调度员
└────────┬────────┘
         ↓
┌─────────────────┐
│ Node Layer      │ ← 手脚！摄像头、屏幕、位置等
│ (节点层)         │
└─────────────────┘
```

**四大组件**：
- **Gateway**：大脑，不运行 AI，只调度
- **Agent**：执行 AI 任务（身份 + 配置 + 状态 + 运行时）
- **Session**：有状态容器（消息历史 + 上下文 + 工具状态）
- **Channel**：协议适配器（飞书/Discord/Telegram 等）

---

## 🔄 Agent Loop（核心循环）

```
1. 接收输入 → 用户发消息
2. 构建上下文 → 组装历史、系统提示、工具列表
3. LLM 推理 → 决定"直接回复"还是"调用工具"
4. 工具执行 → 如需多步骤，调用外部工具
5. 循环或结束 → 多步推理继续
6. 发送响应 → Gateway 发回原通道
```

**关键点**：模型有决策权！

## 👥 子代理系统

**为什么需要**：并行化耗时任务，隔离复杂操作

**深度层级**：
- **Depth 0**: Main Agent
- **Depth 1**: Sub-agent（可进一步派生）
- **Depth 2**: Leaf worker（不可再派生）

**启动方式**：
```python
sessions_spawn(task: "研究主题", mode: "session", thread: true)
```

**Announce 机制**：完成时通知主会话，包含状态、运行时、Token 统计

**最佳实践**：
- 默认深度 1（leaf worker），需要编排时设为 2
- 每个子代理有自己的 Token 消耗
- 设置超时避免无限运行
- 结果重写为正常助理语气，**不转发 raw 内部元数据**

## 🧠 记忆三层架构

```
Layer 1: 会话记忆（临时）
         ↓ 同步关键信息
Layer 2: 任务记忆（memory/tasks/）
         ↓ 同步重要发现
Layer 3: 长期记忆（MEMORY.md + 每日日志）
```

**最佳实践**：
1. Write Immediately（及时写入）
2. WAL Before Responding（回复前先写入）
3. Buffer in Danger Zone（60% 上下文时记录）
4. Recover from Buffer（从缓冲区恢复）
5. Search Before Giving Up（尝试所有来源）

## 🔐 安全原则

**四大原则**：
1. Private things stay private
2. Ask before acting externally
3. Never send half-baked replies
4. Be careful in group chats

**安全审计**：
```bash
openclaw security audit
openclaw doctor --fix
```

**工具策略**：
- Profile: `minimal` | `coding` | `messaging` | `full`
- Deny: `group:automation`, `group:runtime`, `group:fs`
- Sandbox: `non-main` | `all`

**Q&A 准备**：

**Q**: OpenClaw 和 ChatGPT 有什么区别？
**A**: OpenClaw 是运行时平台，能真正执行任务；ChatGPT 是聊天机器人。OpenClaw 本地部署、数据私有、支持持久化记忆、工具能力强。

**Q**: Gateway 运行 AI 模型吗？
**A**: 不运行！Gateway 只是调度员，AI 模型由外部 provider 提供（OpenAI、Anthropic、本地模型等）。

**Q**: 如何保证安全？
**A**: 多层安全：沙箱隔离、工具策略、权限控制、审计日志、凭证存储。

**Q**: 子代理和主代理有什么区别？
**A**: 子代理是后台运行，完成时 announce 结果。可以深度嵌套（默认 1 层，可设 2 层用于编排）。
