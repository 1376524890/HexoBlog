## 🎯 核心知识点

### 1. OpenClaw 是什么？

**核心定位**:
- ✅ **不是聊天机器人**，是 **AI Agent 运行时系统**
- ✅ **不是纯框架**，是 **完整可运行的系统**
- ✅ **不是纯自动化工具**，是 **多智能体协作平台**

**一句话总结**:
> OpenClaw = 智能体网关 + 多通道支持 + 技能系统 + 记忆系统 + 自动化能力

**核心优势**:
1. **完整系统** - 开箱即用，不只是框架
2. **多通道支持** - 统一 API，多平台集成 (WhatsApp/Telegram/Discord/Slack 等)
3. **记忆持久化** - 纯 Markdown 文件，易于维护
4. **技能可插拔** - 模块化设计，易于扩展
5. **多智能体架构** - 灵活隔离，安全可控
6. **节点远程执行** - 跨设备协作能力

### 2. 核心架构

#### 2.1 三层架构

```
┌─────────────────────────────────────┐
│         Agent Layer (智能层)         │
│  - Main Agent (主会话)                │
│  - Subagents (子代理)                 │
│  - ACP Agents (编码代理)              │
└─────────────────────────────────────┘
              ↓
┌─────────────────────────────────────┐
│       Gateway Layer (网关层)         │
│  - 控制平面、策略层、路由              │
│  - 身份认证、工具策略、会话管理        │
│  - 频道适配器 (Discord/WhatsApp 等)    │
└─────────────────────────────────────┘
              ↓
┌─────────────────────────────────────┐
│      Node Layer (节点层)             │
│  - 远程执行表面                       │
│  - 设备能力 (摄像头、屏幕、通知)       │
│  - macOS  companion app               │
└─────────────────────────────────────┘
```

#### 2.2 四核心组件

| 组件 | 作用 | 关键点 |
|------|------|--------|
| **Gateway** | 大脑、路由器 | 不运行 AI 模型，只是调度员 |
| **Agent** | 执行 AI 任务 | 身份 + 配置 + 状态 + 运行时 |
| **Session** | 有状态容器 | 消息历史、上下文、工具状态 |
| **Channel** | 协议适配器 | Telegram、Discord、飞书等 |

**Agent Loop**: 接收输入 → 思考决策 → 执行动作 → 循环或发送响应

### 3. 四大核心理念

#### 3.1 Access control before intelligence ⭐⭐⭐⭐⭐

**访问控制先于智能** - 这是 OpenClaw 最核心的设计原则

**为什么重要？**
- AI 模型可以很聪明，但如果没有权限控制，会非常危险
- 必须先定义"谁能做什么"，再考虑"能做多聪明"
- 权限控制是基础，智能是建立在安全之上的

#### 3.2 隐私优先：私有数据保持私有

**核心原则**:
- 所有数据本地存储
- 支持自托管部署
- 不上传用户数据到云端
- 数据控制权完全在用户手中

**实现方式**:
- 本地运行 Gateway
- 记忆文件存储在工作区
- 凭证文件加密存储
- 支持完全离线使用

#### 3.3 记忆即文件

**记忆系统架构**:
```
~/workspace/
├── MEMORY.md              # 长期记忆（精选）
└── memory/
    ├── 2026-03-12.md      # 今日日志
    ├── 2026-03-11.md      # 昨日日志
    └── ...
```

**特点**:
- ✅ **纯 Markdown 文件** - 易于编辑和版本控制
- ✅ **向量检索** - 支持语义搜索 (`memory_search`)
- ✅ **混合搜索** - BM25 + 向量相似度
- ✅ **自动压缩** - 临近 token 上限时自动整理
- ✅ **时间衰减** - 新内容优先，旧内容降权

**工具**:
- `memory_search` - 语义搜索
- `memory_get` - 精准读取文件

#### 3.4 工具优先

**工具分类**:

| 类别 | 说明 | 示例 |
|------|------|------|
| **本地工具** | 文件系统、进程执行 | `read`, `write`, `exec`, `process` |
| **网络工具** | 网页操作 | `web_search`, `web_fetch`, `browser` |
| **平台工具** | 第三方集成 | `feishu_*`, `message` |
| **系统工具** | 系统管理 | `nodes`, `canvas`, `session_*` |

**工具安全**:
- 使用沙箱隔离敏感操作
- 工具调用需要明确授权
- 敏感工具可配置白名单
- 支持工具 Profile (minimal/coding/messaging/full)

### 5. 安全模型

#### 5.1 权限层级

| 级别 | 说明 | 适用对象 |
|------|------|----------|
| Level 5 | 主 Agent - 完全权限 | 主会话 |
| Level 4 | 可信子 Agent - 受限系统权限 | 需批准 |
| Level 3 | 标准子 Agent - 标准开发权限 | 开发代理 |
| Level 2 | 受限子 Agent - 严格受限权限 | 通用代理 |
| Level 1 | 只读子 Agent - 只读访问 | 读取代理 |

#### 5.2 安全命令

```bash
# 基本审计
openclaw security audit

# 深度审计
openclaw security audit --deep

# 自动修复
openclaw security audit --fix

# JSON 格式输出
openclaw security audit --json
```

#### 5.3 加固配置示例

```json
{
  "gateway": {
    "mode": "local",
    "bind": "loopback",
    "auth": { "mode": "token", "token": "长随机 token" }
  },
  "session": { "dmScope": "per-channel-peer" },
  "tools": {
    "profile": "messaging",
    "deny": ["group:automation", "group:runtime", "group:fs"],
    "fs": { "workspaceOnly": true },
    "exec": { "security": "deny", "ask": "always" },
    "elevated": { "enabled": false }
  }
```

### 6. 技能系统 (Skills)

#### 6.1 什么是 Skill？

Skill 是**AgentSkills 兼容的目录**，包含 `SKILL.md` 和工具使用说明，教 AI 如何使用工具。

#### 6.2 技能来源

1. **Bundled Skills** - 内置技能 (npm 包)
2. **Managed Skills** - `~/.openclaw/skills/` (全局共享)
3. **Workspace Skills** - `<workspace>/skills/` (每个 agent 独立)
4. **ClawHub Skills** - [https://clawhub.com](https://clawhub.com) (在线仓库)

#### 6.3 技能优先级

```
<workspace>/skills > ~/.openclaw/skills > bundled skills
```

#### 6.4 常用技能

- `feishu-doc` - 飞书文档读写
- `feishu-drive` - 飞书云盘管理
- `feishu-perm` - 飞书权限管理
- `feishu-wiki` - 飞书知识库
- `hexo-blog` - Hexo 博客管理
- `weather` - 天气查询
- `healthcheck` - 安全加固
- `coding-agent` - 代码代理
- `xiaohongshu-ops` - 小红书运营
- `proactive-agent` - 主动代理
- `task-tracker` - 任务追踪
- `subagent-network-call` - 御坂网络调用

### 7. 配置管理

#### 7.1 配置文件

**位置**: `~/.openclaw/openclaw.json` (JSON5 格式)

**最小配置**:
```json
{
  "agents": { "defaults": { "workspace": "~/.openclaw/workspace" } },
  "channels": { "whatsapp": { "allowFrom": ["+15555550123"] } }
}
```

#### 7.2 热重载模式

| 模式 | 说明 |
|------|------|
| `hybrid` (默认) | 安全变更热重载，关键变更自动重启 |
| `hot` | 仅热重载安全变更，日志提示需重启 |
| `restart` | 所有变更触发重启 |
| `off` | 禁用文件监听，需手动重启 |

#### 7.3 模型管理

**命令**:
```bash
openclaw models list
openclaw models set <model-id>
openclaw models status
```

**配置示例**:
```json
{
  "agents": {
    "defaults": {
      "model": {
        "primary": "anthropic/claude-sonnet-4-5",
        "fallbacks": ["openai/gpt-5.2"]
      }
    }
  }
}
```

### 9. 节点系统

#### 9.1 节点是什么？

节点是**配对的远程设备**，可以:
- 执行命令
- 访问摄像头
- 屏幕录制
- 展示 Canvas
- 发送通知

#### 9.2 节点命令

```bash
# 列出节点
nodes status

# 执行命令
nodes run --node <node-id> --command "ls -la"

# 拍照
nodes camera snap --node <node-id>

# 屏幕录制
nodes screen record --node <node-id>

# 展示 Canvas
nodes canvas present --node <node-id> --target "https://example.com"
```

### 10. 自动化能力

#### 10.1 定时任务 (Cron)

**命令**:
```bash
openclaw cron list
openclaw cron add --name "daily-check" --every "1d"
openclaw cron run daily-check
openclaw cron enable/disable daily-check
```

#### 10.2 心跳机制 (Heartbeat)

**配置示例**:
```json
{
  "agents": {
    "defaults": {
      "heartbeat": {
        "every": "30m",
        "target": "last",
        "directPolicy": "allow"
      }
    }
  }
}
```

#### 10.3 Webhooks

**配置示例**:
```json
{
  "hooks": {
    "enabled": true,
    "token": "shared-secret",
    "path": "/hooks",
    "mappings": [...]
  }
}
```
