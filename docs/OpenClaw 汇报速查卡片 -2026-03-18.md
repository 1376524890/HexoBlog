## 🎯 核心定义

**OpenClaw** = AI Agent 运行时平台（不是聊天机器人）  
**核心** = 智能网关（Runtime Gateway）  
**使命** = 把 AI 连接到真实世界

---

## 📐 三层架构

```
Agent Layer（智能层）
    ↓
Gateway Layer（网关层）← 大脑！
    ↓
Node Layer（节点层）← 手脚
```

---

## 🔧 四大组件

| 组件 | 作用 | 关键点 |
|------|------|--------|
| **Gateway** | 大脑、路由器 | 不运行 AI，只是调度员 |
| **Agent** | 执行任务 | 身份 + 配置 + 状态 + 运行时 |
| **Session** | 有状态容器 | 消息历史、上下文、工具状态 |
| **Channel** | 协议适配器 | Telegram、Discord、飞书等 |

---

## ⚙️ Agent Loop 六步

1. **接收输入** → 用户消息
2. **构建上下文** → 组装 Session+Prompt+Tools
3. **LLM 推理** → 模型决定"回复"还是"调用工具"
4. **工具执行** → 如需要
5. **循环/结束** → 多步推理继续
6. **发送响应** → Gateway 发送回用户
