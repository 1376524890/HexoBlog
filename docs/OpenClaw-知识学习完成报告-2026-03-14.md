## 🎯 核心知识点（必背）

### 1. OpenClaw 是什么？

**一句话**: OpenClaw 是 **AI Agent 运行时平台**，核心是**智能网关（Runtime Gateway）**。

不是聊天机器人，而是把 AI 连接到真实世界的桥梁。

**四大核心理念**（⭐⭐⭐⭐⭐ 必背）：
1. **Access control before intelligence**（访问控制先于智能）
2. **隐私优先**：私有数据保持私有
3. **记忆即文件**：所有记忆写入 Markdown 文件
4. **工具优先**：第一类工具而非 skill 包裹

### 2. 三层架构

```
Agent Layer（智能层）
  ↓
Gateway Layer（网关层）← 大脑！不运行 AI，只是调度员
  ↓
Node Layer（节点层）← 手脚
```

### 3. 四大核心组件

- **Gateway**: 大脑、路由器（控制平面、策略层、路由）
- **Agent**: 执行 AI 任务（身份 + 配置 + 状态 + 运行时）
- **Session**: 有状态容器（消息历史、上下文、工具状态）
- **Channel**: 协议适配器（Telegram、Discord、飞书等）

### 4. 工具系统（8 大分类）

1. **Runtime**: `exec`, `process`, `gateway`
2. **Filesystem**: `read`, `write`, `edit`, `apply_patch`
3. **Session**: `sessions_list`, `sessions_spawn` 等
4. **Memory**: `memory_search`, `memory_get`
5. **Web**: `web_search`, `web_fetch`, `tavily`
6. **UI**: `browser`, `canvas`
7. **Node**: `nodes`
8. **Messaging**: `message`

### 5. 技能系统（16 个已安装）

### 7. 三层记忆架构

```
Layer 1: 会话记忆（Session Memory）
    ↓ 同步关键信息
Layer 2: 任务记忆（Task Memory）
    ↓ 同步重要发现
Layer 3: 长期记忆（Long-term Memory）
- MEMORY.md：精选记忆
- memory/YYYY-MM-DD.md：每日日志
```

### 8. 安全模型

**核心原则**:
1. Private things stay private
2. Ask before acting externally
3. Never send half-baked replies
4. Be careful in group chats

**权限层级**: Level 1-5（只读 → 完全权限）

**安全命令**:
```bash
openclaw security audit           # 基本检查
openclaw security audit --deep    # 深度检查
openclaw security audit --fix     # 自动修复
```

### 9. Session 管理

**Context 管理** - Compaction 机制:
- 当历史消息超过阈值时智能总结和裁剪
- 自动触发：默认开启
- 手动触发：`/compact "专注于决策和待办事项"`

### 10. Cron 定时任务

**两种执行模式**:
- **Main session jobs** (系统事件): 下次心跳运行
- **Isolated jobs** (隔离会话): 专用 agent 会话运行
