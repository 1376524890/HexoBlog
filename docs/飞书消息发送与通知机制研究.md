## 📡 消息发送机制

### 1. OpenClaw `message` 工具

**基本用法**:
```javascript
message({
  action: "send",
  channel: "feishu",
  target: "user:ou_c0ea02caca01fe1b21994f95366d8c4a",
  message: "这是测试消息"
})
```

**参数说明**:
- `channel`: 渠道类型 (`feishu`)
- `target`: 
  - 私聊：`user:ou_xxx`
  - 群组：`chat:oc_xxx`
- `message`: 消息内容
- `mimeType`: 可选 (如 `application/json`)
- `media`: 可选 (媒体文件 URL 或本地路径)

### 2. 飞书 SDK 原生调用

**通过 `feishu_chat` 工具**:
```javascript
feishu_chat({
  action: "info",
  chat_id: "oc_xxx"  // 群组 ID
})
```

**通过 `feishu_doc` 工具**:
```javascript
feishu_doc({
  action: "create",
  title: "标题",
  content: "内容"
})
```

### 3. 会话发送 `sessions_send`

**跨会话发送**:
```javascript
sessions_send({
  sessionKey: "agent:main:feish...d8c4a",
  message: "这是发送给特定会话的消息"
})
```

**参数说明**:
- `sessionKey`: 会话标识
- `message`: 消息内容
- `timeoutSeconds`: 超时时间

## 🔄 Session 流转机制

### 1. 会话路由

**私聊路由**:
```
飞书私聊消息 → Feishu Channel
  ↓
自动路由到主 Agent (main)
  ↓
创建会话：agent:main:feish...d8c4a
  ↓
会话内对话
```

**群组路由**:
```
飞书群聊消息 → Feishu Channel
  ↓
根据 peer 匹配规则路由
  ↓
创建独立会话
  ↓
会话内对话
```

### 2. 会话 ID 格式

**标准格式**: `agent:main:feishu:ou_xxx`
- `agent`: Agent ID (main/clawd-fan 等)
- `channel`: 渠道类型 (feishu/telegram 等)
- `peer_id`: 用户 Open ID 或群组 ID

### 3. 会话状态管理

**会话存储位置**: `/home/claw/.openclaw/agents/main/sessions/sessions.json`

**会话类型**:
- `direct`: 直接对话
- `system`: 系统会话
- `cron`: 定时任务会话

## 📮 通知机制实现

### 方案一：使用 sessions_send 回发消息

**场景**: Agent 处理完成后，通过 session 发送结果

**实现方式**:
```javascript
// 在 Agent 内部处理完成后
sessions_send({
  sessionKey: "<当前会话 Key>",
  message: "✅ 任务完成！结果如下：\n\n" +
           "1. 文件已创建\n" +
           "2. 测试通过\n" +
           "3. 文档已更新\n\n" +
           "详细结果请查看：`docs/任务报告.md`"
})
```

**优点**:
- ✅ 自动路由到原始对话
- ✅ 保持会话上下文
- ✅ 用户直接在飞书收到回复

**缺点**:
- ❌ 需要获取当前会话 Key
- ❌ 异步任务可能超时

---

### 方案二：使用 message 工具直接发送

**场景**: 独立通知，不依赖会话

**实现方式**:
```javascript
// 在 Agent 内部
message({
  action: "send",
  channel: "feishu",
  target: "user:ou_c0ea02caca01fe1b21994f95366d8c4a",
  message: "🔔 任务提醒\n\n御坂网络 V2 实现完成！\n\n" +
           "详细文档：`docs/御坂网络-V2-迭代完成总结.md`"
})
```

**优点**:
- ✅ 无需获取会话 Key
- ✅ 可以异步发送
- ✅ 可以批量发送

**缺点**:
- ❌ 没有会话上下文
- ❌ 用户可能不知道来源

---

### 方案三：使用心跳检查 + 推送

**场景**: 定期检查任务状态，必要时推送通知

**实现方式**:
```javascript
// 在心跳检测中
// 检查任务状态
const taskStatus = checkTaskStatus(taskId)

if (taskStatus.status === "completed") {
  // 发送通知
  message({
    action: "send",
    channel: "feishu",
    target: "user:ou_c0ea02caca01fe1b21994f95366d8c4a",
    message: "✅ 任务已完成！\n" +
             `任务 ID: ${taskId}\n` +
             `完成时间：${new Date().toISOString()}\n\n` +
             `详细结果：${taskStatus.resultUrl}`
  })
}
```

**优点**:
- ✅ 实时状态监控
- ✅ 主动推送
- ✅ 支持多种触发条件

**缺点**:
- ❌ 需要定期轮询
- ❌ 增加 API 调用

## 🛠️ 实现建议

### 1. 消息发送最佳实践

**使用场景**:
| 场景 | 推荐方式 | 理由 |
|------|----------|------|
| 任务完成通知 | `sessions_send` | 保持会话上下文 |
| 系统状态推送 | `message` | 独立于会话 |
| 错误报警 | `message` + `cron` | 主动推送 |
| 定期报告 | `cron` + `message` | 定时触发 |

**代码示例**:
```javascript
// 在 Agent 任务完成后
async function onTaskComplete(taskId) {
  const result = await getTaskResult(taskId);
  
  // 使用 sessions_send 保持上下文
  try {
    await sessions_send({
      sessionKey: getCurrentSessionKey(),
      message: formatTaskCompleteMessage(result)
    });
  } catch (error) {
    // 如果 session 发送失败，降级使用 message
    await message({
      action: "send",
      channel: "feishu",
      target: getUserOpenId(),
      message: "⚠️ 会话发送失败，请查看独立消息通知"
    });
  }
}
```

### 2. 会话管理最佳实践

**会话 Key 获取**:
```javascript
// 在 Agent 内部获取当前会话
const currentSession = getSessionContext();
const sessionKey = currentSession?.sessionKey;

if (!sessionKey) {
  // 如果没有会话，使用 message 直接发送
  await message({
    action: "send",
    channel: "feishu",
    target: "user:ou_xxx",
    message: "任务完成"
  });
} else {
  // 使用会话发送
  await sessions_send({
    sessionKey: sessionKey,
    message: "任务完成"
  });
}
```

**会话超时处理**:
```javascript
// 检查会话是否过期
const isSessionActive = isSessionValid(sessionKey);

if (!isSessionActive) {
  // 创建新会话或直接发送
  const newSession = createNewSession();
  await sessions_send({
    sessionKey: newSession.key,
    message: "任务完成"
  });
}
```

### 3. 通知频率控制

**避免 spam 的策略**:
```javascript
// 通知频率控制
const NOTIFICATION_THRESHOLD = {
  taskComplete: 60,  // 任务完成：1 分钟
  taskError: 300,    // 任务错误：5 分钟
  heartbeat: 3600,   // 心跳：1 小时
};

async function sendNotification(type, data) {
  const lastNotification = getLastNotificationTime(type);
  const now = Date.now();
  
  if (now - lastNotification < NOTIFICATION_THRESHOLD[type]) {
    return; // 频率过高，不发送
  }
  
  // 发送通知
  await message({
    action: "send",
    channel: "feishu",
    target: "user:ou_xxx",
    message: formatNotification(type, data)
  });
  
  // 记录发送时间
  setLastNotificationTime(type, now);
}
```
