## 🎯 核心定义与定位

### 官方定义

**OpenClaw 是一个自托管网关**，将你最喜欢的聊天应用——WhatsApp、Telegram、Discord、iMessage 等——与 AI 编码代理（如 Pi）连接起来。你在自己的机器（或服务器）上运行单个 Gateway 进程，它成为你的消息应用和始终可用的 AI 助手之间的桥梁。

### 目标用户

开发者和高阶用户，他们想要一个可以从任何地方发送消息的个人 AI 助手，同时不放弃对数据的控制或依赖托管服务。

### 核心特点

| 特点 | 说明 |
|------|------|
| **自托管** | 运行在你的硬件上，遵循你的规则 |
| **多通道** | 单个 Gateway 同时服务 WhatsApp、Telegram、Discord 等 |
| **代理原生** | 为编码代理构建，支持工具使用、会话、记忆和多代理路由 |
| **开源** | MIT 许可，社区驱动 |

## 🏗️ 核心架构（三层 + 四组件）

### 三层架构

```
┌─────────────────────────────────────────────────────────┐
│           Gateway Layer（网关层）← 核心！                  │
│  - 控制平面、策略层、路由                               │
│  - 身份认证、工具策略、会话管理                         │
│  - 频道适配器（Discord/WhatsApp/飞书等）                │
│  ⚠️ Gateway 本身不运行 AI 模型，只是调度员                  │
└─────────────────────────────────────────────────────────┘
          ↑         ↑         ↑
     Chat Apps  Plugins  CLI/UI
```

**关键点**：
- Gateway 是会话、路由和频道连接的单一真实来源
- 通过 WebSocket 协议统一所有通信
- 支持 15+ 个聊天平台
- 单 Gateway 服务多个聊天应用

### 四大核心组件

#### 1. Gateway（网关）
- **作用**: 大脑、路由器
- **核心功能**:
  - WebSocket 连接管理
  - 消息路由
  - 工具调用代理
  - 会话持久化

#### 2. Agent（智能体）
- **作用**: 执行 AI 任务
- **Agent Loop（核心循环）**:
  ```
  1. 接收输入 → 用户通过 Channel 发送消息
  2. 构建上下文 → 组装 Session 历史、系统提示词、工具列表
  3. LLM 推理 → 模型决定是"直接回复"还是"调用工具"
  4. 工具执行 → 如需多步骤，通过 Gateway 调用外部工具
  5. 循环或结束 → 多步推理继续，否则返回最终结果
  6. 发送响应 → Gateway 通过原 Channel 发送给用户
  ```

#### 3. Session（会话）
- **作用**: 有状态容器
- **内容**: 消息历史、上下文信息、工具状态、临时决策

#### 4. Channel（频道）
- **作用**: 协议适配器
- **支持平台**: Telegram、Discord、Slack、WhatsApp、iMessage、飞书、WhatsApp Business、Mattermost 等 15+ 个

## 🧠 三层记忆架构

### 记忆系统设计

```
┌─────────────────────────────────────────────────────────┐
│ Layer 1: 会话记忆（Session Memory）                       │
│ - 当前会话上下文                                        │
│ - 临时决策和中间结果                                    │
└─────────────────────────────────────────────────────────┘
              ↓ 同步关键信息
┌─────────────────────────────────────────────────────────┐
│ Layer 2: 任务记忆（Task Memory）                          │
│ - 任务计划文件                                          │
│ - 子代理执行结果                                        │
└─────────────────────────────────────────────────────────┘
              ↓ 同步重要发现
┌─────────────────────────────────────────────────────────┐
│ Layer 3: 长期记忆（Long-term Memory）                     │
│ - MEMORY.md：精选记忆                                   │
│ - memory/YYYY-MM-DD.md：每日日志                        │
└─────────────────────────────────────────────────────────┘
```

### 记忆管理最佳实践

1. **DECIDE to write**: 决定、偏好、持久事实 → MEMORY.md
2. **Daily notes**: 日常记录 → memory/YYYY-MM-DD.md
3. **定期 review**: 定期清理 MEMORY.md，移除过时信息
4. **Ask to remember**: 重要事项明确让 Agent 写入记忆
5. **立即提交**: 操作后 `git add` 和 `git commit`
