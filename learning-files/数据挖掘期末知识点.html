数据挖掘

填空题知识点随记

第二章

  • 数据集是由数据对象组成的，一个数据对象代表一个实体
  • 数据对象也被称为样本、示例、实例、数据点、对象、元组
  • 数据库行对应实体，列对应属性
  • 属性是一个数据字段，代表对象的一个特征
  • 分类属性，二元属性，序数属性（顺序有意义，大小无意义），区间尺度（无绝对零点，差值有意义比值无意义），比率尺度（有绝对零点，比值有意义）
    数据分析：
  • 度量集中趋势——中位数、最大值、最小值、分位数、异常值、方差等
  • 卡方自由度：（r-1）*（c-1）
  • 度量离散程度-卡方：（实际-期望）^2 / 期望
  • 协方差与相关性分析——乘积的期望-期望的乘积
  • 皮尔逊相关性系数——总体：协方差/总体标准差乘积，样本相关系数
  • 可视化展示
  • KL散度计算中log的底不影响相对大小，只影响数据绝对值，数值本身无意义

第三章：数据仓库

  • 与运营数据库分开的决策支持数据库，用于给数据分析提供支持
  • 数据仓库是一个面向主题的、集成的、随时间变化的且非易失的数据集合，用于支持管理层的决策过程。
  • 数据仓库建模：星型模型，雪花模型，星系模型
  • OLAP操作：上卷，下钻，切块，旋转

第四章：模式挖掘

  • 模式：数据集中频繁出现的项集，子序列或子结构

  • 模式发现：从大量数据中挖掘模式

  • 支持度：绝对支持度——项集出现次数；相对支持度——包含项集的占总体的比例

  • 如果支持度超过设定的阈值则为频繁项集
    关联规则：支持度（S）和置信度(C)

  • 支持度：“同时包含 X 和 Y 的事务占总事务的比例”，本质是项集 X∪Y（X 和 Y 的并集）的相对支持度。大于阈值说明是“流行的”，有统计意义

  • 置信度： “包含 X 的事务中，同时包含 Y 的比例”，本质是 “条件概率 P (Y|X)”。大于阈值说明是可信的
    频繁模式爆炸（TBD数量少但包含的项数多）

  • 如两个TBD分别T1有50个项，T2有100个项，最小支持度为1时，任意子集都可以成为频繁项集，如k=3的三项集，频繁项集有C(50,3)+C(100,3)-C(50,3)=970,200（T1的子集已经在T2的子集中）
    模式压缩——闭合模式与最大模式

  • 闭合模式（无损压缩）：1. X 是频繁的 2. 不存在任何超模式 Y 使得支持度与 X 相同

  • 最大模式（有损压缩）：1. X 是频繁的 2. 不存在任何频繁的超模式 Y
    频繁模式的向下封闭性

  • **正向：**如果一个项集是频繁的（支持度≥minsup），那么它的所有子集必为频繁；

  • **反向：**如果一个项集的任意一个子集是不频繁的，那么该项集本身必为不频繁

  • Apriory算法

  • 原理：向下封闭性（Apriori 性质）
    算法的所有逻辑都基于 “频繁项集的向下封闭性”（之前复习的核心性质）：
    正向：如果一个项集是频繁的（支持度≥minsup），那么它的所有子集必为频繁；
    反向（剪枝关键）：如果一个项集的任意一个子集是不频繁的，那么该项集本身必为不频繁 —— 无需生成或测试这个项集，直接剪枝。
    简单说：“先找所有频繁的小项集，再用小项集生成大项集候选，最后筛选出频繁的大项集”，全程用 “子集是否频繁” 过滤无效候选，避免冗余计算。
    模式挖掘（兴趣度）

  • 置信度和支持度不能完全反应正负相关性等信息

  • 引入兴趣度度量：Lift和 χ²
    Lift :

  • 分子：B 和 C 同时出现的相对支持度（共现概率）；

  • 分母：B 的支持度 × C 的支持度（假设 B 和 C 独立时的理论共现概率）
