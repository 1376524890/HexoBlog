## 📋 完整研究流程

### 阶段 1: 准备（5 分钟）
1. ✅ 理解任务范围
2. ✅ 识别所需技能/工具
3. ✅ 制定研究计划
4. ✅ 设置迭代次数（默认 20 次）

### 阶段 2: 数据收集（10-30 分钟）
1. ✅ 御坂妹妹 16 号搜索 arXiv/论文/代码
2. ✅ 御坂妹妹 16 号抓取网页内容
3. ✅ 御坂妹妹 16 号读取本地文件
4. ✅ 整理研究素材

### 阶段 3: 深度分析（20-40 分钟）
1. ✅ 御坂妹妹 13 号分析现有架构
2. ✅ 御坂妹妹 13 号对比研究结果
3. ✅ 御坂妹妹 13 号找出差异点
4. ✅ 御坂妹妹 11 号评估实现复杂度

### 阶段 4: 迭代优化（核心！2-3 小时）
**执行 20 次迭代循环**:

```
for iteration in 1..20:
    1. 深入研究（御坂妹妹 13 号）
    2. 代码评估（御坂妹妹 11 号）
    3. 整合当前方案
    4. 向 Claude 求建议（sessions_send）
    5. 整合反馈到下一轮
    6. 记录到 memory/archives/iteration-XX.md
```

### 阶段 5: 最终汇报（10 分钟）
1. ✅ 输出完整优化方案
2. ✅ 改进前后对比
3. ✅ 实施路线图
4. ✅ 风险评估
5. ✅ 更新记忆文档

---

## 🎯 关键要点

### ⚠️ 必须做到的 3 件事
1. **每次迭代都要记录** → `memory/archives/iteration-XX.md`
2. **每次迭代都要向 Claude 求建议** → `sessions_send`
3. **20 次迭代必须完整执行** → 不可跳过

### ❌ 不要犯的错误
1. ❌ 使用 `runtime="acp"`（会报错）
2. ❌ Feishu 使用 `thread=True`（不支持）
3. ❌ 跳过 Claude 咨询（质量会下降）
