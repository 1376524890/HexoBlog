# SmartSearch 导入问题修复记录

## 问题描述
`smart_search.py` 使用了相对导入 `from .config import get_config`，但作为命令行脚本运行时无法解析。

## 修复方案

### 1. 条件导入模式
在文件末尾使用条件导入，只在 `if __name__ == "__main__":` 块中执行导入：

```python
# 条件导入：在 __main__ 块中执行，避免类定义时导入失败
if __name__ == "__main__":
    # 导入需要在运行时执行
    from config import get_config
    from utils import ContentProcessor, FallbackChain, FetchResult
    from engines import MultiSearch, SearchResult, SearchResponse
    
    asyncio.run(main())
```

### 2. 延迟初始化全局实例
避免在模块加载时初始化依赖项：

```python
# 全局实例（延迟初始化）
_smart_search_instance = None

def get_smart_search_instance():
    """获取全局智能搜索实例"""
    global _smart_search_instance
    if _smart_search_instance is None:
        _smart_search_instance = SmartSearch()
    return _smart_search_instance
```

### 3. 移除类型注解中的具体类名
为了避免类定义时类型未导入的问题，将类型注解改为通用类型：

```python
async def _layer1_search(self, query: str):  # 移除了 -> SearchResponse
    async with MultiSearch() as searcher:
        return await searcher.search(query)

def _layer2_filter(self, search_response, depth: int) -> List[str]:  # 移除了 SearchResponse 类型
    ...

async def _layer3_fetch(self, urls: List[str]) -> List:  # 移除了具体的返回类型
    ...
```

## 注意事项

1. **网络超时**: 在测试环境中，部分搜索引擎可能超时（17 个引擎中的 7 个），这是预期的行为，降级链会自动处理

2. **依赖安装**: 需要安装以下依赖：
   - aiohttp
   - httpx
   - requests
   - beautifulsoup4
   - lxml

3. **模块导入**: 作为模块导入时需要确保包结构正确：
   ```python
   from smart_search import SmartSearch, get_smart_search_instance
   searcher = get_smart_search_instance()
   ```

4. **性能**: 17 个搜索引擎并行搜索可能需要较长时间（1-2 分钟），可以根据需要调整 `--depth` 和 `--max-results` 参数
