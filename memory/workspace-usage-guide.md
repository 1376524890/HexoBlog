## 🔴 重要系统文件 (禁止操作)

### 绝对禁止移动/删除/修改：

| 文件/目录 | 作用 | 后果 |
|---------|------|------|
| `AGENTS.md` | Agent 身份定义 | 身份丢失 |
| `USER.md` | 用户信息 | 无法识别用户 |
| `SOUL.md` | Agent 人设 | 行为异常 |
| `IDENTITY.md` | 本尊身份 | 身份混乱 |
| `TOOLS.md` | 工具配置 | 功能失效 |
| `MEMORY.md` | 长期记忆 | 记忆丢失 |
| `HEARTBEAT.md` | 心跳配置 | 定时任务失效 |
| `memory/` | 每日日志 | 连续性问题 |
| `skills/` | 技能系统 | 功能缺失 |
| `source/` | 博客源文件 | 博客丢失 |
| `.git/` | Git 仓库 | 版本控制失效 |

**处理原则**：
- ❌ **不要移动** - 这些文件在固定位置
- ❌ **不要删除** - 删除后无法恢复
- ❌ **不要重命名** - 会导致系统错误
- ✅ **可以读取** - 查看内容和更新
- ✅ **可以编辑** - 在允许的情况下更新

## 🟡 可归档文件 (建议操作)

### 定期归档 (按时间/类型)：

#### 1️⃣ 记忆文件归档
- **来源**: `memory/YYYY-MM-DD.md` (超过 30 天的)
- **目标**: `memory/archives/`
- **时机**: 每月底或定期
- **注意**: 只归档，不删除，保留原始文件

#### 2️⃣ OpenClaw 学习文档
- **来源**: `memory/openclaw-*.md` (超过 7 天的)
- **目标**: `memory/archives/`
- **注意**: 如果仍有参考价值，保留在原位

#### 3️⃣ 项目临时文件
- **来源**: `scripts/*.log` (超过 7 天的)
- **目标**: `memory/archives/` 或 `trash`
- **注意**: 先备份再删除

#### 4️⃣ 编译输出
- **来源**: `output/`, `node_modules/`
- **操作**: 可以删除，不影响源码
- **注意**: 重新构建时会生成

## 📅 归档操作指南

### 🔄 月度归档流程

1. **检查记忆文件** (`memory/`)
   ```bash
   # 查看超过 30 天的文件
   find memory -name "*.md" -mtime +30
   ```

2. **移动到归档目录**
   ```bash
   # 创建归档目录 (如果不存在)
   mkdir -p memory/archives/2026-03

   # 移动文件
   mv memory/2026-03-01.md memory/archives/2026-03/
   ```

3. **Git 提交**
   ```bash
   git add memory/archives/
   git commit -m "chore: 归档 2026-03-01 的记忆文件"
   git push
   ```

### 📦 大型文件处理

1. **小说下载文件**
   - 位置：`~/downloads/`
   - 操作：可以移动到 `~/backup/` 或删除
   - 注意：保留 TXT 文件用于阅读

2. **缓存文件**
   - 位置：`venv/`, `__pycache__/`
   - 操作：可以删除
   - 注意：重新运行时会生成

3. **备份文件**
   - 位置：`.openclaw/backup/`
   - 操作：定期清理 (超过 7 天的)
   - 注意：保留最近的备份

## ⚠️ 安全规则

### ✅ 安全操作：
- ✅ 读取文件
- ✅ 查看文件内容
- ✅ 复制文件 (创建副本)
- ✅ 移动文件到 `trash`
- ✅ 压缩文件 (创建 .tar.gz)
- ✅ Git 提交更改
- ✅ 更新文档

### ❌ 危险操作：
- ❌ 直接删除系统文件
- ❌ 移动核心文件到其他地方
- ❌ 重命名重要文件
- ❌ 修改配置文件 (未经确认)
- ❌ 删除 `.git/` 目录
- ❌ 清理 `memory/` 目录

### 🔒 删除前确认：
1. **确认不是系统文件**
2. **确认不是重要数据**
3. **使用 `trash` 而不是 `rm`**
4. **记录到 memory 文件**
5. **Git 提交删除操作**

## 📊 文件分类标准

### 🏷️ 按类型分类：

| 类型 | 特征 | 处理方式 |
|------|------|----------|
| **系统文件** | 在禁止列表中 | 不动 |
| **记忆文件** | `memory/*.md` | 定期归档 |
| **技能文件** | `skills/` 下 | 本地开发 |
| **博客文件** | `source/_posts/` | 审核发布 |
| **临时文件** | `*.log`, `tmp/` | 定期清理 |
| **归档文件** | `archives/` | 只读 |

### 🏷️ 按重要性分类：

| 重要性 | 特征 | 操作建议 |
|--------|------|----------|
| **极高** | 核心系统文件 | 禁止操作 |
| **高** | 记忆/技能/博客 | 谨慎修改 |
| **中** | 项目文件 | 可更新 |
| **低** | 临时/缓存 | 定期清理 |

## 🛠️ 常用命令

### 📂 文件查找
```bash
# 查找超过 30 天的 md 文件
find memory -name "*.md" -mtime +30

# 查找大于 10MB 的文件
find . -type f -size +10M

# 查找所有 log 文件
find . -name "*.log"
```

### 📦 文件归档
```bash
# 归档记忆文件
mv memory/2026-03-01.md memory/archives/2026-03/

# 创建压缩包
tar -czf memory-2026-03.tar.gz memory/archives/2026-03/
```

### 🧹 清理操作
```bash
# 清理临时文件
rm -rf output/
rm -rf node_modules/

# 清理缓存
find . -name "__pycache__" -type d -exec rm -rf {} +
```

### ✅ Git 操作
```bash
# 查看所有更改
git status

# 添加并提交
git add .
git commit -m "chore: 清理临时文件"

# 推送到远程
git push
```
