## 🦞 什么是 OpenClaw？

### 核心概念
**OpenClaw** 是一个**自托管网关**，连接各种聊天应用（WhatsApp、Telegram、Discord、iMessage 等）到 AI 编程代理（如 Pi）。你在自己的机器上运行一个 Gateway 进程，它成为聊天应用和 AI 助手之间的桥梁。

### 关键特性
| 特性 | 说明 |
|------|------|
| **自托管** | 运行在你的硬件上，你的规则 |
| **多通道** | 一个 Gateway 同时服务 WhatsApp、Telegram、Discord 等 |
| **代理原生** | 为编程代理构建，支持工具使用、会话、记忆、多代理路由 |
| **开源** | MIT 许可，社区驱动 |

### 技术栈
- **Node.js**: 22+
- **模型**: 支持多种 LLM 提供商
- **协议**: 自定义 Gateway 协议
- **存储**: SQLite + JSONL 日志

## 🏗️ 核心架构

### 1. Gateway（网关）
**单一真相源**，负责：
- 会话管理
- 路由分发
- 通道连接
- 状态持久化

```
Chat Apps + Plugins → Gateway → Pi Agent
                        ↓
                     CLI / Web UI
                        ↓
                   iOS/Android Nodes
```

### 2. Agent（代理）
每个代理是一个**完全隔离的大脑**：
- **工作空间**（文件、AGENTS.md/SOUL.md/USER.md）
- **状态目录**（认证配置、模型注册表）
- **会话存储**（聊天历史）

### 3. Session（会话）
会话是 OpenClaw 的核心单元：
- **DM 会话**: 按 `session.dmScope` 配置
  - `main` (默认): 所有 DM 共享主会话
  - `per-peer`: 按发送者隔离
  - `per-channel-peer`: 按通道 + 发送者隔离
  - `per-account-channel-peer`: 多账户场景
- **Group 会话**: `agent:<agentId>:<channel>:group:<id>`
- **Cron 会话**: `cron:<job.id>`
- **Node 会话**: `node-<nodeId>`

### 4. Memory（记忆）
记忆系统采用**三层架构**：
1. **每日日志** (`memory/YYYY-MM-DD.md`) - 追加写入
2. **精选记忆** (`MEMORY.md`) - 定期整理，**仅在主会话加载**
3. **长期归档** (可选) - 7 天后自动移动

记忆工具：
- `memory_search` - 语义搜索
- `memory_get` - 精准读取

## 🔧 CLI 命令参考

### 核心命令
```bash
# 安装和配置
npm install -g openclaw@latest
openclaw onboard --install-daemon
openclaw configure

# Gateway 管理
openclaw gateway status
openclaw gateway restart
openclaw gateway start/stop

# 会话管理
openclaw sessions list
openclaw sessions cleanup --dry-run

# 代理管理
openclaw agents list --bindings
openclaw agents add <agentId>

# 通道管理
openclaw channels list
openclaw channels login --channel whatsapp

# 定时任务
openclaw cron list
openclaw cron add --name "Reminder" --at "2026-03-10T07:00:00Z"

# 诊断
openclaw status
openclaw doctor
openclaw health
```

### 全局标志
- `--dev`: 隔离状态到 `~/.openclaw-dev`
- `--profile <name>`: 使用命名配置
- `--no-color`: 禁用 ANSI 颜色
- `--json`: 机器可读输出

## 🎯 多代理路由

### 路由规则（优先级从高到低）
1. `peer` 匹配（精确 DM/群组 ID）
2. `parentPeer` 匹配（线程继承）
3. `guildId + roles`（Discord 角色路由）
4. `guildId`（Discord）
5. `teamId`（Slack）
6. `accountId` 匹配通道账户
7. 通道级别匹配
8. 默认代理（fallback）

### 安全模式
**Secure DM mode**（推荐多用户设置）:
```json
{
  "session": {
    "dmScope": "per-channel-peer"
  }
}
```

## ⏰ 定时任务系统

### Cron vs Heartbeat

| 使用场景 | Cron | Heartbeat |
|----------|------|-----------|
| **精确时间** | ✅ "每天 7 点准时" | ❌ 漂移容忍 |
| **任务隔离** | ✅ 独立会话 | ❌ 主会话 |
| **批量检查** | ❌ | ✅ 邮箱 + 日历 + 通知 |
| **一次性提醒** | ✅ | ❌ |

### 执行模式
- **Main session**: 运行在心跳提示下，使用主上下文
- **Isolated**: 在 `cron:<jobId>` 会话中运行独立代理

### 交付模式
- `announce`: 发送到目标通道
- `webhook`: POST 到 URL
- `none`: 仅内部

## 🧠 记忆系统实践

### 何时写入记忆
- **决策、偏好、持久事实** → `MEMORY.md`
- **日常笔记、运行上下文** → `memory/YYYY-MM-DD.md`
- **"记住这个"** → **必须写入文件！**（RAM 不留记忆）

### 自动记忆刷新
当会话接近自动压缩时，OpenClaw 会触发**静默记忆刷新**：
- 提醒模型在压缩前写入持久记忆
- 默认 `NO_REPLY` 避免打扰
- 由 `agents.defaults.compaction.memoryFlush` 控制

## 🔐 安全实践

### 1. 权限控制
- 主代理凭据**不共享**给其他代理
- 多用户 DM 启用 `per-channel-peer` 隔离
- `openclaw security audit` 检查配置

### 2. Git 工作空间
- 双仓库架构：开发环境 + Hexo 博客
- **删除前确认**，使用 `trash` 代替 `rm`
- 每 6 小时自动备份

### 3. 安全建议
- 私有数据不外泄
- 外部操作前先询问
- 不确定时问清楚

## 📝 汇报要点总结

### 1. OpenClaw 是什么？
> 一个自托管的聊天网关，连接各种通讯应用到 AI 代理，让你随时随刻都能和 AI 对话。

### 2. 核心亮点
- ✅ 自托管（你的数据你做主）
- ✅ 多通道（一个 Gateway 搞定所有 App）
- ✅ 多代理（不同场景不同 AI）
- ✅ 内存持久化（文本即记忆）

### 3. 技术架构
```
用户 → 聊天 App → Gateway → 代理 → AI 模型
              ↓
          工具/内存/状态
```

### 4. 使用场景
- 📱 个人助手：通过微信/Telegram 随时对话
- 💼 工作代理：独立工作空间处理业务
- 🤖 自动化：定时任务 + Webhook
- 📚 记忆管理：自动整理学习笔记

### 5. 安全特性
- 会话隔离
- DM 权限控制
- 本地存储
- 审计工具
