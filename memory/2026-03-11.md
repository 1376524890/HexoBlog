### 核心知识点掌握

#### 1. OpenClaw 是什么？
- **定位**: AI Agent 运行时平台，不是聊天机器人
- **核心**: Runtime Gateway（智能网关）
- **作用**: 把 AI 连接到真实世界的桥梁
- **核心理念**:
  1. Access control before intelligence（访问控制先于智能）⭐⭐⭐⭐⭐
  2. 隐私优先：私有数据保持私有
  3. 记忆即文件：所有记忆写入 Markdown 文件
  4. 工具优先：第一类工具而非 skill 包裹

#### 2. 三层架构
```
Agent Layer（智能层）
  - Main Agent、Subagents、ACP Agents
    ↓
Gateway Layer（网关层）← 大脑！不运行 AI，只是调度员
  - 控制平面、路由、安全、会话管理
    ↓
Node Layer（节点层）← 手脚
  - 设备能力、远程执行、移动端 App
```

#### 3. 四大核心组件
| 组件 | 作用 | 关键点 |
|------|------|--------|
| Gateway | 大脑、路由器 | 不运行 AI 模型，只是调度员 |
| Agent | 执行 AI 任务 | 身份 + 配置 + 状态 + 运行时 |
| Session | 有状态容器 | 消息历史、上下文、工具状态 |
| Channel | 协议适配器 | Telegram、Discord、飞书等 |

#### 4. 工具系统
- **8 大类工具**: Runtime、Filesystem、Session、Memory、Web、UI、Node、Messaging
- **16 个已安装 Skills**: hexo-blog、task-tracker、weather 等
- **MCP 协议**: 标准化 AI 与外部世界的交互接口

#### 5. 多智能体系统（御坂网络第一代）
- **1 号主 Agent**: 全能助手，核心中枢
- **7 个子代理**: 10-17 号，各司其职
- **深度层级**: Depth 0-2，最大嵌套深度 1-5

#### 6. 记忆系统
- **三层架构**: Session Memory → Task Memory → Long-term Memory
- **文件位置**: `MEMORY.md` + `memory/YYYY-MM-DD.md`
- **工具**: `memory_search`、`memory_get`

#### 7. 安全模型
