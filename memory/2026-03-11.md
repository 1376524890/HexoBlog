### 核心概念掌握

**1. OpenClaw 是什么**
- 智能体网关系统，类似"个人版 Autogen"
- 但不是纯框架，是完整系统
- 核心：多通道支持 + 技能系统 + 记忆系统 + 自动化能力

**2. 四层架构**
```
用户交互层 → Gateway(WS 服务) → Agent 层 → 工具层
```

**3. 记忆系统**
- 双重架构：短期 (`memory/YYYY-MM-DD.md`) + 长期 (`MEMORY.md`)
- 向量检索 + 混合搜索 (BM25 + 向量)
- 自动压缩 + 时间衰减

**4. 技能系统**
- AgentSkills 兼容的目录
- ClawHub 管理 (`clawhub install/update/sync`)
- 三层优先级：workspace > managed > bundled

**5. 会话管理**
- DMScope 隔离策略：`main` / `per-peer` / `per-channel-peer`
- 生命周期：创建 → 维护 → 重置 → 清理
- 自动重置：每日 4 点、闲置超时

**6. 节点系统**
- 远程设备执行
- 支持命令执行、摄像头、屏幕录制、Canvas 展示
- 4 级安全权限

**7. 多智能体架构**
- 每个 Agent 独立工作空间
- 路由绑定规则
- 提高专注度和安全性

**8. 自动化能力**
- Cron 定时任务
- Heartbeat 心跳机制
- Webhook 集成

### CLI 常用命令
```bash
openclaw status --deep          # 状态检查
openclaw sessions list          # 会话管理
openclaw channels status        # 通道状态
openclaw skills check           # 技能检查
openclaw models status          # 模型状态
openclaw security audit         # 安全审计
openclaw logs --follow          # 日志追踪
```
