### 核心知识点

#### 1️⃣ OpenClaw 定义

**一句话**：OpenClaw 是 **AI Agent 运行时平台**，核心是**智能网关（Runtime Gateway）**。

**四大核心理念**（必背）：
1. ✅ **Access control before intelligence**（访问控制先于智能）⭐⭐⭐⭐⭐
2. ✅ **隐私优先**：私有数据保持私有
3. ✅ **记忆即文件**：所有记忆写入 Markdown 文件
4. ✅ **工具优先**：第一类工具而非 skill 包裹

#### 2️⃣ 三层架构

```
Agent Layer（智能层）
  ↓
Gateway Layer（网关层）← 大脑！不运行 AI 模型，只是调度员
  ↓
Node Layer（节点层）← 手脚
```

**四大核心组件**：
- **Gateway** - 大脑、路由器
- **Agent** - 执行 AI 任务
- **Session** - 有状态容器
- **Channel** - 协议适配器

**Agent Loop**：接收输入 → 构建上下文 → LLM 推理 → 工具执行 → 循环或结束 → 发送响应

#### 3️⃣ 工具系统（8 大分类）

- Runtime: `exec`, `process`, `gateway`

- Web: `web_search`, `web_fetch`

### 常见问题预判（FAQ）

1. **OpenClaw 和 ChatGPT 的区别？**
   - ChatGPT 是聊天机器人，OpenClaw 是 Agent 运行时，能真正执行任务

2. **数据安全性如何保障？**
   - 自托管、三层权限模型、审计日志、沙箱隔离

3. **能否在云端部署？**
   - 可以，但推荐本地部署保证数据私有

4. **如何扩展功能？**
   - 通过 Skills 系统，自定义或从 ClawHub 安装

5. **是否支持中文？**
   - 支持，所有文档和界面都支持多语言

6. **是否需要付费？**
   - 开源免费，但需要第三方 API

7. **记忆会丢失吗？**
   - 不会，记忆即文件，持久化到磁盘

8. **支持哪些消息平台？**
   - 支持 Telegram、Discord、Slack、WhatsApp、飞书等 15+ 个
