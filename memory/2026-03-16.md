## 📚 学习成果

### 1. OpenClaw 核心概念

**定义**: OpenClaw 是一个自托管网关（self-hosted gateway），连接聊天应用（WhatsApp、Telegram、Discord、iMessage 等）与 AI 编程代理。

**四个核心特点**:
1. 自托管 - 运行在你的硬件上
2. 多通道 - 单个 Gateway 服务多个平台
3. 代理原生 - 为 AI 代理构建
4. 开源 - MIT 许可证

### 2. 系统架构

```
聊天应用 → Gateway → AI Agent
              ↓
         CLI / Web UI / 节点
```

- **Gateway**: 唯一事实来源，维护通道连接，通过 WebSocket 暴露 API
- **客户端**: macOS app、CLI、Web UI
- **节点**: macOS/iOS/Android/headless

### 3. 会话管理系统

**会话 Key 结构**:
- 直接聊天：`agent:<agentId>:<mainKey>`
- 群聊：`agent:<agentId>:<channel>:group:<id>`

**DM 隔离模式 (dmScope)**:
- `main` - 所有 DM 共享主会话（单用户）
- `per-peer` - 按发送者隔离（多用户）
- `per-channel-peer` - 按通道 + 发送者隔离（推荐）
- `per-account-channel-peer` - 多账户场景

**安全警告**: 如果代理接收多人的 DM，必须启用安全 DM 模式（`dmScope: "per-channel-peer"`），否则隐私信息会泄露！

### 4. 多代理路由系统

每个代理拥有独立的工作空间、状态目录和会话存储。

**路由规则（最具体优先）**:
1. peer 匹配（精确 DM/群组 id）
2. parentPeer 匹配（线程继承）
3. guildId + roles（Discord）
4. guildId（Discord）
5. teamId（Slack）
6. accountId 匹配
7. 通道级匹配
8. 回退到默认代理

### 5. 工具系统

**11 大类工具**:
- group:runtime - exec, bash, process
- group:fs - read, write, edit, apply_patch
- group:sessions - sessions_list, sessions_history, sessions_send, sessions_spawn, session_status
- group:memory - memory_search, memory_get
- group:web - web_search, web_fetch
- group:ui - browser, canvas
- group:automation - cron, gateway
- group:messaging - message
- group:nodes - nodes
- group:openclaw - 所有内置工具

**工具策略**:
- profile: minimal/coding/messaging/full
- byProvider: 按提供者限制
- allow/deny: 精细控制

### 6. 节点系统

**支持平台**: macOS、iOS、Android、headless

**核心功能**:
- Canvas - 可视化界面渲染
- Camera - 拍照、视频录制
- Screen Record - 屏幕录制
- Location - 位置获取
- Notifications - 系统通知
- Voice Wake - 语音唤醒

### 7. 安全与认证

**关键安全要点**:
1. DM 安全模式 - 多用户场景必须启用
2. 认证令牌 - 所有连接必须签署 nonce
3. 工具策略 - 通过 allow/deny 控制权限
4. 沙箱隔离 - 每代理独立沙箱
