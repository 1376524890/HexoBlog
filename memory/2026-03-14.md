## 🎯 核心知识点总结

### 1. OpenClaw 是什么？

**一句话定义**: OpenClaw 是 AI Agent 运行时平台，核心是**智能网关（Runtime Gateway）**，不是聊天机器人，而是把 AI 模型连接到真实世界的桥梁。

**四大核心理念**（必背）:
1. ✅ Access control before intelligence（访问控制先于智能）⭐⭐⭐⭐⭐
2. ✅ 隐私优先（私有数据保持私有）⭐⭐⭐⭐
3. ✅ 记忆即文件（所有记忆写入 Markdown 文件）⭐⭐⭐⭐⭐
4. ✅ 工具优先（第一类工具而非 skill 包裹）⭐⭐⭐

### 2. 核心架构

**三层架构**:
- **Agent Layer（智能层）**: Main Agent、Subagents、ACP Agents
- **Gateway Layer（网关层）← 大脑**: 控制平面、策略层、路由、身份认证、会话管理（不运行 AI 模型，只是调度员）
- **Node Layer（节点层）← 手脚**: 设备能力（相机/屏幕）、远程执行、移动端 App

**四大核心组件**:
- **Gateway**: 大脑、路由器（生命周期管理、消息路由、工具协调、安全控制）
- **Agent**: 执行 AI 任务（身份 + 配置 + 状态 + 运行时）
- **Session**: 有状态容器（消息历史、上下文、工具状态）
- **Channel**: 协议适配器（Telegram、Discord、飞书等）

**Agent Loop（核心循环）**:
1. 接收输入 → 用户通过 Channel 发送消息
2. 构建上下文 → 组装 Session 历史、系统提示词、工具列表
3. LLM 推理 → 模型决定"直接回复"或"调用工具"
4. 工具执行 → 如需多步骤，通过 Gateway 调用外部工具
5. 循环或结束 → 多步推理继续，否则返回最终结果
6. 发送响应 → Gateway 通过原 Channel 发送给用户

### 5. 记忆系统（三层架构）

```
Layer 1: 会话记忆（Session Memory）
- 当前会话上下文
- 临时决策和中间结果
    ↓ 同步关键信息
Layer 2: 任务记忆（Task Memory）
- 任务计划文件
- 子代理执行结果
    ↓ 同步重要发现
Layer 3: 长期记忆（Long-term Memory）
- MEMORY.md：精选记忆
- memory/YYYY-MM-DD.md：每日日志
```

**最佳实践**:
- ✅ DECIDE to write: 决定、偏好、持久事实 → MEMORY.md
- ✅ Daily notes: 日常记录 → memory/YYYY-MM-DD.md
- ✅ 定期 review: 定期清理 MEMORY.md
- ✅ Ask to remember: 重要事项明确让 Agent 写入记忆
- ✅ 立即提交：操作后 `git add` 和 `git commit`

**安全操作规则**:
- ✅ 使用 `trash` 而不是 `rm`
- ✅ 操作前备份：修改 `MEMORY.md` 前自动备份
- ✅ 检查 Git 状态：操作前确认 `git status`
- ✅ 安全检查：运行 `python3 scripts/safety-check-memory.py`
- ✅ 立即提交：操作后 `git add` 和 `git commit`

### 6. 安全模型

**核心原则**:
1. ✅ 单一信任边界（Gateway 和 Node 属于同一信任域）
2. ✅ 私有数据保持私有
3. ✅ 权限最小化
4. ✅ 审计追踪
