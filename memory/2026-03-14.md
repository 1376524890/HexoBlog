## 📚 学习内容总结

### 一、核心架构学习（10 分钟阅读 + 总结）

#### 三层架构
- **Agent Layer（智能层）** - 主 Agent、子代理、编码代理
- **Gateway Layer（网关层）** - 大脑，不运行 AI 模型，只是调度员
- **Node Layer（节点层）** - 手脚，设备能力、远程执行、移动端

#### 四大核心组件
1. **Gateway** - 大脑、路由器，核心职责：
   - 生命周期管理
   - 消息路由
   - 工具协调
   - 安全控制
   - 状态持久化

2. **Agent** - 执行 AI 任务
3. **Session** - 有状态容器，通过 Compaction 压缩机制管理上下文
4. **Channel** - 协议适配器，支持多个平台

#### Agent Loop（核心循环）⭐⭐⭐⭐⭐
1. 接收输入
2. 构建上下文
3. LLM 推理
4. 工具执行
5. 循环或结束
6. 发送响应

### 二、四大核心理念（必背）⭐⭐⭐⭐⭐

1. **Access control before intelligence** - 访问控制先于智能
2. **隐私优先** - 私有数据保持私有
3. **记忆即文件** - 所有记忆写入 Markdown 文件
4. **工具优先** - 第一类工具而非 skill 包裹

### 五、记忆系统（5 分钟）⭐⭐⭐⭐⭐

#### 三层记忆架构
```
Layer 1: 会话记忆
- 当前会话上下文
- 临时决策和中间结果
       ↓
Layer 2: 任务记忆
- 任务计划文件
- 子代理执行结果
       ↓
Layer 3: 长期记忆
- MEMORY.md: 精选记忆
- memory/YYYY-MM-DD.md: 每日日志
```

#### 记忆管理最佳实践
1. DECIDE to write: 决定、偏好、持久事实 → MEMORY.md
2. Daily notes: 日常记录 → memory/YYYY-MM-DD.md
3. 定期 review: 定期清理 MEMORY.md
4. Ask to remember: 重要事项明确让 Agent 写入记忆
5. 立即提交：操作后 `git add` 和 `git commit`
