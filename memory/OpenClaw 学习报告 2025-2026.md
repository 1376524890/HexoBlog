## 🏗️ 二、核心架构与功能演进

### 1. 网关架构 (Gateway)

**架构定位**：运行时网关（Runtime Gateway）

```
┌─────────────────────────────────────────────────────────────┐
│                      OpenClaw Gateway                        │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐    │
│  │ Telegram │  │ Discord  │  │  Slack   │  │  其他平台 │    │
│  └────┬─────┘  └────┬─────┘  └────┬─────┘  └────┬─────┘    │
│       └─────────────┴─────────────┴─────────────┘           │
│                         │                                    │
│                    Channel Layer                            │
│                         │                                    │
│              ┌──────────┴──────────┐                        │
│              │   Session Manager   │                        │
│              └──────────┬──────────┘                        │
│                         │                                    │
│              ┌──────────┴──────────┐                        │
│              │     Agent Pool      │                        │
│              └──────────┬──────────┘                        │
│                         │                                    │
│              ┌──────────┴──────────┐                        │
│              │   Tool Registry     │ ← Skills/MCP          │
│              └──────────┬──────────┘                        │
│                         │                                    │
│              ┌──────────┴──────────┐                        │
│              │   LLM Providers     │                        │
│              └─────────────────────┘                        │
└─────────────────────────────────────────────────────────────┘
```

**核心职责**：
- **生命周期管理**：启动、停止、监控所有 Agent 实例
- **消息路由**：将消息分发到正确的 Session 和 Agent
- **工具协调**：管理 Skill 注册，处理工具调用请求
- **安全控制**：执行沙箱策略，管理权限边界
- **状态持久化**：维护 Session 历史，处理上下文压缩

### 3. 核心组件

#### 3.1 Agent（AI 执行体）
- 每个 Agent 拥有独立的身份、配置、状态、运行时
- 运行在**隔离环境**中，通过 Bridge Protocol 与 Gateway 通信
- 可以执行代码、访问文件、调用外部 API

#### 3.2 Session（有状态的容器）
- **消息历史**：完整对话记录
- **上下文窗口**：经过压缩的有效上下文
- **工具状态**：工具调用的中间结果
- **元数据**：创建时间、活跃时间、关联 Channel

**关键机制**：**Compaction（上下文压缩）**
- 当历史超过阈值时，智能总结和裁剪
- 保留关键信息，释放空间

#### 3.3 Nodes（设备节点）
- **macOS 节点**：系统命令执行、通知、屏幕录制
- **iOS 节点**：Canvas、语音触发、相机、屏幕录制
- **Android 节点**：Camera、屏幕录制、通知、位置、SMS、联系人、日历

### 4. 工具系统 (Tools)

**工具类型**（2026 年新增核心能力）：

| 类别 | 工具 | 说明 |
|------|------|------|
| **Browser** | `browser` | 浏览器控制（snapshot、action、screenshot） |
| **Canvas** | `canvas` | 画布渲染、A2UI 推送 |
| **Nodes** | `nodes` | 设备控制（相机、屏幕、位置等） |
| **Cron** | `cron` | 定时任务管理 |
| **Sessions** | `sessions_*` | 会话间通信（跨 Agent 协作） |
| **Web** | `web_search`, `web_fetch` | 网络搜索和内容抓取 |
| **Message** | `message` | 跨平台消息发送 |
| **Gateway** | `gateway` | 网关重启、配置管理 |
| **Runtime** | `exec`, `process`, `read`, `write`, `edit` | 运行时能力 |

**工具配置示例**：
```json
{
  "tools": {
    "profile": "coding",
    "allow": ["group:fs", "browser"],
    "deny": ["group:automation"]
  }
}
```

**工具组 (Tool Groups)**：
- `group:runtime`：exec, bash, process
- `group:fs`：read, write, edit, apply_patch
- `group:sessions`：sessions_list, sessions_history, sessions_send, sessions_spawn
- `group:messaging`：message
- `group:ui`：browser, canvas
- `group:automation`：cron, gateway
- `group:nodes`：nodes
- `group:web`：web_search, web_fetch

## 🎉 三、2026 年新增功能

### 1. 会话间通信 (Sessions)

**核心功能**：
- `sessions_list`：列出活跃会话
- `sessions_history`：查看历史对话
- `sessions_send`：向其他会话发送消息
- `sessions_spawn`：创建子 agent
- `session_status`：查看会话状态

**使用场景**：
- 御坂网络第一代的核心通信机制
- 多 Agent 协作、任务分发
- 上下文隔离、故障隔离

### 2. A2UI (Agent to User Interface)

**Canvas 功能增强**：
- 代理驱动的可视化工作空间
- `canvas` → `present` 推送界面
- `canvas` → `a2ui_push` 动态内容
- `canvas` → `snapshot` 界面截图

**应用场景**：
- 可视化数据展示
- 交互式界面
- 动态内容渲染

### 3. Voice Wake + Talk Mode

**语音功能**：
- **Voice Wake**：macOS/iOS 上的唤醒词
- **Talk Mode**：Android 上的连续语音对话
- 支持 ElevenLabs TTS
- 系统 TTS 降级方案

### 4. 本地 Canvas 渲染

**Canvas 功能**：
- `/__openclaw__/canvas/` - 可编辑的 HTML/CSS/JS
- `/__openclaw__/a2ui/` - A2UI 宿主
- 使用与 Gateway 相同的端口（默认 18789）

### 5. 会话管理增强

**会话策略**：
- `main`：直接对话
- `group`：群组隔离
- `activation_modes`：激活模式
- `queue_modes`：队列模式

**上下文压缩**：
- 当历史超过阈值时自动压缩
- 保留关键信息，释放 token 空间

### 6. 安全沙箱增强

**沙箱模式**：
- `non-main`：非主会话在 Docker 沙箱中运行
- `bash` 工具沙箱化
- 工具访问控制（allowlist/denylist）
- `elevated` 权限控制

**安全默认值**：
- DM 配对政策（pairing/open）
- 工具访问限制
- Docker 隔离

## 🔍 九、技术细节

### 1. WebSocket 协议

**连接流程**：
```
Client → Gateway: req:connect
Gateway → Client: res (ok)
Gateway → Client: event:presence
Gateway → Client: event:tick
Client → Gateway: req:agent
Gateway → Client: res:agent
Gateway → Client: event:agent (streaming)
Gateway → Client: res:agent (final)
```

**协议特点**：
- WebSocket 传输
- JSON 格式
- 必须首先发送 `connect`
- 支持请求/响应/事件三种类型

### 2. Session 生命周期

**创建**：用户发送消息时自动创建
**活跃**：接收/发送消息
**压缩**：超过阈值时自动压缩
**清理**：长时间未活跃时清理

### 3. 工具调用流程

**Agent Loop**：
1. 接收输入
2. 构建上下文
3. LLM 推理
4. 决定调用工具或直接回复
5. 执行工具
6. 获取结果
7. 循环或结束
8. 发送响应
