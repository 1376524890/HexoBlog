## 📝 核心知识点速记

### 核心定义
- **OpenClaw**: AI Agent 运行时平台，核心是智能网关
- **Gateway**: 不运行 AI 模型，只是调度员
- **Agent**: 身份 + 配置 + 状态 + 运行时
- **Session**: 有状态的会话容器
- **Channel**: 协议适配器（Telegram、Discord、飞书等 15+ 个）

### 四大核心理念
1. Access control before intelligence（访问控制先于智能）⭐⭐⭐⭐⭐
2. 隐私优先：私有数据保持私有
3. 记忆即文件：所有记忆写入 Markdown 文件
4. 工具优先：第一类工具而非 skill 包裹

### 三层架构
```
Agent Layer（智能层）→ 大脑
  ↓
Gateway Layer（网关层）← 路由器
  ↓
Node Layer（节点层）→ 手脚
```

### Agent Loop
1. 接收输入 → 用户通过 Channel 发送消息
2. 构建上下文 → 组装 Session 历史、系统提示词、工具列表
3. LLM 推理 → 模型决定是"直接回复"还是"调用工具"
4. 工具执行 → 如需多步骤，通过 Gateway 调用外部工具
5. 循环或结束 → 多步推理继续，否则返回最终结果
6. 发送响应 → Gateway 通过原 Channel 发送给用户

### 工具系统
- **8 大分类**: Runtime、Filesystem、Session、Memory、Web、UI、Node、Messaging
- **Feishu 集成**: 文档、云盘、知识库、多维表格
- **MCP 协议**: 基于 Anthropic 的 Model Context Protocol

### 御坂网络第一代
- 7 个子代理：10 号通用、11 号 Code、12 号 Write、13 号 Research、14 号 File、15 号 Sys、17 号 Memory
- 权限分级：Level 1-5
- 调用方式：sessions_spawn

### 三层记忆架构
```
Layer 1: 会话记忆（Session Memory）
  ↓
Layer 2: 任务记忆（Task Memory）
  ↓
Layer 3: 长期记忆（Long-term Memory）
```

### 安全模型
- 5 级权限控制
- 审计命令完善
- 加固基线配置
