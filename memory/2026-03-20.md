## 📝 学习心得

### 最重要的 10 个发现

1. **OpenClaw 不是聊天机器人，是做事的 Agent**
   - 能真正执行任务，不只是生成文本
   - 通过工具系统连接到真实世界

2. **四大核心理念至关重要**
   - 访问控制先于智能（最重要！）
   - 隐私优先：私有数据保持私有
   - 记忆即文件：纯 Markdown 持久化
   - 工具优先：第一类工具而非 skill 包裹

3. **三层架构清晰**
   - Agent Layer（智能层）
   - Gateway Layer（网关层 - 大脑，不运行 AI）
   - Node Layer（节点层 - 手脚）

4. **御坂网络第一代架构**
   - 7 个子代理各司其职
   - 权限等级分明
   - 专业分工，效率更高

5. **三层记忆架构**
   - 会话记忆（临时）
   - 任务记忆（中间）
   - 长期记忆（永久）
   - WAL 协议确保不丢失

6. **8 大工具分类**
   - Runtime、Filesystem、Session、Memory
   - Web、UI、Node、Messaging
   - Feishu 全家桶集成完善

7. **16 个 Skills**
   - 模块化、标准化、可扩展
   - 从 ClawHub 获取或自定义
   - 安全审查必须先做

8. **安全模型完善**
   - 5 级权限控制
   - 审计命令完善
   - 工具 Profile 可配置
   - 沙箱隔离保护

9. **Agent Loop 核心流程**
   - 接收输入 → 构建上下文 → LLM 推理 → 工具执行 → 发送响应
   - 模型拥有决策权

10. **记忆持久化**
    - 所有记忆写入 Markdown 文件
    - 即使会话重启也能恢复
    - 语义检索支持高效查找

### 与 ChatGPT 的本质区别

| 维度 | ChatGPT | OpenClaw |
|------|---------|----------|
| 定位 | 聊天机器人 | Agent 运行时平台 |
| 能力 | 生成文本 | 真正执行任务 |
| 记忆 | 会话内临时 | 持久化到磁盘 |
| 工具 | 有限 | 丰富（文件系统、命令、浏览器等）|
| 部署 | 云端 | 本地/私有 |
| 安全 | 受限于平台 | 多层次控制 |
