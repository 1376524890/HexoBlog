## 🎯 核心知识点回顾（必背）

### 1. OpenClaw 定义（精通）⭐⭐⭐⭐⭐

**一句话**：
> **OpenClaw 是 AI Agent 运行时平台**，核心是**智能网关（Runtime Gateway）**。  
> **它不是聊天机器人**，而是把 AI 连接到真实世界的桥梁。

**四大核心理念（必背）**：
1. **Access control before intelligence**（访问控制先于智能）⭐⭐⭐⭐⭐
2. **隐私优先**：私有数据保持私有
3. **记忆即文件**：所有记忆写入 Markdown 文件
4. **工具优先**：第一类工具而非 skill 包裹

### 2. 三层架构（精通）⭐⭐⭐⭐⭐

```
Agent Layer（智能层）
  - Main Agent、Subagents、ACP Agents
    ↓
Gateway Layer（网关层）← 大脑！
  - 控制平面、策略层、路由、身份认证、会话管理
  - 频道适配器（15+ 个聊天平台）
  ⚠️ Gateway 本身不运行 AI 模型，只是调度员
    ↓
Node Layer（节点层）← 手脚
  - 远程执行表面
  - 设备能力（摄像头、屏幕、通知、位置）
  - macOS companion app, iOS/Android nodes
```

**关键记忆点**：
- Gateway **不是 AI 模型**，只是调度员和控制平面
- Agent 拥有决策权，可以主动调用工具
- Node 提供硬件访问能力

### 3. Session Key 格式（熟练）⭐⭐⭐⭐⭐

| 类型 | 格式 | 说明 |
|------|------|------|
| 直接聊天 | `agent:<agentId>:main` 或 `agent:<agentId>:direct:<peerId>` | 单用户场景 |
| 群组聊天 | `agent:<agentId>:<channel>:group:<id>` | 群聊场景 |
| 频道聊天 | `agent:<agentId>:<channel>:channel:<id>` | 频道场景 |
| Cron 任务 | `cron:<jobId>` | 定时任务 |
| Webhook | `hook:<uuid>` | 外部 webhook |
| Node 运行 | `node-<nodeId>` | 节点远程执行 |

**dmScope 配置（多用户场景关键）**：
- `main`: 所有 DM 共享主会话（单用户场景）
- `per-peer`: 按发送者 ID 隔离
- `per-channel-peer`: 按渠道 + 发送者隔离（**多用户推荐**）⭐
- `per-account-channel-peer`: 按账户 + 渠道 + 发送者隔离（**多账户推荐**）⭐

### 6. 三层记忆架构（精通）⭐⭐⭐⭐⭐

```
Layer 1: 会话记忆（Session Memory）
- 当前会话上下文
- 临时决策和中间结果
    ↓ 同步关键信息
Layer 2: 任务记忆（Task Memory）
- 任务计划文件
- 子代理执行结果
    ↓ 同步重要发现
Layer 3: 长期记忆（Long-term Memory）
- MEMORY.md：精选记忆
- memory/YYYY-MM-DD.md：每日日志
```

**WAL Protocol（写后读协议）**:
- **STOP** — 不要立即回复
- **WRITE** — 更新记忆文件
- **THEN** — 回复用户

**触发条件**: 修正、专有名词、偏好、决策、编辑、数值

### 7. 安全模型（精通）⭐⭐⭐⭐⭐

**5 级权限体系**:
| 级别 | 类型 | 权限范围 |
|------|------|----------|
| Level 5 | 主 Agent | 完全权限 |
| Level 4 | 可信子 Agent | 受限系统权限（需批准）|
| Level 3 | 标准子 Agent | 标准开发权限 |
| Level 2 | 受限子 Agent | 严格受限权限 |
| Level 1 | 只读子 Agent | 只读访问 |

**安全机制**:
- **设备配对**: 所有连接包含 device identity，新设备需要批准
- **沙箱隔离**: Agent 运行在隔离环境中
- **工具控制**: `tools.allow` / `tools.deny`
- **审计日志**: `openclaw security audit --deep`

**安全审计命令**:
```bash
openclaw security audit       # 基本检查
openclaw security audit --deep # 深度检查
openclaw security audit --fix  # 自动修复
```

### 8. Cron vs Heartbeat（熟练）⭐⭐⭐⭐

**使用 Heartbeat 当**:
- 多个检查可以批量处理（邮件 + 日历 + 通知）
- 需要对话上下文
- 时间可以稍有漂移（每~30 分钟）
- 想通过组合周期性检查减少 API 调用

**使用 Cron 当**:
- 精确时间要求（"每周一 9:00 整"）
- 任务需要与主会话历史隔离
- 想要不同的模型或思考级别
- 一次性提醒（"20 分钟后提醒我"）
- 输出应直接传递到 channel 而不涉及主会话
