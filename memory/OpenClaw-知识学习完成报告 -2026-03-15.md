## 🎓 核心知识点掌握

### 1. OpenClaw 是什么？

> **一句话**: OpenClaw 是 AI Agent 运行时平台，不是聊天机器人，而是把 AI 连接到真实世界的桥梁。

### 2. 四大核心理念

| 理念 | 理解 |
|------|------|
| **Access control before intelligence** | 访问控制先于智能，安全是第一位的 |
| **隐私优先** | 数据在自己手中，不用担心泄露 |
| **记忆即文件** | 所有记忆写入 Markdown 文件，不丢失 |
| **工具优先** | 第一类工具而非 skill 包裹 |

### 3. 三层架构（记忆口诀）

```
Agent Layer（智能层）← 大脑
  ↓
Gateway Layer（网关层）← 路由器/调度员
  ↓
Node Layer（节点层）← 手脚
```

**关键认知**: Gateway 本身**不运行 AI 模型**，只是调度员！

### 4. 四大核心组件

| 组件 | 作用 | 关键点 |
|------|------|--------|
| **Gateway** | 大脑、路由器 | 不运行 AI 模型，只是调度员 |
| **Agent** | 执行 AI 任务 | 身份 + 配置 + 状态 + 运行时 |
| **Session** | 有状态容器 | 消息历史、上下文、工具状态 |
| **Channel** | 协议适配器 | Telegram、Discord、飞书等 |

### 5. 御坂网络第一代

**11 个 Agent**:
- 主 Agent (main) - 任务调度
- 通用代理 (general-agent) - 琐碎任务
- Code 执行者 (code-executor) - 代码编写
- 内容创作者 (content-writer) - 文章撰写
- 研究分析师 (research-analyst) - 信息搜索
- 文件管理器 (file-manager) - 文件操作
- 系统管理员 (system-admin) - 系统配置
- 网络爬虫 (web-crawler) - 网页抓取
- 记忆整理专家 (memory-organizer) - 记忆维护
- 质量审核者 (reviewer) - 质量审核
- 状态监控者 (patrol) - 状态监控

### 6. 三层记忆架构

```
Layer 1: 会话记忆（Session Memory）
  ↓
Layer 2: 任务记忆（Task Memory）
  ↓
Layer 3: 长期记忆（Long-term Memory）
  - MEMORY.md: 精选记忆
  - memory/YYYY-MM-DD.md: 每日日志
```

### 7. 安全模型（5 级权限）

| 级别 | 类型 | 权限范围 |
|------|------|----------|
| Level 5 | 主 Agent | 完全权限 |
| Level 4 | 可信子 Agent | 受限系统权限 |
| Level 3 | 标准子 Agent | 标准开发权限 |
| Level 2 | 受限子 Agent | 严格受限权限 |
| Level 1 | 只读子 Agent | 只读访问 |
