# OpenClaw 心跳通知问题修复 - 2026-03-13

## 问题描述 (2026-03-13 12:22 UTC)

**用户询问**: "我怎么没看到定时任务往我这里发消息"

## 问题分析

### 根本原因
- ❌ `HEARTBEAT.md` 中配置了飞书通知任务，引用脚本 `scripts/heartbeat_to_feishu.py`
- ❌ 但该脚本文件**实际上不存在**
- ✅ 心跳检测本身在运行（通过 heartbeat 会话）
- ❌ 结果只发送到 heartbeat 会话，没有发送到飞书用户

### 配置错误
`HEARTBEAT.md` 中有一行：
```markdown
### 飞书通知任务 (新增)
- ✅ 心跳结果通知 - 每 30 分钟 - `heartbeat_to_feishu`
  - 自动将检测结果发送到飞书当前会话
  - 脚本位置：`scripts/heartbeat_to_feishu.py`
  - 通知格式：Markdown 格式，包含所有检查结果
```

这个配置在 2026-03-12 晚上添加，但脚本从未创建过！

## 解决方案

### 1. 删除错误配置
修改 `HEARTBEAT.md`：
```markdown
### 飞书通知机制
- ✅ 手动触发发送：通过 message 工具直接发送
- ✅ 自动机制：心跳会话 (heartbeat) 自动接收检测结果
```

### 2. 恢复 cron.json
恢复为原始格式（保留 full-backup 任务）：
```json
{
  "tasks": [
    {
      "id": "full-backup",
      "name": "御坂大人系统完整备份",
      "schedule": "0 4 * * *",
      ...
    }
  ]
}
```

### 3. 手动发送测试
使用 message 工具手动发送心跳检测结果到飞书：
- ✅ 发送成功：`om_x100b540da008dc80b4bd446d14e4bca`
- ✅ 格式：Markdown 格式，包含系统健康状态
