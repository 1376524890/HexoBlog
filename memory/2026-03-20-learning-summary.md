### 1. OpenClaw 是什么？
- **定义**: AI Agent 运行时平台，不是聊天机器人
- **核心理念**: 访问控制先于智能、隐私优先、记忆即文件、工具优先
- **与 ChatGPT 的区别**: 能真正执行任务，不只是聊天

### 2. 核心架构（三层）
```
Agent Layer（智能层）
  ↓
Gateway Layer（网关层）← 大脑！不运行 AI，只是调度员
  ↓
Node Layer（节点层）← 手脚
```

### 3. 工具系统
- **8 大类工具**: Runtime、Filesystem、Session、Memory、Web、UI、Node、Messaging
- **Feishu 集成**: 完整的文档、云盘、知识库、多维表格工具
- **MCP 协议**: 基于 Anthropic 的 Model Context Protocol

### 4. 多智能体协作（御坂网络）
- 7 个子代理：10-17 号，各司其职
- 权限分级：Level 1-5
- 调用方式：sessions_spawn

### 5. 记忆系统（三层架构）
```
Layer 1: 会话记忆（Session Memory）
  ↓
Layer 2: 任务记忆（Task Memory）
  ↓
Layer 3: 长期记忆（Long-term Memory）
```

### 6. 安全与最佳实践

## 💡 关键洞察

1. **Gateway 是核心** - 唯一事实来源，统一管理所有通道和会话
2. **不运行 AI 模型** - Gateway 本身只是调度员，智能在 Agent 层

4. **工具系统很强大** - 20+ 工具，精细权限控制
5. **多代理很灵活** - 独立工作空间，动态路由
