## 📚 核心学习内容

### 1. OpenClaw 是什么？
**官方定义**：AI Agent 运行时平台
**准确理解**：智能运行时网关（Smart Runtime Gateway）

**核心特点**：
- Self-hosted（本地部署，数据隐私）
- Multi-channel（多平台支持）
- Agent-native（AI 代码助手原生支持）
- Open source（MIT 许可）

### 2. 四大核心组件

| 组件 | 职责 | 说明 |
|------|------|------|
| **Gateway** | 中央枢纽 | 生命周期管理、消息路由、工具协调 |
| **Agent** | AI 执行体 | 实际执行任务的实例，隔离运行 |
| **Session** | 会话容器 | 有状态会话，管理消息历史和上下文 |
| **Channel** | 消息通道 | 协议适配器，支持多平台 |

### 3. Agent Loop 工作原理

```
接收输入 → 构建上下文 → LLM 推理 → 工具执行 → 循环或结束 → 发送响应
```

**关键点**：模型拥有决策权，主动决定需要什么工具、如何组织答案

### 4. 核心概念

#### Skills（工具包）
- 通过 JSON Schema 描述工具
- 模块化设计，易于扩展
- 示例：Task Tracker、Morning Brief

#### MCP 协议
- Model Context Protocol
- 统一 AI 与外部世界交互的标准
- 一个工具可以被任何支持 MCP 的系统使用

#### Cron 定时任务
- 配置自动任务执行
- 注意时区和环境差异
- 示例：记忆检查点（每 6 小时）、自动备份

#### 三层记忆宫殿
1. **每日日志**：`memory/YYYY-MM-DD.md` - 原始记录，无限存储
2. **精选记忆**：`MEMORY.md` - 精华提取，<3000 字符
3. **长期归档**：`life/archives/` - 高价值保存

### 5. 踩坑记录

| 问题 | 原因 | 解决 |
|------|------|------|
| Session 间通信限制 | 未配置跨 agent 访问 | 设置 `tools.sessions.visibility=all` |
| ACP runtime 未配置 | 后端未安装 | 改用 `runtime: "subagent"` |
| 子 agent 完成后不汇报 | 发送给主 session | 修改 main 会话的 SOUL.md |
| 文档更新不及时 | 忘记同步 | 系统性地更新所有文档 |
| Cron 时区问题 | 系统时区差异 | 在脚本中显式指定时区 |

## 💡 关键收获

1. **OpenClaw 不是简单的聊天机器人，而是智能运行时基础设施**
2. **Session 重启=失忆是系统性问题，需要持久化记忆系统**
3. **三层记忆宫殿既保证了原始记录的完整性，又保证了核心记忆的精华提取**
4. **Skills 系统通过 JSON Schema 实现标准化，MCP 协议统一了 AI 与外部世界的交互**
5. **Cron 定时任务可以配置自动任务，但需要注意时区和环境差异**
