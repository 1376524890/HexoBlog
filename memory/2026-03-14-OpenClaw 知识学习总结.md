## 🎯 核心知识点掌握

### 1. OpenClaw 定义（精通）⭐⭐⭐⭐⭐
> **OpenClaw 是 AI Agent 运行时平台**，不是聊天机器人，而是把 AI 连接到真实世界的桥梁。

**核心特点**:
- 自托管：数据在自己手中
- 多通道：一个网关服务多个平台
- 智能体原生：为 AI 编码智能体设计
- 开源：MIT 许可

**四大核心理念**:
1. **Access control before intelligence**（访问控制先于智能）⭐⭐⭐⭐⭐
2. **隐私优先**：私有数据保持私有
3. **记忆即文件**：所有记忆写入 Markdown 文件
4. **工具优先**：第一类工具而非 skill 包裹

### 2. 三层架构（精通）⭐⭐⭐⭐⭐
```
Agent Layer（智能层）
  ↓
Gateway Layer（网关层）← 大脑！不运行 AI 模型，只是调度员
  ↓
Node Layer（节点层）← 手脚
```

**四大核心组件**:
- **Gateway**: 大脑、路由器（不运行 AI，只是调度员）
- **Agent**: 执行 AI 任务（身份 + 配置 + 状态 + 运行时）
- **Session**: 有状态容器（消息历史、上下文、工具状态）
- **Channel**: 协议适配器（Telegram、Discord、飞书等）

### 3. 工具系统（精通）⭐⭐⭐⭐⭐

### 6. 记忆系统（精通）⭐⭐⭐⭐⭐

**三层架构**:
```
Layer 1: 会话记忆（Session Memory）
- 当前会话上下文
- 临时决策和中间结果
    ↓
Layer 2: 任务记忆（Task Memory）
- 任务计划文件
- 子代理执行结果
    ↓
Layer 3: 长期记忆（Long-term Memory）
- MEMORY.md：精选记忆
- memory/YYYY-MM-DD.md：每日日志
```

**最佳实践**:
1. DECIDE to write：决定、偏好、持久事实 → MEMORY.md
2. Daily notes：日常记录 → memory/YYYY-MM-DD.md
3. 定期 review：定期清理 MEMORY.md
4. Ask to remember：重要事项明确让 Agent 写入记忆
5. 立即提交：`git add` 和 `git commit`

### 7. 安全模型（精通）⭐⭐⭐⭐⭐

**安全架构核心**:
- 单一信任边界（Gateway 和 Node 属于同一信任域）
- 私有数据保持私有
- 权限最小化
- 审计追踪

**权限层级（5 级）**:
- Level 5: 主 Agent - 完全权限
- Level 4: 可信子 Agent - 受限系统权限（需批准）
- Level 3: 标准子 Agent - 标准开发权限
- Level 2: 受限子 Agent - 严格受限权限
- Level 1: 只读子 Agent - 只读访问

**安全审计命令**:
```bash
openclaw security audit       # 基本检查
openclaw security audit --deep # 深度检查
openclaw security audit --fix  # 自动修复
```

### 8. 多 Agent 路由（熟练）⭐⭐⭐⭐

**路由规则**（最具体优先）:
1. `peer` 匹配（精确 DM/群组/频道 id）
2. `parentPeer` 匹配（线程继承）
3. `guildId + roles`（Discord 角色）
4. `guildId`（Discord）
5. `teamId`（Slack）
6. `accountId` 匹配
7. channel-level 匹配
8. 回退到默认 Agent

**一个 Agent 包含**:
- **Workspace**（文件、AGENTS.md/SOUL.md/USER.md）
- **状态目录**（agentDir）用于认证配置文件
- **会话存储**在 `~/.openclaw/agents/<agentId>/sessions`
