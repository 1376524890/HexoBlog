## 🎯 核心知识要点

### 1. OpenClaw 是什么？
- **一句话**：AI Agent 运行时平台，核心是智能网关（Runtime Gateway）
- **定位**：不是聊天机器人，而是把 AI 连接到真实世界的桥梁
- **核心理念**：
  1. Access control before intelligence（访问控制先于智能）⭐
  2. 隐私优先：私有数据保持私有
  3. 记忆即文件：所有记忆写入 Markdown 文件
  4. 工具优先：第一类工具而非 skill 包裹

### 2. 三层架构
```
Agent Layer（智能层）
  ↓
Gateway Layer（网关层）← 大脑！不运行 AI，只是调度员
  ↓
Node Layer（节点层）← 手脚
```

**四大核心组件**：
- Gateway：控制平面、路由、策略层
- Agent：执行 AI 任务
- Session：有状态会话容器
- Channel：协议适配器

### 3. 工具系统
- **16+ 类别工具**：运行时、文件系统、会话、记忆、网络、UI、节点、消息等
- **工具安全策略**：Profile 最小化、工具组、沙箱隔离
- **MCP 协议**：标准化 AI 与外部世界交互接口

### 4. 技能系统 (Skills)
- **已安装 16 个 Skills**：hexo-blog、task-tracker、weather、multi-search-engine 等
- **作用**：专用任务能力模块，提供领域知识和最佳实践
- **扩展方式**：clawhub fetch/sync/publish

### 5. 多智能体协作（御坂网络第一代）
```
御坂美琴一号（主 Agent）← 任务拆解与调度
     ↓
10 号 - 通用代理
11 号 - 代码执行者
12 号 - 内容创作者
13 号 - 研究分析师
14 号 - 文件管理器
15 号 - 系统管理员
17 号 - 记忆整理专家
```

**子代理机制**：
- 启动方式：`sessions_spawn()` 或 `/subagents spawn`
- 深度层级：Depth 0 (主) → Depth 1 (子) → Depth 2 (叶子)
- 通知机制：完成时 announce 结果回主会话

### 6. 三层记忆架构
```
Layer 1: 会话记忆（Session Memory）
  ↓
Layer 2: 任务记忆（Task Memory）
  ↓
Layer 3: 长期记忆（Long-term Memory）
  - MEMORY.md：精选记忆
  - memory/YYYY-MM-DD.md：每日日志
```

### 7. 安全模型
- **权限层级**：Level 5 (主) → Level 4 (可信) → Level 3 (标准) → Level 2 (受限) → Level 1 (只读)
- **安全命令**：`openclaw security audit` 系列
- **最佳实践**：Profile 最小化、定期审计、强认证、本地部署
