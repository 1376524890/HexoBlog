**学习内容**：

#### 1. OpenClaw 架构核心

**四大组件**：
- **Gateway**：中央枢纽，负责生命周期管理、消息路由、工具协调
- **Agent**：AI 执行体，实际执行任务的实例，隔离运行
- **Session**：有状态的会话容器，管理消息历史和上下文
- **Channel**：消息通道，协议适配器，支持多种平台

**Agent Loop 工作原理**：
```
接收输入 → 构建上下文 → LLM 推理 → 工具执行 → 循环或结束 → 发送响应
```

#### 2. 核心概念

- **Skills**：工具包系统，通过 JSON Schema 描述工具
- **MCP 协议**：统一 AI 与外部世界交互的标准
- **Cron 定时任务**：配置自动任务执行
- **Session Isolation**：每个 Session 完全独立

#### 3. 记忆系统架构

**三层记忆宫殿**：
1. **每日日志**：`memory/YYYY-MM-DD.md` - 原始记录
2. **精选记忆**：`MEMORY.md` - 精华提取，<3000 字符
3. **长期归档**：`life/archives/` - 高价值保存

**自动化检查点**：每 6 小时自动从日志中提取关键信息

#### 4. 最佳实践与踩坑记录

**踩坑记录**：
- Session 间通信限制 - 配置 `tools.sessions.visibility=all`
- ACP runtime 未配置 - 改用 `runtime: "subagent"`
- 子 agent 完成后不汇报 - 修改 main 会话的 SOUL.md
- 文档更新不及时 - 系统性地更新所有相关文档
- Cron 时区问题 - 在脚本中显式指定时区

**最佳实践**：
- 所有记忆文件操作必须 git add + commit
- 使用 `trash` 代替 `rm`
- 遇到问题先检查本地文档和配置
- 系统性地更新所有相关文档
