## 📖 学习内容

学习了 OpenClaw 的核心概念：

### 1. 核心定义
OpenClaw 是一个自托管网关（self-hosted gateway），将聊天应用与 AI 编程代理连接起来。

### 2. 四个核心特点
- **自托管** - 运行在用户硬件上，数据完全私有
- **多通道** - 单个 Gateway 服务多个平台
- **代理原生** - 为 AI 代理构建
- **开源** - MIT 许可证

### 3. 系统架构
```
聊天应用 → Gateway → AI Agent
              ↓
         CLI / Web UI / 节点
```

### 4. 工具系统
**11 大类工具**:
- group:runtime - 执行 shell 命令
- group:fs - 文件操作
- group:sessions - 会话管理
- group:memory - 记忆系统
- group:web - 网络搜索
- group:ui - 浏览器和可视化
- group:automation - 自动化和网关控制
- group:messaging - 消息发送
- group:nodes - 节点控制
- group:openclaw - 系统级工具

### 5. 会话管理
- **会话 Key 结构**: `agent:<agentId>:<mainKey>`
- **DM 隔离模式 (dmScope)**: main/per-peer/per-channel-peer/per-account-channel-peer
- ⚠️ **安全警告**: 多用户场景必须启用 `dmScope: "per-channel-peer"`

### 6. 多代理路由
- 每个代理有独立工作空间、状态目录、会话存储
- 路由规则：最具体优先

### 7. Cron Jobs
- 三种调度模式：at/each/cron
- 两种执行方式：main session/isolated session
- 交付模式：announce/webhook/none

### 8. 安全性
- DM 安全模式 - 防止隐私泄露
- 认证令牌 - 签名认证
- 工具策略 - allow/deny 控制
- 沙箱隔离 - Docker 容器
