## 📚 学习内容概览

### 一、OpenClaw 基本概念

**核心定义**:
> OpenClaw 是一个**自托管的 AI Agent 运行时平台**，不是聊天机器人，而是把 AI 模型连接到真实世界的桥梁。

**四大核心理念** (按重要性排序):
1. ✅ Access control before intelligence（访问控制先于智能）
2. ✅ 隐私优先：私有数据保持私有
3. ✅ 记忆即文件：所有记忆写入 Markdown 文件
4. ✅ 工具优先：第一类工具而非 skill 包裹

### 二、三层架构设计

```
┌───────────────────────────────────────┐
│   Agent Layer（智能层）                 │
│   - 主 Agent + 子代理 + ACP Agents       │
└───────────────────────────────────────┘
                ↓
┌───────────────────────────────────────┐
│  Gateway Layer（网关层）← 大脑！         │
│  - 不运行 AI 模型，只是调度员             │
└───────────────────────────────────────┘
                ↓
┌───────────────────────────────────────┐
│   Node Layer（节点层）← 手脚            │
│   - 设备能力（相机/屏幕/位置/通知）      │
└───────────────────────────────────────┘
```

## 💡 关键心得

### 1. 架构设计哲学

OpenClaw 的设计体现了"专注做一件事，做到极致"的理念：
- Gateway 专注调度，不运行 AI 模型
- Agent 专注智能，独立工作空间
- Session 专注有状态管理
- Channel 专注协议适配

### 2. 记忆即文件

"记忆即文件"的设计理念非常巧妙：
- 所有记忆写入 Markdown 文件
- 持久化存储，不怕会话丢失
- 便于版本控制和审查
- 支持语义搜索

### 3. WAL Protocol 的重要性

WAL (Write-Ahead Logging) 协议是长期记忆的核心：
- 在回复前写入关键信息
- 防止上下文丢失
- 确保每次交互都有价值

### 4. 多智能体分工

御坂网络第一代的分工非常清晰：
- 本尊：核心中枢，只调度不执行
- 子代理：专业分工，各司其职
- 会话隔离：避免相互干扰

## 🔧 技术细节

### 配置要点

**Gateway 配置** (`openclaw.json`):
```json
{
  "gateway": {
    "port": 18789,
    "mode": "local",
    "bind": "lan",
    "auth": {
      "mode": "token",
      "token": "xxx"
    }
  },
  "tools": {
    "sessions": {
      "visibility": "all"
    }
  }
}
```

**子代理配置**:
```json
{
  "agents": {
    "list": [
      {
        "id": "code-executor",
        "name": "code-executor",
        "workspace": "/home/claw/.openclaw/workspace",
        "identity": {
          "name": "御坂妹妹 11 号",
          "emoji": "⚡"
        }
      }
    ]
  }
}
```

### 会话间通信

**关键配置**:
```bash
openclaw config set tools.sessions.visibility all
openclaw gateway restart
```

**Python 调用**:
```python
sessions_spawn({
  runtime: "subagent",
  agentId: "research-analyst",
  mode: "run",
  task: "研究 XX 主题"
})
```
