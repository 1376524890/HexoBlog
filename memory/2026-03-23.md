## 🧠 核心知识点掌握

### 1. OpenClaw 定义

> **OpenClaw 是 AI Agent 运行时平台**，核心是**智能网关（Runtime Gateway）**。  
> **它不是聊天机器人，而是把 AI 连接到真实世界的桥梁。**

**关键点**:
- 自托管网关（Self-hosted Gateway）
- 连接各种聊天应用（WhatsApp、Telegram、Discord、iMessage、飞书等）
- 提供 AI 编码助手（AI coding agents）
- 一个 Gateway 服务多个聊天平台

### 2. 四大核心理念（必背）⭐⭐⭐⭐⭐

1. **Access control before intelligence**（访问控制先于智能）
2. **隐私优先**：私有数据保持私有
3. **记忆即文件**：所有记忆写入 Markdown 文件
4. **工具优先**：第一类工具而非 skill 包裹

### 3. 三层架构

```
┌─────────────────────────────────────────────────────────┐
│              Agent Layer（智能层）                        │
│  - Main Agent（主 Agent）                               │
│  - Subagents（子代理）                                  │
│  - ACP Agents（编码代理）                               │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│           Gateway Layer（网关层）← 大脑！                  │
│  - 控制平面、策略层、路由                               │
│  - 身份认证、工具策略、会话管理                         │
│  - 频道适配器（15+ 个聊天平台）                          │
│  ⚠️ Gateway 本身不运行 AI 模型，只是调度员                  │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│              Node Layer（节点层）← 手脚                   │
│  - 远程执行表面                                         │
│  - 设备能力（摄像头、屏幕、通知、位置）                 │
│  - macOS companion app                                  │
│  - iOS 和 Android nodes                                  │
└─────────────────────────────────────────────────────────┘
```

**记忆口诀**: 智能层（脑）→ 网关层（路由）→ 节点层（手）

### 4. 四大核心组件

#### Gateway（网关）
- **作用**: 大脑、路由器
- **关键点**: 不运行 AI 模型，只是调度员
- **默认端口**: 18789

#### Agent（智能体）
- **Agent Loop**: 接收输入 → 构建上下文 → LLM 推理 → 工具执行 → 发送响应
- **组成**: 身份、配置、状态、运行时

#### Session（会话）
- **作用**: 有状态容器
- **键格式**: `agent:<agentId>:main` / `direct:<peerId>` / `group:<id>` / `channel:<id>`

#### Channel（频道）
- **支持平台**: 15+ 个（Telegram、Discord、飞书、WhatsApp 等）

## 💡 学习心得

### 核心设计亮点

1. **隔离性**: 每个 agent 完全独立，互不干扰
2. **灵活性**: 支持多种部署场景（单用户、多用户、家庭、企业）
3. **安全性**: 多层权限控制，沙箱隔离，设备认证
4. **可维护性**: 自动会话清理，配置热重载，性能优化

### 系统架构优势

系统通过 WebSocket 协议统一所有通信，Gateway 作为中央枢纽管理所有渠道连接，Agent 作为独立大脑处理业务逻辑，Node 作为设备延伸扩展物理能力。这种架构非常清晰且可扩展。

### 御坂网络设计

多智能体架构通过专业分工提高效率，主 Agent 负责任务拆解和调度，子代理处理具体领域任务。这种设计既保证了专业性，又保持了系统的灵活性。
