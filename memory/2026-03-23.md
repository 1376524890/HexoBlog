**核心学习内容**:

#### 核心定义
- OpenClaw 是 AI Agent 运行时平台
- 核心是智能网关（Runtime Gateway）
- 不是聊天机器人，而是把 AI 连接到真实世界的桥梁

#### 四大核心理念
1. Access control before intelligence（访问控制先于智能）⭐⭐⭐⭐⭐
2. Privacy First：私有数据保持私有
3. Memory as Files：所有记忆写入 Markdown 文件
4. Tools First：第一类工具而非 skill 包裹

#### 三层架构
- **Agent Layer（智能层）**: Main Agent、Subagents、ACP Agents
- **Gateway Layer（网关层）**: 控制平面、策略层、路由（大脑！）
- **Node Layer（节点层）**: 远程执行表面（手脚）

#### 四大核心组件
1. **Gateway**: 不运行 AI 模型，只是调度员，默认端口 18789
2. **Agent**: 执行 AI 任务，核心是 Agent Loop
3. **Session**: 有状态容器，支持多种 Key 格式
4. **Channel**: 协议适配器，支持 15+ 个平台

#### 工具系统（8 大分类）
- Runtime、Filesystem、Session、Memory、Web、UI、Node、Messaging

#### 御坂网络第一代
- 1 个主 Agent（御坂美琴一号）
- 7 个子代理（10-17 号）
- 专业分工，提高效率

#### 三层记忆架构
- Layer 1: 会话记忆
- Layer 2: 任务记忆
- Layer 3: 长期记忆（MEMORY.md + memory/YYYY-MM-DD.md）

#### 安全模型
- 5 级权限控制
- 沙箱隔离
- 安全审计命令
- 设备身份认证

#### 支持平台（15+ 个）
- Telegram、Discord、Slack、WhatsApp、飞书、iMessage 等

## ⚠️ 需要记住的重点

1. **Gateway 不运行 AI 模型**，只是调度员
2. **模型拥有决策权**，主动决定调用什么工具
3. **dmScope 配置** 对于多用户 DM 很重要
4. **安全审计命令**: `openclaw security audit --deep`
5. **Session Key 格式**: `agent:<agentId>:main` 等
6. **三层记忆架构**: WAL Protocol 确保数据完整性
