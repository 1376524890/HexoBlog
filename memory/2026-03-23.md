### 核心知识点

#### 1. OpenClaw 是什么？
- **定义**: AI Agent 运行时平台，不是聊天机器人
- **定位**: 把 AI 连接到真实世界的桥梁
- **核心**: 自托管网关（self-hosted gateway）
- **功能**: 连接聊天应用（WhatsApp、Telegram、Discord、iMessage 等）与 AI 编码代理

#### 2. 核心理念（四大原则）
1. **Access control before intelligence** - 访问控制先于智能 ⭐⭐⭐⭐⭐
2. **隐私优先** - 私有数据保持私有
3. **记忆即文件** - 所有记忆写入 Markdown 文件
4. **工具优先** - 第一类工具而非 skill 包裹

#### 3. 三层架构
```
Agent Layer（智能层）
  ↓
Gateway Layer（网关层）← 大脑！不运行 AI，只是调度员
  ↓
Node Layer（节点层）← 手脚
```

#### 4. 四大核心组件
| 组件 | 作用 | 关键点 |
|------|------|--------|
| Gateway | 大脑、路由器 | 不运行 AI 模型，只是调度员 |
| Agent | 执行 AI 任务 | 身份 + 配置 + 状态 + 运行时 |
| Session | 有状态容器 | 消息历史、上下文、工具状态 |
| Channel | 协议适配器 | Telegram、Discord、飞书等 |

#### 6. 三层记忆架构
```
Layer 1: 会话记忆（Session Memory）
- 当前会话上下文
- 临时决策和中间结果
    ↓ 同步关键信息
Layer 2: 任务记忆（Task Memory）
- 任务计划文件
- 子代理执行结果
    ↓ 同步重要发现
Layer 3: 长期记忆（Long-term Memory）
- MEMORY.md：精选记忆
- memory/YYYY-MM-DD.md：每日日志
```

#### 7. 安全模型（五级权限）
| 级别 | 名称 | 权限 |
|------|------|------|
| Level 5 | 主 Agent | 完全权限 |
| Level 4 | 可信子 Agent | 受限系统权限（需批准）|
| Level 3 | 标准子 Agent | 标准开发权限 |
| Level 2 | 受限子 Agent | 严格受限权限 |
| Level 1 | 只读子 Agent | 只读访问 |

#### 8. Session 管理
- **Session Key 格式**: 不同来源使用不同格式（直接聊天、群组、Cron、Webhook 等）
- **dmScope 配置**: 支持 `main`、`per-peer`、`per-channel-peer`、`per-account-channel-peer`
- **安全建议**: 多用户场景使用 `dmScope: per-channel-peer` 防止隐私泄露
- **维护策略**: 支持自动清理（pruneAfter、maxEntries、rotateBytes）

#### 9. Cron 任务系统
- **Cron vs Heartbeat**:
  - Cron: 精确时间、隔离执行、直接输出
  - Heartbeat: 批量处理、对话上下文、时间可漂移

#### 10. Channels 系统
- 支持超过 20 种聊天渠道
- 每种渠道有自己的 API 适配器和配置
- Feishu、Telegram、WhatsApp 是主要使用渠道
