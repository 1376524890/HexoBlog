## 📚 一、OpenClaw 是什么

### 1.1 核心定义
OpenClaw 是一个**个人 AI 助手**平台，运行在自己的设备上。它是一个单一控制平面（Gateway），可以连接到你已经使用的各种消息渠道。

**核心口号**: "EXFOLIATE! EXFOLIATE!" 🦞

### 1.2 项目定位
| 特点 | 说明 |
|------|------|
| **个人化** | 单用户/单信任边界，不是多租户共享 AI |
| **本地化** | 运行在本地设备上，数据本地存储 |
| **混合模型** | 支持本地模型和远程 API 模型混合使用 |
| **开源开放** | MIT 许可证，活跃开源社区 |

### 1.3 支持的渠道（20+）
- **即时通讯**: WhatsApp、Telegram、Slack、Discord
- **消息平台**: Google Chat、Signal、iMessage、Microsoft Teams、Matrix
- **企业应用**: Feishu、LINE、Mattermost、Nextcloud Talk、Nostr
- **其他**: Synology Chat、Tlon、Twitch、Zalo、WebChat
- **原生支持**: macOS、iOS、Android

## 🏗️ 二、架构设计

### 2.1 整体架构
```
各种消息渠道 (WhatsApp/Telegram/Slack 等)
                ↓
    ┌─────────────────────────────┐
    │         Gateway             │
    │   (控制平面，WebSocket)      │
    │    ws://127.0.0.1:18789     │
    └──────────────┬──────────────┘
                   ↓
        ┌──────────┼──────────┐
        ↓          ↓          ↓
    Pi 代理      CLI       WebChat
    (RPC 模式)   (命令行)   (Web 界面)
        ↓          ↓          ↓
    macOS 应用   工具调用   远程访问
    iOS 节点
    Android 节点
```

### 2.2 核心组件详解

#### Gateway (网关)
- **角色**: 长期运行的守护进程
- **职责**:
  - 维护所有消息渠道的连接
  - 暴露 WebSocket API
  - 验证请求（JSON Schema 类型检查）
  - 发射事件（agent、chat、presence、health、heartbeat、cron）

#### Clients (客户端)
| 客户端 | 用途 |
|--------|------|
| macOS 应用 | 原生桌面体验 |
| CLI 命令行 | 命令行操作 |
| Web 控制界面 | 浏览器控制 |
| WebChat | Web 聊天界面 |

#### Nodes (节点)
- **支持平台**: macOS/iOS/Android/无头模式
- **连接方式**: 通过 WebSocket 连接，声明 `role: node`
- **提供能力**:
  - Canvas (可视化工作空间)
  - 相机 snap/clip (拍照/录像)
  - 屏幕录制
  - 位置信息
  - 通知管理
- **安全要求**: 需要配对 (pairing) 批准

### 2.3 WebSocket 协议
```json
// 第一帧必须是 connect 握手
{type:"connect", token:"your-token"}

// 握手后请求
{type:"req", id: 1, method: "sessions_list", params: {}}

// 响应
{type:"res", id: 1, ok: true, payload: [...]}

// 事件推送
{type:"event", event: "chat.message", payload: {...}, seq: 1}
```

## 🔐 三、安全模型

### 3.1 信任边界原则
**核心理念**: 个人助手安全模型，单一信任边界

- ✅ **支持**: 一个用户/一个信任边界 per gateway
- ❌ **不支持**: 多个相互敌对的共享一个 gateway
- 💡 **解决方案**: 如果需要混合信任，使用多个 gateway 或多个 OS 用户/主机

### 3.2 关键安全检查清单

#### 基础安全配置 (60 秒加固)
```json5
{
  gateway: {
    mode: "local",
    bind: "loopback",
    auth: { mode: "token", token: "replace-with-long-random-token" },
  },
  session: {
    dmScope: "per-channel-peer",  // 隔离 DM 会话
  },
  tools: {
    profile: "messaging",
    deny: ["group:automation", "group:runtime", "group:fs", "sessions_spawn", "sessions_send"],
    fs: { workspaceOnly: true },
    exec: { security: "deny", ask: "always" },
    elevated: { enabled: false },
  },
  channels: {
    whatsapp: { dmPolicy: "pairing", groups: { "*": { requireMention: true } } },
  },
}
```

#### 文件权限要求
```bash
chmod 700 ~/.openclaw
chmod 600 ~/.openclaw/openclaw.json
chmod 600 ~/.openclaw/credentials/*
```

#### DM 访问控制模式
| 模式 | 说明 |
|------|------|
| `pairing` (默认) | 未知发送者需要配对代码 |
| `allowlist` | 只有白名单可以 DM |
| `open` | 任何人可以 DM（需要显式允许） |
| `disabled` | 禁用 DM |

### 3.3 提示词注入防护

**防护原则**:
1. DM 访问控制（配对/白名单）
2. 组内 mention gating（避免公房间常开）
3. 将链接、附件、粘贴的指令视为威胁
4. 沙箱运行敏感工具执行
5. 限制高风险工具（`exec`、`browser`、`web_fetch`、`web_search`）
6. **模型选择很重要**: 使用最新一代、指令强化的模型

**不安全的配置标志**（应保持未设置/false）:
- `hooks.mappings[].allowUnsafeExternalContent`
- `hooks.gmail.allowUnsafeExternalContent`

## ⚙️ 四、核心功能

### 4.1 多渠道集成实现

| 渠道 | 实现库 | 认证方式 |
|------|--------|----------|
| WhatsApp | Baileys | 设备链接 |
| Telegram | grammY | bot token |
| Slack | Bolt | bot + app tokens |
| Discord | discord.js | bot token |
| Signal | signal-cli | 配置认证 |
| iMessage | BlueBubbles (推荐) | macOS 认证 |
| Feishu | 原生集成 | 企业权限 |

### 4.2 多智能体路由

**核心能力**: 将不同的入站渠道/账户/对等体路由到隔离的智能体

#### 配置示例
```json5
{
  agents: {
    list: [
      {
        name: "Fast Chat",
        model: "anthropic/claude-sonnet-4-5",
        tools: { profile: "messaging" },
      },
      {
        name: "Coding Agent",
        model: "anthropic/claude-opus-4-6",
        tools: { profile: "minimal", allowed: ["sessions_spawn"] },
      },
    ],
  },
}
```

#### 会话路由规则
- **DM**: `agent:<agentId>:<mainKey>` (默认共享)
- **组聊**: `agent:<agentId>:<channel>:group:<id>` (隔离)
- **线程**: 追加 `:topic:<threadId>`

### 4.3 会话管理

#### 会话范围 (`dmScope`)
| 范围 | 说明 | 适用场景 |
|------|------|----------|
| `main` (默认) | 所有 DM 共享主会话 | 个人单用户 |
| `per-peer` | 按发送者 ID 隔离 | 多用户但单渠道 |
| `per-channel-peer` | 按渠道 + 发送者隔离 | 推荐多用户收件箱 |
| `per-account-channel-peer` | 按账户 + 渠道 + 发送者隔离 | 推荐多账户 |

#### 会话维护参数
```json5
{
  session: {
    maintenance: {
      mode: "warn",        // warn | enforce
      pruneAfter: 2592000000,  // 30 天 (毫秒)
      maxEntries: 500,
      rotateBytes: 10485760,   // 10MB
      maxDiskBytes: 1073741824 // 可选，1GB
    }
  }
}
```

#### 会话重置策略
- **每日重置**: 默认 4:00 AM 本地时间
- **空闲重置**: 通过 `idleMinutes` 配置
- **命令重置**: 使用 `/new` 或 `/reset`

### 4.4 工具系统

#### 内置工具列表
| 工具 | 功能 |
|------|------|
| **Browser** | 浏览器控制（openclaw 管理的 Chrome/Chromium） |
| **Canvas** | 可视化工作空间（A2UI 推送/重置/评估） |
| **Nodes** | 相机 snap/clip、屏幕录制、位置、通知 |
| **Cron + Wakeups** | 定时任务 |
| **Webhooks** | 外部触发器 |
| **Gmail Pub/Sub** | Gmail 邮件触发 |
| **Sessions** | 智能体间通信 |

#### 工具配置文件
| 配置文件 | 包含工具 | 适用场景 |
|----------|----------|----------|
| `messaging` (默认) | 最小化工具集 | 日常聊天 |
| `minimal` | 仅消息 | 基本功能 |
| `coding` | 包含编程工具 | 代码开发 |
| `developer` | 全工具集 | 开发者 |
| `admin` | 管理员权限 | 系统管理 |

### 4.5 技能系统 (Skills)

#### 技能来源
1. **内置技能** - 开箱即用
2. **工作区技能** - `~/.openclaw/workspace/skills/`
3. **ClawHub 技能** - 可自动从注册表获取

#### 技能规范
- 每个技能在独立文件夹中
- 包含 `SKILL.md` 定义技能行为和权限
- 支持权限边界控制

#### 安全注意
- 技能文件夹被视为**可信代码**
- 只从可信来源安装技能
- 使用显式 `plugins.allow` 白名单
- 修改后需要重启 Gateway

## 🔄 五、安装与部署

### 5.1 运行环境要求

| 要求 | 最低配置 | 推荐配置 |
|------|----------|----------|
| Node.js | >= 22 | 最新 LTS |
| 包管理器 | npm / pnpm / bun | pnpm |
| OS | macOS / Linux / Windows(WSL2) | macOS / Linux |
| RAM | 512MB | 4GB+ |
| CPU | 1 核心 | 4 核心+ |
| 磁盘 | 500MB | 10GB+ |

### 5.2 安装方式

#### 全局安装（推荐）
```bash
curl -fsSL https://openclaw.ai/install.sh | bash
openclaw onboard --install-daemon
```

#### Git 安装（可编辑源码）
```bash
curl -fsSL https://openclaw.ai/install.sh | bash -s -- --install-method git
```

#### 从源码构建（开发）
```bash
git clone https://github.com/openclaw/openclaw.git
cd openclaw
pnpm install
pnpm build
pnpm ui:build
openclaw onboard --install-daemon

# 开发模式（热重载）
pnpm gateway:watch
```

#### Docker 部署
```bash
docker run -d \
  -v ~/.openclaw:/root/.openclaw \
  -p 18789:18789 \
  openclaw/openclaw
```

### 5.3 部署选项对比

| 选项 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| **本地部署** | 数据本地化、配置简单 | 依赖本地设备在线 | 个人使用 |
| **远程部署 (VPS)** | 24/7 运行、可远程访问 | 需要网络配置 | 需要持续运行 |
| **Docker 部署** | 隔离性好、易于部署 | 网络配置复杂 | 容器化环境 |

### 8.3 安全实践

#### 1. 文件权限设置
```bash
chmod 700 ~/.openclaw
chmod 600 ~/.openclaw/openclaw.json
chmod 600 ~/.openclaw/credentials/*
```

#### 2. 网络暴露控制
- **推荐**: `gateway.bind: "loopback"`（本地访问）
- **远程访问**: 使用 Tailscale Serve（推荐）或 SSH 隧道
- **避免**: `0.0.0.0` 公开暴露

#### 3. 访问控制策略
- **DM**: 使用 `pairing` 或 `allowlist`
- **组聊**: 启用 `requireMention`
- **工具**: 限制高风险工具

#### 4. 定期安全审计
```bash
openclaw security audit
openclaw security audit --deep
openclaw security audit --fix
openclaw security audit --json
```

### 8.4 故障排除

#### 第一步：快速检查
```bash
openclaw status
openclaw models status
openclaw gateway status
```

#### 第二步：深度检查
```bash
openclaw status --all
openclaw health --verbose
openclaw logs --follow
```

#### 第三步：诊断修复
```bash
openclaw doctor
openclaw doctor --generate-gateway-token
```

#### 常见错误处理

| 错误 | 原因 | 解决方法 |
|------|------|----------|
| **"Wake up my friend" stuck** | 网关未启动或模型认证失败 | 重启网关，检查模型认证 |
| **Unauthorized** | token 过期或无效 | 检查 token，重新生成 |
| **Model is not allowed** | 模型不在 allowlist | 添加模型到 allowlist |
| **No credentials found** | provider 认证未配置 | 配置 provider 认证 |
