## 🎯 核心问题：OpenClaw 是什么？

**一句话**：OpenClaw 是一个**AI Agent 运行时平台**，也可以理解为**智能网关**。

**类比理解**：
- 传统网关：路由 HTTP 请求、协议转换、负载均衡
- OpenClaw 网关：路由用户消息、协议转换（Telegram/Discord→内部格式）、Agent 池管理

---

## 🏗️ 四大核心组件

| 组件 | 职责 | 类比 |
|------|------|------|
| **Gateway** | 中央调度器 | 大脑/路由器 |
| **Agent** | AI 执行体 | 具体的 AI 助手 |
| **Session** | 有状态容器 | 对话历史 |
| **Channel** | 消息通道 | Telegram/Discord 适配器 |

**核心流程（Agent Loop）**：
```
用户消息 → Gateway 路由 → 构建上下文 → LLM 推理 → 
决定：直接回复 or 调用工具 → 工具执行 → 发送响应
```

## 🛠️ Skill 系统与 MCP

### MCP 是什么？
**Model Context Protocol** - 标准化 AI 与外部世界的交互接口。

### OpenClaw Skills
- 每个 Skill 是一个独立工具包
- 通过 JSON Schema 描述工具
- 可发布到社区共享

### 典型案例
1. **任务追踪** - 持久化任务记忆，重启后不丢失
2. **股票查询** - 多数据源 + 本地缓存 + 自动降级
3. **自动备份** - cron 定时 + Python 脚本 + 7 天保留

**方案**：2×4090 + vLLM + Qwen3.5-35B-A3B-FP8

**效果对比**：
| 指标 | API | 本地 vLLM |
|------|-----|-----------|
| 首字延迟 | 500ms | 200ms |
| 吞吐量 | 50 tok/s | 120 tok/s |
| 成本 | ¥200/月 | ¥50/月（电费） |

**关键配置**：

## 📚 核心经验总结

### 技能开发
1. **简单够用** - 先跑通最简单的方案
2. **格式可读** - Markdown/JSON，人机都能读
3. **自动化优先** - 脚本优于手动
4. **多数据源降级** - 防止单一故障

### 记忆系统
1. **三层架构** - 原始/精选/归档
2. **自动检查点** - LLM 自动提取
3. **定期归档** - 保持精简
4. **记录决策** - 不仅"做什么"，还要"为什么"

### 多智能体
1. **会话配置** - `tools.sessions.visibility=all`
2. **子 agent 回复** - 发送给主 session，不是直接给用户
3. **文档同步** - 更新系统后更新所有文档

### 本地部署
1. **硬件是一次性** - API 是持续成本
2. **SSH 自动重连** - 健康检查是关键
3. **量化选择** - FP8 是平衡点
